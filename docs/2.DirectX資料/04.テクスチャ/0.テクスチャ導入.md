# **テクスチャ**

---

## **テクスチャとは**

テクスチャとは、3D モデルや 2D スプライトに貼り付ける「画像データ」を指す。  
床・壁・キャラクターの外見は、ほとんどテクスチャで表現する。  

DirectX12 でテクスチャを使うには、  
単に画像ファイルを読み込むだけでなく、  
GPU が高速にアクセスできる専用メモリ（VRAM）へ転送する手順が必要となる。  

DirectX12 でテクスチャ（画像データ）を読み込むには、  
画像ファイルをピクセルデータへ変換する処理が必要となる。  
この処理には、Microsoft が提供する拡張ライブラリ  
<span style ="color: red;">**DirectXTex**</span> を使用する。  

---

## **DirectXTex とは**

DirectXTex は  
PNG・DDS など多様な画像フォーマットの読み込みや  
テクスチャ変換処理を提供する公式ライブラリ。  
ソースコードは GitHub で公開されている。  

本資料では Git の <span style ="color: red;">**サブモジュール**</span> という機能を使い、  
メインブランチのソースコードを  
自身のプロジェクトへ取り込む手順を示す。  

---

## **サブモジュールとは**

サブモジュールとは、自分の Git リポジトリの中に  
別の Git リポジトリを「参照」として組み込む仕組みを指す。  

- ライブラリのソースをコピーせずに管理できる
- 取り込んだ時点のコミットを固定して記録できる
- 本家リポジトリの更新を後から取り込む事もできる

ソースコードを直接コピーすると、  
「どのバージョンをどこから持ってきたか」が分からなくなる。  
サブモジュールを使えば、取り込み元の URL とコミットが  
自分のリポジトリに記録されるため、管理が容易となる。  

---

## **サブモジュールの追加手順**

自身のプロジェクトのルートフォルダで以下の手順を実行する。  
前提として、自身のプロジェクトが Git 管理されている  
必要がある点に注意する。  

### **サブモジュールとして追加する**

external フォルダの下に取り込む例を示す。  

```
git submodule add https://github.com/microsoft/DirectXTex.git external/DirectXTex
```

`git submodule add` は、指定した URL のリポジトリを  
クローンした上で、参照情報を自分のリポジトリに登録する。  
ブランチを指定しない場合は既定のブランチ  
（DirectXTex では main ブランチ）が取り込まれる。  

### **登録内容を確認する**

追加が完了すると、リポジトリのルートに  
<span style ="color: red;">**.gitmodules**</span> というファイルが生成される。  

```
[submodule "external/DirectXTex"]
	path = external/DirectXTex
	url = https://github.com/microsoft/DirectXTex.git
```

.gitmodules には「どのフォルダに・どの URL の  
リポジトリを取り込んだか」が記録される。  

### **コミットする**

.gitmodules とサブモジュールの参照をコミットする。  

```
git add .gitmodules external/DirectXTex
git commit -m "DirectXTex をサブモジュールとして追加"
```

---

## **クローン時の注意**

サブモジュールを含むリポジトリを新しくクローンした直後は、  
サブモジュールのフォルダは空の状態となる。  
以下のいずれかの方法でソースコードを取得する。  

```
# クローン済みのリポジトリで後から取得する場合
git submodule update --init --recursive

# クローンと同時に取得する場合
git clone --recursive <自身のリポジトリの URL>
```

---

## **プロジェクトへの組み込み**

取り込んだ DirectXTex を Visual Studio で使うには、  
以下の設定を行う。  

1. `external/DirectXTex/DirectXTex/DirectXTex_Desktop_2022.vcxproj` を  
   ソリューションに「既存のプロジェクト」として追加する  
2. 自身のプロジェクトの「参照」に DirectXTex を追加する  
3. プロジェクトのインクルードディレクトリに  
   `external/DirectXTex/DirectXTex` を追加する  

これで `#include "DirectXTex.h"` が利用可能となる。  

---

## **注意点**

<span style ="color: red;">**最も重要なポイント**</span> は、サブモジュールが  
「ソースのコピー」ではなく「参照の記録」という点。  
クローン直後はフォルダが空となるため、  
`git submodule update --init --recursive` を忘れないよう注意する。
