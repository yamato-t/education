# **テクスチャ**

---

## **テクスチャとは**

テクスチャとは、3D モデルや 2D スプライトに貼り付ける「画像データ」を指す。  
床・壁・キャラクターの外見は、ほとんどテクスチャで表現する。  

DirectX12 でテクスチャを使うには、  
単に画像ファイルを読み込むだけでなく、  
GPU が高速にアクセスできる専用メモリ（VRAM）へ転送する手順が必要となる。  

---

## **テクスチャ利用の全体的な流れ**

テクスチャを描画で使えるようにするまでの手順を以下に示す。  

1. **GPUリソースの作成**  
   VRAM 上にテクスチャ用の領域を確保する  

2. **テクスチャデータの読み込み**  
   CPU 側に画像ファイルをロードし、ピクセルデータを取得する  

3. **アップロードバッファの準備**  
   CPU → GPU のデータ転送に使う中継バッファを作成する  

4. **VRAM へのコピー**  
   コマンドリストでアップロードバッファから VRAM へ転送する  

5. **シェーダリソースビュー（SRV）の作成**  
   シェーダがテクスチャを参照するためのビューを登録する  

6. **シェーダとルートシグネチャの設定**  
   シェーダでテクスチャを受け取れるよう定義する  

---

## **GPUリソースの作成**

テクスチャの実体を VRAM に置くためのリソースを作成する。  
ヒープタイプは <span style ="color: red;">**D3D12_HEAP_TYPE_DEFAULT**</span> を使用する。  
DEFAULT ヒープは GPU 専用メモリで、描画時の読み取り速度が最も速い。  
ただし CPU から直接書き込めないため、後述のアップロードバッファが必要となる。  

```c++
D3D12_HEAP_PROPERTIES defaultHeap{};
defaultHeap.Type = D3D12_HEAP_TYPE_DEFAULT;

D3D12_RESOURCE_DESC texDesc{};
texDesc.Dimension        = D3D12_RESOURCE_DIMENSION_TEXTURE2D;
texDesc.Width            = img->width;              // 横幅（ピクセル）
texDesc.Height           = img->height;             // 縦幅（ピクセル）
texDesc.DepthOrArraySize = 1;
texDesc.MipLevels        = 1;
texDesc.Format           = DXGI_FORMAT_R8G8B8A8_UNORM; // RGBA 各 8bit
texDesc.SampleDesc.Count = 1;
texDesc.Layout           = D3D12_TEXTURE_LAYOUT_UNKNOWN;
texDesc.Flags            = D3D12_RESOURCE_FLAG_NONE;

ID3D12Resource* textureResource;
device->CreateCommittedResource(
    &defaultHeap,
    D3D12_HEAP_FLAG_NONE,
    &texDesc,
    D3D12_RESOURCE_STATE_COPY_DEST, // コピー先として初期化
    nullptr,
    IID_PPV_ARGS(&textureResource)
);
```

## **テクスチャデータの読み込み**

画像ファイルを CPU 上に読み込み、ピクセルデータを取得する。  
ここでは Microsoft が提供する拡張ライブラリ  
<span style ="color: red;">**DirectXTex**</span> を使用する。  
DirectXTex（DirectX Texture Processing Library）は  
PNG・DDS など多様な画像フォーマットの読み込みや  
テクスチャ変換処理を提供する公式ライブラリ。  

```c++
#include "DirectXTex.h"

DirectX::ScratchImage image;

// WIC 経由で PNG / JPG 等を読み込む
DirectX::LoadFromWICFile(
    L"texture.png",           // ファイルパス（ワイド文字列）
    DirectX::WIC_FLAGS_NONE,
    nullptr,
    image
);

// ピクセルデータへのポインタを取得
const DirectX::Image* img = image.GetImages();
// img->pixels    : ピクセルデータの先頭アドレス
// img->width     : 横幅（ピクセル）
// img->height    : 縦幅（ピクセル）
// img->rowPitch  : 1 行分のバイト数
```

取得した `img` の情報を、この後のアップロードバッファへのコピーで利用する。  

---

## **アップロードバッファの準備とデータ転送**

CPU から VRAM に直接データを書き込む事はできない。  
そのため、CPU と GPU 双方がアクセスできる  
<span style ="color: red;">**アップロードバッファ（UPLOAD ヒープ）**</span> を中継として使用する。  

```
CPU メモリ → アップロードバッファ → VRAM（テクスチャリソース）
```

### **アップロードバッファのサイズ取得**

テクスチャデータは GPU 内部のフォーマットに合わせたサイズが必要となる。  
`GetCopyableFootprints` で必要なバッファサイズを取得する。  

```c++
UINT64 uploadSize = 0;
D3D12_PLACED_SUBRESOURCE_FOOTPRINT footprint{};
UINT numRows = 0;
UINT64 rowSizeBytes = 0;

device->GetCopyableFootprints(
    &texDesc, 0, 1, 0,
    &footprint,
    &numRows,
    &rowSizeBytes,
    &uploadSize     // 必要なアップロードバッファサイズ
);
```

### **アップロードバッファの作成**

```c++
D3D12_HEAP_PROPERTIES uploadHeap{};
uploadHeap.Type = D3D12_HEAP_TYPE_UPLOAD; // CPU が書き込めるメモリ

D3D12_RESOURCE_DESC uploadDesc{};
uploadDesc.Dimension        = D3D12_RESOURCE_DIMENSION_BUFFER;
uploadDesc.Width            = uploadSize;
uploadDesc.Height           = 1;
uploadDesc.DepthOrArraySize = 1;
uploadDesc.MipLevels        = 1;
uploadDesc.Format           = DXGI_FORMAT_UNKNOWN;
uploadDesc.SampleDesc.Count = 1;
uploadDesc.Layout           = D3D12_TEXTURE_LAYOUT_ROW_MAJOR;
uploadDesc.Flags            = D3D12_RESOURCE_FLAG_NONE;

ID3D12Resource* uploadBuffer;
device->CreateCommittedResource(
    &uploadHeap,
    D3D12_HEAP_FLAG_NONE,
    &uploadDesc,
    D3D12_RESOURCE_STATE_GENERIC_READ,
    nullptr,
    IID_PPV_ARGS(&uploadBuffer)
);
```

### **CPU からピクセルデータを書き込む**

```c++
BYTE* mappedData = nullptr;
uploadBuffer->Map(0, nullptr, reinterpret_cast<void**>(&mappedData));

// ピクセルデータを行ごとにコピー
for (UINT row = 0; row < numRows; ++row)
{
    memcpy(
        mappedData + footprint.Footprint.RowPitch * row,
        img->pixels + img->rowPitch * row,
        img->rowPitch // 1 行分のバイト数
    );
}

uploadBuffer->Unmap(0, nullptr);
// ScratchImage はスコープを外れると自動解放される
```

## **VRAM へのコピー**

コマンドリストを使って、アップロードバッファから  
VRAM 上のテクスチャリソースにコピーする。  

```c++
// コピー元（アップロードバッファ）の情報
D3D12_TEXTURE_COPY_LOCATION src{};
src.pResource       = uploadBuffer;
src.Type            = D3D12_TEXTURE_COPY_TYPE_PLACED_FOOTPRINT;
src.PlacedFootprint = footprint;

// コピー先（VRAM のテクスチャリソース）の情報
D3D12_TEXTURE_COPY_LOCATION dst{};
dst.pResource        = textureResource;
dst.Type             = D3D12_TEXTURE_COPY_TYPE_SUBRESOURCE_INDEX;
dst.SubresourceIndex = 0;

commandList->CopyTextureRegion(&dst, 0, 0, 0, &src, nullptr);
```

コピー後、テクスチャをシェーダが読める状態に遷移させる。  
これを <span style ="color: red;">**リソースバリア**</span> と呼び、  
GPU 内部でのリソースの「使用状態」を切り替える命令となる。  

```c++
D3D12_RESOURCE_BARRIER barrier{};
barrier.Type                   = D3D12_RESOURCE_BARRIER_TYPE_TRANSITION;
barrier.Transition.pResource   = textureResource;
barrier.Transition.StateBefore = D3D12_RESOURCE_STATE_COPY_DEST;
barrier.Transition.StateAfter  = D3D12_RESOURCE_STATE_PIXEL_SHADER_RESOURCE;
barrier.Transition.Subresource = D3D12_RESOURCE_BARRIER_ALL_SUBRESOURCES;

commandList->ResourceBarrier(1, &barrier);
```

コマンド実行・GPU 処理完了を確認してから  
アップロードバッファを解放してかまわない。

## **シェーダリソースビュー（SRV）の作成**

テクスチャリソースをシェーダが参照するために  
<span style ="color: red;">**SRV（Shader Resource View）**</span> を作成する。  
SRV は `D3D12_DESCRIPTOR_HEAP_FLAG_SHADER_VISIBLE` フラグを持つ  
ディスクリプタヒープに登録する必要がある。  

```c++
// SRV 用ディスクリプタヒープの作成
D3D12_DESCRIPTOR_HEAP_DESC srvHeapDesc{};
srvHeapDesc.Type           = D3D12_DESCRIPTOR_HEAP_TYPE_CBV_SRV_UAV;
srvHeapDesc.NumDescriptors = 1;
srvHeapDesc.Flags          = D3D12_DESCRIPTOR_HEAP_FLAG_SHADER_VISIBLE;

ID3D12DescriptorHeap* srvHeap;
device->CreateDescriptorHeap(&srvHeapDesc, IID_PPV_ARGS(&srvHeap));

// SRV の作成
D3D12_SHADER_RESOURCE_VIEW_DESC srvDesc{};
srvDesc.Shader4ComponentMapping = D3D12_DEFAULT_SHADER_4_COMPONENT_MAPPING;
srvDesc.Format                  = DXGI_FORMAT_R8G8B8A8_UNORM;
srvDesc.ViewDimension           = D3D12_SRV_DIMENSION_TEXTURE2D;
srvDesc.Texture2D.MipLevels     = 1;

device->CreateShaderResourceView(
    textureResource,
    &srvDesc,
    srvHeap->GetCPUDescriptorHandleForHeapStart()
);
```

## **シェーダとルートシグネチャの設定**

テクスチャを使う場合、ルートシグネチャに  
ディスクリプタテーブルとスタティックサンプラーを追加する。  

<span style ="color: red;">**サンプラー**</span> とは、テクスチャのピクセルを  
「どう読み取るか」を決める設定で、  
拡大時にぼかすか・くっきりさせるかなどを指定できる。  

```c++
// サンプラーの設定
D3D12_STATIC_SAMPLER_DESC sampler{};
sampler.Filter           = D3D12_FILTER_MIN_MAG_MIP_LINEAR; // 線形補間
sampler.AddressU         = D3D12_TEXTURE_ADDRESS_MODE_WRAP;
sampler.AddressV         = D3D12_TEXTURE_ADDRESS_MODE_WRAP;
sampler.AddressW         = D3D12_TEXTURE_ADDRESS_MODE_WRAP;
sampler.ShaderRegister   = 0;
sampler.ShaderVisibility = D3D12_SHADER_VISIBILITY_PIXEL;

// ディスクリプタテーブルの設定（t0 スロットに SRV を 1 つ）
D3D12_DESCRIPTOR_RANGE range{};
range.RangeType          = D3D12_DESCRIPTOR_RANGE_TYPE_SRV;
range.NumDescriptors     = 1;
range.BaseShaderRegister = 0;

D3D12_ROOT_DESCRIPTOR_TABLE table{};
table.NumDescriptorRanges = 1;
table.pDescriptorRanges   = &range;

D3D12_ROOT_PARAMETER rootParam{};
rootParam.ParameterType    = D3D12_ROOT_PARAMETER_TYPE_DESCRIPTOR_TABLE;
rootParam.DescriptorTable  = table;
rootParam.ShaderVisibility = D3D12_SHADER_VISIBILITY_PIXEL;

D3D12_ROOT_SIGNATURE_DESC rootSigDesc{};
rootSigDesc.NumParameters     = 1;
rootSigDesc.pParameters       = &rootParam;
rootSigDesc.NumStaticSamplers = 1;
rootSigDesc.pStaticSamplers   = &sampler;
rootSigDesc.Flags = D3D12_ROOT_SIGNATURE_FLAG_ALLOW_INPUT_ASSEMBLER_INPUT_LAYOUT;
```

シェーダ側ではテクスチャとサンプラーを受け取り、  
UV 座標を使ってピクセルの色をサンプリングする。  

```hlsl
Texture2D    g_texture : register(t0);
SamplerState g_sampler : register(s0);

struct PSInput
{
    float4 position : SV_POSITION;
    float2 uv       : TEXCOORD;
};

float4 ps(PSInput input) : SV_TARGET
{
    return g_texture.Sample(g_sampler, input.uv);
}
```

描画コマンド記録時には、SRV ヒープをセットしてから  
ディスクリプタテーブルをバインドする。  

```c++
ID3D12DescriptorHeap* heaps[] = { srvHeap };
commandList->SetDescriptorHeaps(1, heaps);

commandList->SetGraphicsRootDescriptorTable(
    0, // ルートパラメータのインデックス
    srvHeap->GetGPUDescriptorHandleForHeapStart()
);
```

## **まとめ**

テクスチャを使えるようにするまでの流れを整理する。  

| 手順 | 内容 | ヒープタイプ |
|------|------|------------|
| GPUリソース作成 | VRAM に領域を確保 | DEFAULT |
| データ読み込み | CPU に画像をロード | − |
| アップロードバッファ | CPU→GPU の中継バッファを作成 | UPLOAD |
| VRAM へコピー | コマンドリストで転送・バリア設定 | − |
| SRV 作成 | シェーダへの参照を登録 | − |

<span style ="color: red;">**最も重要なポイント**</span> は  
「CPU から VRAM に直接書き込めない」という制約。  
アップロードバッファを中継として使う二段階の転送が  
DirectX12 テクスチャ転送の基本となる。
