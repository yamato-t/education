# **DirectX12 3D描画基礎 Part2**

前回は、カメラとオブジェクトのコンスタントバッファを個別に分けて、  
描画処理（GPU）で利用する所まで進んだ。  
同じ方法で「オブジェクトを複数描画する」という事が可能になる。

ただ複数のオブジェクトを 3D 描画する場合、  

- オブジェクト同士の描画優先（どのオブジェクトを最前面で描くか）
- 重なった時にどの様に描画するか（べた塗 or 色を重ねる？）

など、考慮しなければならない事が増える。  
今回はそれらの設定に関して説明していく。

---

## **デプスバッファ（深度バッファ）**

デプスバッファは3D描画において最も重要な要素の一つ。  
オブジェクトの前後関係を正確に判定し、  
手前にあるオブジェクトが奥のオブジェクトを隠すという、  
自然な描画結果を実現する。  

### **デプスバッファの仕組み**

デプスバッファはピクセルごとにカメラからの距離（深度値）を記録する。  
新しいピクセルを描画する際、既存の深度値と比較し、  
より手前にある場合のみピクセルを更新する。  
この処理により、描画順序に依存せず正確な前後関係を表現できる。  

これを**デプス（深度 あるいは Z）テスト**と呼ぶ。

深度値は通常0.0（ニアクリップ面）から1.0（ファークリップ面）の範囲で表現され、  
カメラに近いほど小さな値となる。  

### **デプスバッファリソースの作成**

デプスバッファは専用のテクスチャリソースとして作成する。  
RenderTarget、つまり画用紙と同じ部類。  

フォーマットは DXGI_FORMAT_D32_FLOATやDXGI_FORMAT_D24_UNORM_S8_UINT を使用する。  

```c++
// デプスバッファ用のテクスチャリソースの作成
D3D12_HEAP_PROPERTIES heapProps = {};
heapProps.Type = D3D12_HEAP_TYPE_DEFAULT;
D3D12_RESOURCE_DESC depthDesc = {};
depthDesc.Dimension = D3D12_RESOURCE_DIMENSION_TEXTURE2D;
depthDesc.Width = width;
depthDesc.Height = height;
depthDesc.DepthOrArraySize = 1;
depthDesc.MipLevels = 1;
depthDesc.Format = DXGI_FORMAT_D32_FLOAT;
depthDesc.SampleDesc.Count = 1;
depthDesc.SampleDesc.Quality = 0;
depthDesc.Layout = D3D12_TEXTURE_LAYOUT_UNKNOWN;
depthDesc.Flags = D3D12_RESOURCE_FLAG_ALLOW_DEPTH_STENCIL;

// デプスバッファの初期値
D3D12_CLEAR_VALUE clearValue = {};
clearValue.Format = depthDesc.Format;
clearValue.DepthStencil.Depth = 1.0f;
clearValue.DepthStencil.Stencil = 0;


ID3D12Resource* depthBuffer;
device->CreateCommittedResource(
    &heapProps,
    D3D12_HEAP_FLAG_NONE,
    &depthDesc,
    D3D12_RESOURCE_STATE_DEPTH_WRITE,
    &clearValue,
    IID_PPV_ARGS(&depthBuffer)
);
```

### **デプスビューの作成**

作成したデプスバッファをパイプラインで使用するには、  
DSV（Depth Stencil View）を作成する必要がある。  

DSV はディスクリプタヒープも専用の物を利用する。

```c++
// DSV用ディスクリプタヒープの作成
D3D12_DESCRIPTOR_HEAP_DESC dsvHeapDesc = {};
dsvHeapDesc.Type = D3D12_DESCRIPTOR_HEAP_TYPE_DSV;
dsvHeapDesc.NumDescriptors = 1;
dsvHeapDesc.Flags = D3D12_DESCRIPTOR_HEAP_FLAG_NONE;

ID3D12DescriptorHeap* dsvHeap;
device->CreateDescriptorHeap(&dsvHeapDesc, IID_PPV_ARGS(&dsvHeap));

// DSVの作成
D3D12_DEPTH_STENCIL_VIEW_DESC dsvDesc = {};
dsvDesc.Format = DXGI_FORMAT_D32_FLOAT;
dsvDesc.ViewDimension = D3D12_DSV_DIMENSION_TEXTURE2D;
dsvDesc.Flags = D3D12_DSV_FLAG_NONE;

D3D12_CPU_DESCRIPTOR_HANDLE dsvHandle = 
    dsvHeap->GetCPUDescriptorHandleForHeapStart();
device->CreateDepthStencilView(
    depthBuffer, 
    &dsvDesc, 
    dsvHandle
);
```

### **デプスステートの設定**

パイプラインステートオブジェクト（PSO）で深度テストの動作内容を設定する。  

```c++
// デプスステートの設定
D3D12_DEPTH_STENCIL_DESC depthStateDesc = {};
depthStateDesc.DepthEnable = true;
depthStateDesc.StencilEnable = false;
depthStateDesc.DepthWriteMask = D3D12_DEPTH_WRITE_MASK_ALL;
depthStateDesc.DepthFunc = D3D12_COMPARISON_FUNC_LESS;

// PSOに設定
psoDesc.DepthStencilState = depthStateDesc;
psoDesc.DSVFormat = DXGI_FORMAT_D32_FLOAT;
```

### **深度比較関数の種類**

DepthFuncで指定できる比較関数は以下の通り。  

| 比較関数 | 説明 | 用途 |
|---------|------|------|
| NEVER | 常に描画しない | デバッグ用 |
| LESS | 既存値より小さい場合描画 | 通常の3D描画 |
| EQUAL | 既存値と等しい場合描画 | 特殊な用途 |
| LESS_EQUAL | 既存値以下の場合描画 | デカール等 |
| GREATER | 既存値より大きい場合描画 | 逆深度バッファ |
| NOT_EQUAL | 既存値と異なる場合描画 | 特殊な用途 |
| GREATER_EQUAL | 既存値以上の場合描画 | 逆深度バッファ |
| ALWAYS | 常に描画する | デプステスト無効化 |

### **レンダーターゲットとデプスバッファの設定**

描画コマンド実行前に、レンダーターゲットとデプスバッファを設定する。  

```c++
// レンダーターゲットとデプスの設定
D3D12_CPU_DESCRIPTOR_HANDLE rtvHandle = 
    rtvHeap->GetCPUDescriptorHandleForHeapStart();
D3D12_CPU_DESCRIPTOR_HANDLE dsvHandle = 
    dsvHeap->GetCPUDescriptorHandleForHeapStart();

commandList->OMSetRenderTargets(1, &rtvHandle, FALSE, &dsvHandle);
```

### **デプスバッファのクリア**

フレームの開始時にデプスバッファを初期化する必要がある。  

```c++
// デプスバッファのクリア
commandList->ClearDepthStencilView(
    dsvHandle,
    D3D12_CLEAR_FLAG_DEPTH,
    1.0f,  // クリア値（最大深度）
    0,     // ステンシルクリア値
    0,
    nullptr
);
```

### **デプスフォーマットの選択**

デプスバッファのフォーマットは精度と用途に応じて選択する。  

| フォーマット | 深度ビット数 | ステンシルビット数（※） | 用途 |
|-------------|-------------|-------------------|------|
| DXGI_FORMAT_D32_FLOAT | 32 | 0 | 高精度深度 |
| DXGI_FORMAT_D24_UNORM_S8_UINT | 24 | 8 | 深度+ステンシル |
| DXGI_FORMAT_D16_UNORM | 16 | 0 | 低メモリ消費 |
| DXGI_FORMAT_D32_FLOAT_S8X24_UINT | 32 | 8 | 最高精度 |



### **デプスバッファを使用しない場合**

デプスバッファを使用しない場合、描画の順番によって前後関係が決定される。  

```c++
// デプステスト無効化の例
depthStateDesc.DepthEnable = false;
```

この設定では、後から描画されたオブジェクトが常に手前に表示されるため、  
3D空間の自然な表現ができなくなる。  
複数のオブジェクトが重なる場合、正確な前後関係が表現できない。  

### **ステンシル**

デプスバッファには「ステンシル」と呼ばれる情報も含める事が出来る。  
ステンシルとは、各ピクセルごとに「印」を記録できる仕組みを指す。

印を記録する事で、  

- 一度目の描画で「印」をピクセルに記録する
- 二度目の描画で「特定の印」が記録されたピクセルだけ重ね描きする

といった複雑な描画制御が可能になる。  
たとえば、

- この範囲だけに影を落とす
- 穴あきの範囲にだけ色を塗る

といった表現に利用できる。

## **レンダーステートの設定**

レンダーステートは描画時のブレンド方法や深度テスト、  
カリング設定などを制御する重要な要素である。  
DirectX12ではPipeline State Object（PSO）作成時に、  
これらの設定を行う必要がある。  

### **ブレンドステートの基礎**

ブレンドステートは描画する色と既に描画されている色を、  
どのように合成するかを定義する。  
半透明表現やエフェクト描画に不可欠な機能。  

ブレンディングの基本式は以下の通り。  

$FinalColor = SourceColor \times SourceBlend \pm DestColor \times DestBlend$

ここで、SourceColorは今から描画するピクセルの色、  
DestColorは既にレンダーターゲットに描かれている色を示す。  

### **アルファブレンド（半透明合成）**

最も一般的なブレンドモードで、アルファ値に基づいた半透明表現を実現する。  
ガラスや水、パーティクルエフェクトなどの表現に使用される。  

```c++
// アルファブレンド設定
D3D12_RENDER_TARGET_BLEND_DESC alphaBlendDesc = {};
alphaBlendDesc.BlendEnable = TRUE;
alphaBlendDesc.SrcBlend = D3D12_BLEND_SRC_ALPHA;
alphaBlendDesc.DestBlend = D3D12_BLEND_INV_SRC_ALPHA;
alphaBlendDesc.BlendOp = D3D12_BLEND_OP_ADD;
alphaBlendDesc.SrcBlendAlpha = D3D12_BLEND_ONE;
alphaBlendDesc.DestBlendAlpha = D3D12_BLEND_ZERO;
alphaBlendDesc.BlendOpAlpha = D3D12_BLEND_OP_ADD;
alphaBlendDesc.RenderTargetWriteMask = D3D12_COLOR_WRITE_ENABLE_ALL;

D3D12_BLEND_DESC blendDesc = {};
blendDesc.RenderTarget[0] = alphaBlendDesc;

// PSOに設定
psoDesc.BlendState = blendDesc;
```

この設定により以下の計算が行われる。  

$FinalColor.rgb = SourceColor.rgb \times SourceAlpha + DestColor.rgb \times (1 - SourceAlpha)$

### **加算合成**

加算合成はソースカラーとデスティネーションカラーを加算する。  
光や炎、爆発などの発光表現に適しており、  
重ねるほど明るくなる特性を持つ。  

```c++
// 加算合成設定
D3D12_RENDER_TARGET_BLEND_DESC additiveBlendDesc = {};
additiveBlendDesc.BlendEnable = TRUE;
additiveBlendDesc.SrcBlend = D3D12_BLEND_SRC_ALPHA;
additiveBlendDesc.DestBlend = D3D12_BLEND_ONE;
additiveBlendDesc.BlendOp = D3D12_BLEND_OP_ADD;
additiveBlendDesc.SrcBlendAlpha = D3D12_BLEND_ONE;
additiveBlendDesc.DestBlendAlpha = D3D12_BLEND_ZERO;
additiveBlendDesc.BlendOpAlpha = D3D12_BLEND_OP_ADD;
additiveBlendDesc.RenderTargetWriteMask = D3D12_COLOR_WRITE_ENABLE_ALL;
```

計算式は以下の通り。  

$FinalColor.rgb = SourceColor.rgb \times SourceAlpha + DestColor.rgb$

### **乗算合成**

乗算合成はソースとデスティネーションを乗算する。  
影や暗闇の表現、色の調整に使用される。  
重ねるほど暗くなる特性を持つ。  

```c++
// 乗算合成設定
D3D12_RENDER_TARGET_BLEND_DESC multiplyBlendDesc = {};
multiplyBlendDesc.BlendEnable = TRUE;
multiplyBlendDesc.SrcBlend = D3D12_BLEND_DEST_COLOR;
multiplyBlendDesc.DestBlend = D3D12_BLEND_ZERO;
multiplyBlendDesc.BlendOp = D3D12_BLEND_OP_ADD;
multiplyBlendDesc.SrcBlendAlpha = D3D12_BLEND_ONE;
multiplyBlendDesc.DestBlendAlpha = D3D12_BLEND_ZERO;
multiplyBlendDesc.BlendOpAlpha = D3D12_BLEND_OP_ADD;
multiplyBlendDesc.RenderTargetWriteMask = D3D12_COLOR_WRITE_ENABLE_ALL;
```
計算式は以下の通り。  

$FinalColor.rgb = SourceColor.rgb \times DestColor.rgb + DestColor.rgb \times zero$


### **減算合成**

減算合成はデスティネーションからソースを減算する。  
特殊なエフェクトや色の反転表現に使用される。  

```c++
// 減算合成設定
D3D12_RENDER_TARGET_BLEND_DESC subtractBlendDesc = {};
subtractBlendDesc.BlendEnable = TRUE;
subtractBlendDesc.SrcBlend = D3D12_BLEND_SRC_ALPHA;
subtractBlendDesc.DestBlend = D3D12_BLEND_ONE;
subtractBlendDesc.BlendOp = D3D12_BLEND_OP_REV_SUBTRACT;
subtractBlendDesc.SrcBlendAlpha = D3D12_BLEND_ONE;
subtractBlendDesc.DestBlendAlpha = D3D12_BLEND_ZERO;
subtractBlendDesc.BlendOpAlpha = D3D12_BLEND_OP_ADD;
subtractBlendDesc.RenderTargetWriteMask = D3D12_COLOR_WRITE_ENABLE_ALL;
```
計算式は以下の通り。  

$FinalColor.rgb = DestColor.rgb \times one - SourceColor.rgb \times SourceAlpha$


### **ブレンドモード比較表**

| ブレンドモード | SrcBlend | DestBlend | BlendOp | 用途 |
|----------------|----------|-----------|---------|------|
| アルファブレンド | SRC_ALPHA | INV_SRC_ALPHA | ADD | 半透明、UI |
| 加算 | SRC_ALPHA | ONE | ADD | 光、炎、爆発 |
| 乗算 | DEST_COLOR | ZERO | ADD | 影、暗闇 |
| 減算 | SRC_ALPHA | ONE | REV_SUBTRACT | 特殊エフェクト |

### **デプスステートとブレンドの組み合わせ**

半透明オブジェクトを描画する際は、  
深度テストとブレンドステートの組み合わせに注意が必要。  

```c++
// 半透明オブジェクト用のデプス設定
D3D12_DEPTH_STENCIL_DESC depthStateDesc = {};
depthStateDesc.DepthEnable = TRUE;
depthStateDesc.DepthWriteMask = D3D12_DEPTH_WRITE_MASK_ZERO;  // 書き込み無効
depthStateDesc.DepthFunc = D3D12_COMPARISON_FUNC_LESS;
depthStateDesc.StencilEnable = FALSE;

// PSOに設定
psoDesc.DepthStencilState = depthStateDesc;
```

<span style="color: red;">**半透明オブジェクトを描画する際の注意点**</span>

- 深度書き込みを無効化する（DepthWriteMask = ZERO）
- 深度テストは有効にする（DepthEnable = TRUE）
- 不透明オブジェクトを先に描画する
- 半透明オブジェクトは奥から手前の順で描画する
- ソート処理による描画順制御が必要となる

半透明と深度の関係はかなり相性が悪い。

### **ラスタライザステートの設定**

カリング設定やワイヤーフレーム表示などを制御する。  

```c++
// ラスタライザ設定
D3D12_RASTERIZER_DESC rasterizerDesc = {};
rasterizerDesc.FillMode = D3D12_FILL_MODE_SOLID;
rasterizerDesc.CullMode = D3D12_CULL_MODE_BACK;
rasterizerDesc.FrontCounterClockwise = FALSE;
rasterizerDesc.DepthClipEnable = TRUE;
rasterizerDesc.MultisampleEnable = FALSE;
rasterizerDesc.AntialiasedLineEnable = FALSE;

// PSOに設定
psoDesc.RasterizerState = rasterizerDesc;
```

カリングモードの種類は以下の通り。  

- `CULL_MODE_NONE` - カリングなし（両面描画）
- `CULL_MODE_FRONT` - 前面カリング
- `CULL_MODE_BACK` - 背面カリング（通常はこれを使用）

## **複数のレンダーステートの利用**

実際のゲームアプリケーションでは、  
内容が異なる複数のレンダーステートやデプスステートを利用して描画を行っていく。

ただ、各種ステートは PSO 毎に設定される為、複数のステートを利用したい場合は、  
自然と「複数の PSO を作成して使い分ける」という形になっていく。

## **まとめ**

ここまで DirectX12 における3D描画の基礎として、  
以下の重要な要素を解説した。  

<span style="color: red;">**主要項目**</span>

- 座標変換の仕組みと行列の役割
- コンスタントバッファによるシェーダへのデータ転送
- シーン共通データとオブジェクト固有データの区別
- デプスバッファによる前後関係の正確な表現
- レンダーステートによる様々なブレンドモード

これらの知識を組み合わせることで、  
より複雑な3Dシーンの描画が可能となる。  
次のステップとしては、テクスチャマッピング、ライティング、  
シャドウマッピングなどの技術を学習することで、  
さらにリアルな3D表現を実現できる。  

3D描画の理解において最も重要なのは、  
各要素がどのように連携して最終的な画像を生成するかを、  
パイプライン全体として把握すること。  
座標変換、コンスタントバッファ、デプステスト、ブレンディングは、  
それぞれが独立した機能ではなく、相互に関連しながら、  
魅力的な3D映像を作り出す基盤となる。
