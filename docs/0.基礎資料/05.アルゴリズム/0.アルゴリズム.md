# **アルゴリズム**

ゲームに限らず、全てのアプリケーションにおいて、  
コーディングを進める上で「誰もが割と頻繁に実装しなければならない処理」がある。  
例えば**物の並び替えや物の検索**。  

アクションゲームのロックオン機能で考える。  
画面の中に複数の敵がいる場合、最初に遠い敵をロックオンされると、  
ユーザー体験としては宜しくない。

そういった場合**ロックオンする順番を状況に応じて変えなければならない**。  
ここで<span style ="color: red;">**並び替えの処理**</span>が必要になる。

もう一つ、一定範囲にいる全ての敵にダメージを与える攻撃を考える。  

攻撃の発生位置から**一定範囲にいる敵を知る必要がある**。  
ここで<span style ="color: red;">**検索の処理**</span>が必要になってくる。

この様に、頻繁に利用される並び替えや検索の処理だが、  
 <span style ="color: red;">**特定の手順を行うと自然にそうなる方法**</span> が色々考案されている。

これらを総称して<span style ="color: red;">**アルゴリズム**</span>と呼ぶ。


---
## **学ぶ目的**

- **プログラムにおける「問題解決への考え方」の代表例である為**  
冒頭でも述べた通り、プログラムには「誰もが経験するような問題」がある。  
「あるデータを並べ替えたい」や「あるデータを検索したい」等が挙げられれる。  
これまでに、この「並べ替え」や「検索」の為の色々な方法が考案されており、    
プログラムでアルゴリズムを調べるとまずこれらが出てくる。  

- **「効率が良い」という事を意識できるようになる為**   
アルゴリズムを利用する際、必ず「効率」を考える必要が出てくる。  
 <span style ="color: red;">「効率が良い」を意識するというのは、日常生活全てにおいて大事な物</span>。  
当然プログラミングにも必要な意識となる。  
色々なやり方があり、望んだ結果が得られるならそれは全て正解ではあるが、  
エンジニアとして評価される為には、その結果を出すまでに、  
<span style ="color: red;">効率がいい形で実現できているか</span> が重要になってくる。  
アルゴリズムは「評価されるプログラム」を作る為に必要。
　　　
- **「何かをプログラムで作成する」というのは「その『何か』を実現する為のアルゴリズムを考える」と同義である為**   
代表的な問題に対する定番のアルゴリズムはあるが、  
自分が作りたいと思ってる物、解決したいと思っている問題全てに、  
定番のアルゴリズムがある訳ではない。  
自分が作りたい物に対するアルゴリズムは、  
前例なども調べつつ、それらを組み合わせるなどして、  
自分で試行錯誤していかないといけない事を意識しなければならない。  
つまりアルゴリズムは<span style ="color: red;">**自分で考案しなければならない物**</span> というでもある。

## **アルゴリズムとは**

### **目的地への行き方**

例えば「目的地に出来るだけ早く到着したい」と考えた場合、  
その行き方には色々な方法がある。  
まず何を利用するか？という点で電車や自動車や自転車、バイクやバス等が挙げられる。  
仮に電車を選んだ場合、次はどの線を利用するか？を考える事になる。  
JR線、御堂筋線、阪急線等色々挙げられる。  
さらに、どのルートが一番早く着くかを考え、  
最終的に自分が最も好ましいと思う行き方を選ぶはず。  
これは「目的地に早く到着する」というアルゴリズムを考えてるのと同じ。  

### **野菜の切り方**

野菜の切り方は、アルゴリズムの説明で出てくる機会が多い。  
例えば、ニンジンを輪切りにして星形に切った飾りを沢山作りたい場合、  

- 先に輪切りにして、輪切りにしたもの一つ一つを星形にくり抜いていく  
- 輪切りにする前に切れ込みを入れて、そのあと輪切りする  

などの方法が挙げられる。  
どちらの方が早く完成するか。

## <span style ="color: yellow;">効率に関する補足</span>

プログラムは<span style ="color: red;">**人がコンピュータにやらせたい事**</span>をコーディングの形で指示する。

「0 から 10 までの総和を計算したい」となった場合、  
当然「0 から 10 までの総和を計算しろ」という指示をコーディングして実行させる。

皆さんならどの様にコーディングするか？  
for ループで計算する人もいれば、while ループで計算する人もいるはず。  
あるいは、総和の式をコーディングに取り入れる事も可能。

```c++
// for ループの例
int sum = 0;
for (int i = 0; i <= 10; ++i)
{
	sum += i;
}
printf("%d\n", sum);

// while ループの例
sum = 0;
int i = 10;
while (i != 0)
{
	sum += i;
	--i;
}
printf("%d\n", sum);

// 総和の公式の例
sum = (10 / 2) * (10 + 1);
printf("%d\n", sum);

```

つまり「0 から 10 までの総和を計算する方法」は、  
<span style ="color: red;">**一つしかないとは限らない**</span> という事。

ここでそれぞれの  <span style ="color: red;">**処理の回数**</span> にも注目する。

ループしているものは、  
 ```i``` の値を見比べる処理が 10 回、```i``` を ```sum``` に足し合わせる処理を 10 回行っている。  
対して、総和の公式を利用したものは、  
割り算処理 1 回、かけ算処理 1 回、足し算処理 1 回行っている。

では 0 から 1000 の総和を計算する事になった場合、  
それぞれの処理の回数はどうなるか？  
実際にやってみると計算回数に圧倒的な差が出てくる。

処理の回数が少ないという事は、  
それだけコンピュータにとって「効率が良い」という事に繋がる。
