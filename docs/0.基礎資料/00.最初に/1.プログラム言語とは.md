# **プログラム言語**  
---

## **プログラム言語とは**  

プログラム言語とは <span style ="color: red;">**コンピュータに何かをさせる為の指示書を作る際に利用する言語**</span>を指す。  
あくまで<span style ="color: red;">**言語**</span>である事。  
ここでいう「言語」とは、日本語や英語といった言語と意味は大して変わらない。

コンピュータは思ったより賢くない。  
何かをさせたい場合は、させたい事を  
**言語のルールを守り詳細に順を追って正確に指示していかなければならない**。  

プログラム言語には多くの種類がある。  
ゲーム開発でエンジニアとして仕事をしていく為には、  
まず <span style="color: red;">**C/C++言語**</span> を押さえておかなければならない。

C/C++ は様々な言語のベースとなっており、  
C/C++ を把握しておけば、他言語を学ぶ際にもある程度スムーズに入る事が出来る。

## **コンパイルとビルド**  

プログラム言語による指示書は「人間にとってわかりやすい」物であり、  
「コンピュータにとっては分かりにくい」物になっている。   
その為、コンピュータが分かりやすい形にする為の「翻訳」が必要になる。  
これを<span style ="color: red;">**コンパイル**</span>という。  
コンパイルはファイル単位で行われる。

プログラム言語を翻訳する際に、文法や記述ルールを間違えていると、  
コンピュータは正しく翻訳できなくなる。  
これらは**コンパイルエラー**と呼ばれる。

コンパイルした結果を利用して、  
アプリケーションとして実行できるまで組み立てる事を<span style ="color: red;">**ビルド**</span>と呼ぶ。  
ビルドが完了する事で、アプリケーションの実行ファイルが出来上がる。

コンパイル（翻訳）は成功したが、結果が上手くかみ合わず、  
実行ファイルが作れない事を**ビルドエラー**と呼ぶ。

## **コーディングで詰まった時**  

コーディングで手が止まってしまう時は、  
<span style="color: red;">**何が分かっておらず手が動かないのか**</span>を考える事が大切。

### **コンピュータの事が分かっているか**  
n進数、CPU、GPU、メモリが分かっているかどうか。  
今はあまり意識する事がないかもしれないが、  
仕事となると新人関係なく、この手の知識はある程度必要になる。

### **プログラム言語の文法が分かっているか**  
プログラム言語の書き方、機能、ルールなどをわかっているかどうか。  
前期の講義ではまずここに注力していく。

### **やりたい事を細かく分割して整理出来ているか**  
「今自分は何をコーディングしたいのか」を改めて考えてみる。  
<span style="color: red;">**やりたい事や作りたい物を、細かく分けた形まで整理する事がまず最初に必要**</span>。  
落ち着いて分解し、細かい単位（要件）まで整理出来れば、  
自然と「今コーディングする物」が見えてきやすい。  

### **方法が思いつけているか**  
「望んだ結果を得る方法」を思いつけているのかどうか。  
課題や要望、依頼を実現する為の方法をちゃんと考えられているのか。  

上手くいかない事、思った通りにいかない事が出てきた時、  
これらの事を意識し、何が引っ掛かっているのかを把握する事が最初にやるべき事。
