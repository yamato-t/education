# **参照**
---

C++では、オブジェクトへのアクセス方法に  
「参照」という方法が追加された。

ポインタとは別物ではあるが、ポインタと似たメリットがある上、  
ポインタを直接扱うよりも安全である為ここで挙げておく。

---

## **参照の利用**

```c++
int  value = 1;

// 「value」を参照する変数「valueRef」を宣言
int& valueRef = value;
// 「valueRef」を更新する事で、参照先である「value」の内容が変わる
valueRef = 10;

std::cout << "value = " << value << std::endl; // 10 と表示される
```
上記の様に、型に`＆`をつける事で、`=` の右側の**変数を参照する変数**が宣言できる。  
さらに参照変数を変更すると、参照先の内容を書き換えられる様になる。  
作用としてはポインタ変数に似ている。

ポインタ変数を利用した場合の記述は下記の通り。

```c++

int  value = 1;

// 「value」のアドレスを持つ「valueRef」を宣言
int* valueRef = &value;
// 「valueRef」が指すアドレスの中身、つまり「value」を更新する
*valueRef = 10;

std::cout << "value = " << value << std::endl; // 10 と表示される

```

---
## **参照とポインタの関係**

ポインタ変数について再度整理する。  
ポインタ変数には、メモリ上に存在するオブジェクトのメモリアドレスが入っており、  
<span style="color: red;">**メモリアドレスが指している領域全体（オブジェクトそのもの）を複製して持っている訳では無い**</span> 。

なお、変数でそのまま受け取った時は、  
別に新しい領域が確保され、内容が複製される形になる。

参照もこれらポインタ変数を使用した際の考え方と、  
同じように認識してもらえればいい。  

---

## **参照の用途**

主に関数の引数や戻り値に利用される機会が多い。  
関数の場合、ポインタ変数と同じく引数で渡す事で、  
関数内で変更した際、外側の変数も変更できる様になる。  
関数の戻り値に利用した場合も同様。  
これは**参照渡し**と呼ばれている。


```c++
// 整数オブジェクトを参照渡しで受け取る関数
void function(int& valueRef)
{
	// 内容を更新する
	valueRef = 10;
}

// エントリー関数
int main()
{
	int value = 0;

	// 引数で渡す時はそのまま渡す
	function(value);

	std::cout << "value = " << value << std::endl; // 10 と表示される
}

```


また書き換える必要が無い場合でも「**参照はオブジェクトを複製しない**」という特徴から、  
サイズの大きなオブジェクトを引数として渡す場合や、戻り値として返す場合に、  
参照渡しは利用される。


```c++

// 大量のメモリを使う MyStruct 構造体
struct MyStruct
{
	int values_[10000000] = {};
};


// MyStruct オブジェクトを const 参照渡しで受け取る関数
// 参照で渡している為、MyStruct は複製されずに済む
void function(MyStruct& ref)
{
	// 何らかの処理
}

// エントリー関数
int main()
{
	// MyStruct オブジェクトを作成する
	MyStruct my;

	// そのまま引数に渡す
	function(my);
}

```

これは、無駄なオブジェクトの生成を避ける事で、  
メモリの使用量を減らしたり、処理時間を短縮する事が狙い。

## **ポインタと参照の違い**

ポインタ変数には「無効」という意味の `NULL` がある。  
`NULL` でありながら利用しようとすると、当然ヌルアクセスという不具合が発生する。  
参照はメモリ上に存在するオブジェクトが無いとそもそも宣言出来ず、  
`NULL` にあたる物は存在しない。


```c++

// 整数型の変数(オブジェクト)を宣言
int value = 0;

// オブジェクトを指定して参照を宣言
int& valueRef = value;			// 問題ない

// 参照はメモリ上に存在しているオブジェクトを指定しないと宣言出来ない
// 以降の記述は全てコンパイルエラーになる

int& valueRefError1;			// 初期化していないのでエラー
int& valueRefError2 = 1;		// 1は単なる値でありメモリ上のオブジェクトではない為エラー
int& valueRefError3 = NULL;		// valueRefError2 と同じ理由でエラー

```

関数の引数で参照が利用されている場合、  
利用する側は**「必ず実体がある」という事を保証した使い方**でないといけない。  

```c++

// NULL が入った「valuePtr」を宣言
int* valuePtr = NULL;

// ポインタがヌルでない事を「関数を使う側」が保証しなければならない
// なお、この場合 valuePtr は NULL なので挙動がおかしくなるはず
function(*valuePtr);

```
