# **ポインタの危険性**

---

## **関係ないアドレスの内容を不正に上書きする**

下記のプログラムをどう思うか？

```c++
int array[4];
int* p = array;
for (int i = 0; i < 5; ++i) {
    printf("%p\n", p);
    p = p + 1;
}
```

このプログラムは危険ではあるが問題は起こらない。  
配列の要素数は 4 で、明らかに要素数を超えたアドレスを取得している。  
ただアドレスを表示しているだけである為、問題は起こらない。  

しかし下記のようなプログラムだった場合は大いに問題。

```c++
int array[4];
int* p = array;
for (int i = 0; i < 5; ++i) {
    printf("%p\n", p);
    p = p + 1;
    *p = 0;
}
```

最後の行で `p` が持っているアドレスに保持されている値をゼロに上書きしている。  

今この for ループでは `array` 配列の要素数を超えたアドレスを取得している。  
その要素数を超えたアドレスは <span style="color: red;">**誰が何のために使っているか分からない**</span>。  

たまたま使われていないかもしれないし、別の変数で使われているかもしれない。  
つまり**このプログラムは何が起こるか分からない**。  
だから大いに問題になる。

## **変数の寿命とポインタ変数**

最初に話した「変数には寿命がある」という事を思い出した上で、  
このプログラムを確認してどの様な印象を受けるか？

```c++
int main() 
{
    int* p;
    {
        int value = 0;
        p = &value;
        *p = 10;
    }

    // 何か色々処理する

    *p = 100;

    // 何か色々処理する

    return 0;
}
```

このプログラムは大いに問題。

`value` はすでに寿命で利用できなくなっているが、  
`value` が使えていた頃のアドレスだけは `p` で保持されている状態になっている。

変数の寿命が消えるとコンピュータが現在利用していないアドレスとして認識し、  
そのアドレスを別の事に再利用するようになるとお話しした。

では `p` のアドレスに入っている内容を 100 に書き換えたが、  
これで何が起こると思うか？  

そのアドレスがその時に何に使われているかは分からない。  
つまり何が起こるかわからない。

ポインタを使う時は、 

- <span style="color: red;">そのポインタが指しているアドレスがちゃんと利用できるのか</span>  
- <span style="color: red;">その変数の寿命が尽きていないかどうか</span>

などをしっかりと意識した上で利用しなければならない。

## **無効なポインタ変数を指す NULL キーワード**

変数の寿命が尽きた時、  
その変数のアドレスを持っているポインタ変数も使えなくする必要がある。  

ポインタも使えなくする方法は簡単。  
「何のアドレスも持っていない」という事を意味する値を代入すればいい。

一般的にそういう場合は **NULL というキーワード**を利用する。

```c++
p = NULL;
```

変数の寿命が切れる直前で `NULL` を代入する。

そしてその下でアドレスから値を代入する前に、  
**NULL チェック**を行う。

```c++
// ポインタ変数 p が有効か？
if(p != NULL)
{
    *p = 100;
}
```

今のこのコードではあまり意味がないが、  
例えば下記の場合はNULLチェックが必要になる。

```c++
int main() 
{
    int a   = 1;
    int* p  = &a;

    srand(time(NULL));

    if(rand() % 2 == 0)
    {
        int value = 0;

        p  = &value;
        *p = 10;

        p  = NULL;
    }

    if(p != NULL)
    { 
        *p = 100;
    }

    return 0;
}
```
