# **ファイル分割**
---

## **ファイルを分ける**

今まではひとつのファイルにすべてのコードを記述していたが、  
関数化した事によりファイルを分ける事もできるようになる。

実際のゲームは何万・何十万行という膨大なコード量になる。  
それをすべてひとつのファイルに書く事は現実的ではない。

ファイルを分ける事自体は難しくない。  
まず `function.h` と `function.cpp` というファイルを新規作成する。

```c++
// function.h
#pragma once

int f(int, float);
```

拡張子が `.h` のファイルを <span style ="color: red;">**ヘッダファイル**</span> と呼ぶ。  
ここには関数のプロトタイプ宣言のみを記述する。

一行目の `#pragma once` は非常に重要な意味があるが、  
現時点ではその恩恵を受けられる場面がない。後ほど説明する。

```c++
// function.cpp
#include <stdio.h>

int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);

    return answer;
}
```

`function.cpp` にはプロトタイプ宣言した関数の定義を記述する。

最後に `main` 関数があるファイルを編集する。

```c++
// ConsoleApplication1.cpp
#include "function.h"

int main()
{
    int answer1 = f(1, 1.0f);
    int answer2 = f(2, 2.0f);
    int answer3 = f(3, 3.0f);

    return 0;
}
```

`function.h` で宣言した関数を利用する為、インクルードの指定が変わっている。  
また`<>` ではなく `""` になっている点に注意してほしい。  
自作したヘッダファイルはダブルクォーテーションでインクルードする。

こうする事でファイルを分け、機能をさらに整理していく事ができる。

---

## **ファイルを分けた事による問題**

ヘッダファイルに直接定義を書く事はできないのか、という疑問があるかもしれない。  
結論から言うと書ける。  
試しに `function.cpp` の内容をカットして `function.h` に貼り付けてみてほしい。  

ただしその場合、`main` 関数がある cpp ファイルで  
`function.h` を二回インクルードすると多重定義のコンパイルエラーが発生する。  
インクルードは**ファイルに記述されている内容をその位置に取り込んでいるだけ**である為。  
その為二回インクルードされると、同じ定義を二回行っている事になる。

そこで重要になってくるのが `#pragma once`、  
<span style ="color: red;">**インクルードガード**</span> とも呼ばれる記述。  
これは複数回インクルードされるのを回避する為にある。

ヘッダファイルで関数などを定義し始めると、こういった問題にあたるので、  
常にインクルードガードを記述しておく事が無難。

基本的に関数の定義は `.cpp` に、宣言は `.h` に分けておくと整理しやすくなる。  
また、どの `.h` にどの `.cpp` の関数が宣言されているかを分かりやすくする為、  
関数を定義する `.cpp` には対応する `.h` をインクルードしておくと良い。

## **ファイル分割とグローバル変数**

ファイルを分けた際、各ファイルで利用しているグローバル変数には注意が必要。

```c++
// ConsoleApplication1.cpp
#include "function.h"

// グローバル変数宣言
int g_value;

int main()
{
    int answer1 = f(1, 1.0f);
    int answer2 = f(2, 2.0f);
    int answer3 = f(3, 3.0f);

    return 0;
}
```

グローバル変数は、そのファイル内のどこでも利用する事ができる。  
問題になるのが、全く同じ名前のグローバル変数が別ファイルにも宣言・定義されている場合。

```c++
// function.cpp
#include <stdio.h>
#include "function.h"

// 同じ名前のグローバル変数がここにもある
int g_value;

int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);
    return answer;
}
```

実際に試してみてほしい。  
「複数回定義されているシンボル」といった内容のエラーが出るはず。  
つまり多重定義のリンクエラーが発生している。

コンパイルは各 `.cpp` ファイル毎に行われる。  
それぞれがコンパイルされた後、翻訳が完了した内容を結合して  
実行ファイルを組み立てようとするが、  
その際に同じ名前の変数が二つ存在している為、上手く繋げられないというエラー。

先ほどの二重インクルード時の多重定義はコンパイルエラーだった。  
今回も多重定義だがリンクエラーとして発生している。  
この違いは、同じファイル内で多重定義が起きているかどうか。  
同じファイルで多重定義が起きると、翻訳自体に失敗するのでコンパイルエラーになる。

この場合「同じ変数名ですが、ファイル毎に別々に扱います」という事を明確にする必要がある。  
やり方は簡単で、変数の宣言時に型の前に `static` というキーワードをつける。

```c++
static int g_value;
```

これで各ファイル別に `g_value` という変数を利用する事ができる。  
これらの変数はあくまで別物である為、  
片方に値を代入しても、もう片方に同じ値が入ったりはしない。

なおここでの `static` は**静的変数と違う意味での利用になるので注意**。

## **ファイル間の変数の共有**

こうなると次に「ヘッダに変数をひとつ宣言して各ファイルで共有できないのか」  
という疑問が出るかもしれない。  
残念ながら普通に記述するとこれはできない。  
ヘッダに定義すると、各ファイルにインクルードする事で多重定義のエラーが発生する。  
かといって `static` を付けると、変数は別物になるので共有できなくなる。

各ファイルでひとつの変数を共有する為には、  
「その変数の本体がどこで定義されているのか」を明確にする必要がある。  
まず、本体となる変数をいずれかの `.cpp` に定義する。

```c++
// function.cpp
#include <stdio.h>
#include "function.h"

// 本体として定義する
int g_value;

int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);
    return answer;
}
```

次にヘッダでは「ある場所に定義された本体を別のファイルでも使う」という事を明記する。  
変数の宣言時、型の前に `extern` というキーワードをつける。

```c++
// function.h
#pragma once

// 本体を別のファイルでも利用できるようにする
extern int g_value;

int f(int, float);
```

これを <span style ="color: red;">**extern 宣言**</span> という。

こうする事で、`function.cpp` に置いている本体の変数を  
各ファイルで共有する事ができる。

ただし、個人的には `extern` 宣言はお勧めしない。  
あちこちで使われ始めると意図しない値に更新されてしまう事があり、  
その原因の調査にも時間がかかってしまう為。

ひとつの変数を各ファイルで共有したい場合は、  
取得用・更新用の関数を用意した方が管理しやすくなる。
