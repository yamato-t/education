# **関数**

---

数当てゲームを一致するまで繰り返せるように実装した。  
ここまでの内容でゲームとしては概ね完成しており、  
学んできた知識でもある程度のゲームは実現できる。

ただ、現状のプログラムコードは仕事や就職作品として  
そのまま使えるかというと、残念ながらそうではない。

まず一番の問題は<span style ="color: red;">**コードの整理整頓が足りていない**</span>点。  
つまり**動きはするがコードがあまり綺麗ではない**という状態。

整理整頓と一言で言ってもピンとこないと思う。  
まず「綺麗ではないコード」がどういうものなのかを考えてみる。  
以下に、代表的な例を二つ挙げる。

---

## **同じ処理を何度も記述している**

```c++
int main()
{
    // 入力した数値を５乗した数を足す

    int a = 0;
    scanf_s("%d", &a);
    int x = 1;
    for (int i = 0; i < 5; ++i)
    {
        x *= a;
    }

    int b = 0;
    scanf_s("%d", &b);
    int y = 1;
    for (int i = 0; i < 5; ++i)
    {
        y *= b;
    }

    int c = 0;
    scanf_s("%d", &c);
    int z = 1;
    for (int i = 0; i < 5; ++i)
    {
        z *= c;
    }

    int answer = x + y + z;
    printf("%d\n", answer);

    return 0;
}
```

よく見ると各所で同じ記述をしているのがわかる。  
入力と計算の処理を一か所にまとめ、複数の場所で使いまわせるようにできれば、  
上記のプログラムはかなり簡潔に出来る。

---

## **どこからどこまでが何をしているのかわかりにくい**

```c++
int main() {
    int t = (int)time(nullptr);
    int num = t % 10;
    int count = 0;

    while (true)
    {
        ++count;

        int input;
        scanf_s("%d", &input);
        int score   = 0;
        int d       = num - input;
        if (d == 0)
        {
            score = 100;
        }
        else if (d > -2 && d < 2)
        {
            score = 50;
        }
        switch (score)
        {
        case 100:
            printf("Clear\n");
            break;
        case 50:
            printf("Close\n");
            break;
        default:
            printf("No\n");
            break;
        }

        if (score >= 100)
        {
            break;
        }
    }
    printf("Continue Count [ %d ] \n", count);

    switch (count)
    {
    case 1:
        printf("Wonderful\n");
        break;
    case 2:
        printf("Cool\n");
        break;
    case 3:
        printf("Well Well\n");
        break;
    default:
        printf("Boo\n");
        break;
    }

    return 0;
}
```

このプログラムは数当てゲームを少し改造したものだが、  
パッと見でそれに気付ける方は中々いない。  
なぜ気付けないかというと、どこからどこまでが何の処理をしているのかが  
一見では分かりにくい為。

逆に「ここからここまでがこの処理をしている」という事を明確にするだけで、  
大分印象は変わる。  
今まではコメントをつける事で明確化していたが、  
複雑なゲームになれば、コメントで機能を明確化するのも無理が出てくる。

各処理を具体的に部品にして、それを繋げる形にすると、  
全体の把握も行いやすく、各所の機能も把握しやすい綺麗なコードになる。

---

## **関数**

あまり綺麗ではない部分は積極的に整理していかなければならない。  
コードを綺麗にする為にまず最初に知らなければならないのが、  
<span style ="color: red;">**関数**</span> と呼ばれる仕組み。  
関数を用意する事で、処理を部品化する事ができる。

では早速、簡単な関数を作ってみる。

```c++
void f()
{
    int answer = 1;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);
}
```

関数を作成した。  
`void f()` と記述し、続く波かっこの中に処理を記述する。  
`f` は関数名を指す。  
この様に関数を作成する事を <span style ="color: red;">**関数を定義する**</span> という。

では関数を利用してみる。

```c++
int main()
{
    f();
    f();
    f();

    return 0;
}
```

`main` では `printf` を一切記述していない。  
代わりに作成した `f` という名前の関数を呼び出している。

処理を関数化すると、その関数を呼び出すだけで  
関数の中で行っている処理を実行する事ができる。  
一度記述した処理を使いまわせるという事。

---

## **引数**

上記の例では余りにも単純な関数で、使いまわせる以外の便利な点が見受けられない。  
もう少し便利な関数になるようにしてみる。  
まず関数内で計算している `answer` の値を、状況に応じて変えられる様にする。

```c++
void f(int number)
{
    int answer = number;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);
}
```

`f` の次の `()` の中で変数が宣言されるようになった。  
こうする事で、関数を呼び出す際に値を渡す事ができるようになる。  
この様に、渡された値を受け取る変数を <span style ="color: red;">**引数**</span> という。

```c++
int main()
{
    f(1);
    f(2);
    f(3);

    return 0;
}
```

呼び出しの際に渡された値が、関数の中で利用されている事がわかると思う。  
なお、引数には型がある。  
今は `int` 型の引数なので、整数を渡すようにしている。

引数が複数ある場合は、カンマで区切る事で増やせる。

```c++
void f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);
}

int main()
{
    f(1, 1.0f);
    f(2, 2.0f);
    f(3, 3.0f);

    return 0;
}
```

引数には変数を渡す事もできる。

```c++
int main()
{
    int number = 1;
    f(number, 1.0f);

    return 0;
}
```

変数に入っている値が引数に代入される形。  


## **引数で渡された変数**

引数に渡された変数を関数の中で更新すると、  
関数の外側の変数はどうなるだろうか。

```c++
void f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);

    number = 10;
}
```

`main` 関数にある `number` の値が10になるのかどうか。  
結論から言うと変わらない。**外の変数と関数の引数は別物**。  
引数は引数として新しく変数が宣言され、別メモリ上に確保される形になる。

## **戻り値**

関数の中で行った処理の結果を、呼び出し元に返す事もできる。

```c++
int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);

    return answer;
}
```

`void` だった部分が `int` に変わった。  
そして最後の行に `return answer;` が追加されている。  
こうする事によって、関数内で計算した結果を呼び出し元に返せるようになる。  
この返される値を <span style ="color: red;">**戻り値**</span> という。

関数名の前のキーワードは戻り値の型を示している。  
`void` は何も返さない、つまり戻り値がない場合に指定する型。  
戻り値は型を指定するだけで、変数宣言は必要ない。

```c++
int main()
{
    int answer1 = f(1, 1.0f);
    int answer2 = f(2, 2.0f);
    int answer3 = f(3, 3.0f);

    return 0;
}
```

呼び出し元では、関数名の前に変数を宣言し、  
`=` で戻り値を代入している。

---

## **関数の宣言**

関数は利用する前に「その関数が存在する事」を明記しておく必要がある。  
例えば、定義より前に関数を呼び出そうとするとコンパイルエラーになる。

```c++
int main()
{
    int answer1 = f(1, 1.0f);
    int answer2 = f(2, 2.0f);
    int answer3 = f(3, 3.0f);

    return 0;
}

int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);

    return answer;
}
```

まだ翻訳していないキーワードが出てきた為、  
どう解釈するべきか分からなくてエラーという状態。

そうなると「関数の定義は常に利用される前に記述する必要がある」  
という事になりそうだが、実はそうではない。

先に <span style ="color: red;">**関数を宣言**</span> すれば、関数の定義を後に記述する事ができる。

```c++
// 関数 f の宣言
int f(int, float);

// メイン関数
int main()
{
    int answer1 = f(1, 1.0f);
    int answer2 = f(2, 2.0f);
    int answer3 = f(3, 3.0f);

    return 0;
}

// 関数 f の定義
int f(int number, float value)
{
    int answer = number + (int)value;
    printf("関数 f が呼び出されました。answer は [%d] です\n", answer);

    return answer;
}
```

関数の定義の一行目のみを抜粋し、引数名を省いて最後にセミコロンを付ける。  
これを <span style ="color: red;">**関数プロトタイプ（プロトタイプ宣言）**</span> と呼ぶ。

定義はしていないので中身の処理はないのだが、  
プロトタイプ宣言する事で名前やどんな型の引数があるかを先に翻訳させ、  
定義の前に利用する事ができるようになる。  
なお引数名を省略したが、記述しておくことも可能。

---

## **エラー**

全く同じ名前と引数の関数を複数定義すると、  
<span style ="color: red;">**多重定義**</span> でコンパイルエラーが発生する。  
変数でも同じ名前の変数を用意するとコンパイルエラーになるのと同じ。  
翻訳の際にどちらを使うのかが分からなくて翻訳できなくなる。

関数の場合、複数回宣言する事はエラーにならないが、  
定義が複数あるとエラーになる。

また、プロトタイプ宣言だけがあり定義をしていない場合、  
<span style ="color: red;">**リンクエラー**</span> という新しいエラーが発生する。

- <span style ="color: red;">**コンパイルエラー**</span>
    - そもそも翻訳ができなかった事によって起こるエラー
    - 同一ファイル内での多重定義などが該当する
- <span style ="color: red;">**リンクエラー**</span>
    - 翻訳は完了したが、翻訳した結果を繋げた際に問題があると発生するエラー
    - プロトタイプ宣言はある為、翻訳は問題なく終わっているが、  
      実行ファイルを組み立てる際に関数の中身がないという問題が起こる

## **静的変数**

関数内で宣言した変数は**ローカル変数**になる。  
関数の外では使えず、関数が呼び出される度に新しく作られる。

例えば下記の例では関数 f を何度呼び出しても同じ結果になる。

```c
#include <stdio.h>

// 関数 f を定義
void f()
{
	// number は f のローカル変数
	int number = 0;
	number++;

	printf("number は [%d] です\n", number);
}

// main 関数
int main()
{
	// 何度呼び出しても結果が同じ
	f(); // number は [1] です
	f(); // number は [1] です
	f(); // number は [1] です

	return 0;
}

```

ここで**関数 f を呼ぶたびに 1 増やしていきたい**となった場合を考える。    
ぱっと思いつくのはグローバル変数を利用する方法。

```c
#include <stdio.h>

// number をグローバル変数にしてソースファイル全体で使えるようにする
int number = 0;

// 関数 f を定義
void f()
{
	// number はグローバル変数
	number++;

	printf("number は [%d] です\n", number);
}

// main 関数
int main()
{
	// 呼び出す度に結果が変わる
	f(); // number は [1] です
	f(); // number は [2] です
	f(); // number は [3] です

	return 0;
}
```

もう一つ、**静的変数**を使う方法もある。  
静的変数とは、  
**一度宣言されたらずっと生き残り続ける変数**を指す。

通常のローカル変数は、波かっこを出ると使えなくなり、  
保存していた値も消えてしまう。

ところが静的変数は、波かっこの中でしか使えない事は同じだが、  
**波かっこを出ても保存している値が消えない**。  

静的変数にする為には、変数宣言時の型の前に `static` というキーワードを付ける。

```c
#include <stdio.h>

// 関数 f を定義
void f()
{
	// 静的変数として number を宣言と同時に初期化する
	static int number = 0;

	// number は f のローカル変数だが、static キーワードが付いているため、関数 f が呼び出される度に値が保持される
	number++;

	printf("number は [%d] です\n", number);
}

// main 関数
int main()
{
	// 呼び出す度に結果が変わる
	f(); // number は [1] です
	f(); // number は [2] です
	f(); // number は [3] です

	return 0;
}

```

関数を呼び出す度に
```c
// 静的変数として number を宣言と同時に初期化する
static int number = 0;
```
が処理されている様に見えるが、  
静的変数の初期化は、最初に処理された一度だけしか行われない。  
その為、更新された値を持ち続ける事が出来る。

但し静的変数が使えるのは、**静的変数を宣言した波かっこの中のみ**。  
「変数が使える」と「変数の値が残り続ける」は違うので留意する事。

## **問題**

関数の基本的な説明はここまで。  
試しに、一つ目のプログラムを関数で整理してみる。  

同じ処理を何度も記述している部分が問題だった。  
処理を使いまわす事を意識して関数を作ってみてほしい。  
関数名、戻り値の型、引数の構成、どこからどこまでを関数化するか、  
これらは人によって変わってくるので、各々で考えてみてほしい。

```c++
int main()
{
    // 入力した数値を５乗した数を足す

    int a = 0;
    scanf_s("%d", &a);
    int x = 1;
    for (int i = 0; i < 5; ++i)
    {
        x *= a;
    }

    int b = 0;
    scanf_s("%d", &b);
    int y = 1;
    for (int i = 0; i < 5; ++i)
    {
        y *= b;
    }

    int c = 0;
    scanf_s("%d", &c);
    int z = 1;
    for (int i = 0; i < 5; ++i)
    {
        z *= c;
    }

    int answer = x + y + z;
    printf("%d\n", answer);

    return 0;
}
```

できた方は二つ目のプログラムも関数で整理してみてほしい。  
二つ目のプログラムは処理の使いまわしではなく、  
どこからどこまでがどういう機能を担っているのかを考え、  
その機能を明確にする事を意識して関数化してみてほしい。
