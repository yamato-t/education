# **文字と配列**

---

## **文字列**

配列を文字で利用すると、文章を表現する事ができる。  
文字は `char` 型を利用していたが、  
この `char` 型を配列にすると、<span style ="color: red;">**文字列**</span> という物が扱えるようになる。

```c++
// 文字の表示
char s = 'a';
printf("%c\n", s);

// 文字列の表示
char str[10] = "abc";
printf("%s\n", str);
```

`str` は `char` の配列になっている。  
宣言と同時に代入しているが、今までの様にカンマ区切りではなく、  
ダブルクォーテーションで複数の文字を囲んでいる。  
通常の配列とは少々違い、文字列の初期化はこの様に記述する。

`char` 型配列の `str` は要素数が 10 。  
今は 3 文字しか入っていないが、最大 10 文字保存できる。

### **文字列の特殊ルール**

では少し書き方を変えてみる。  
まず通常の配列の様に値を代入してみる。

```c++
// 文字列の表示
//char str[10] = "abc";

char str[10];
str[0] = 'a';
str[1] = 'b';
str[2] = 'c';

printf("%s\n", str);
```

文字の配列なので、当然この様な書き方もできる。  
動かす前に結果を考えてみてほしい。

結果はおかしな表示になるはず。

もう一つ試す。  
次はこの様にコーディングしてみてほしい。

```c++
// 文字列の表示
char str[10] = "abcdefghij";

printf("%s\n", str);
```

一見するとおかしくない。  
代入したい文字は 10 文字で、要素数も 10 。  
おかしな事は何も無いように思える。

ところがコンパイルエラーになる。  
そのエラーの内容をよく見てみてほしい。

「10 文字しか入らないので、11 文字では初期化出来ません」と言われている。  

### **ヌル文字**

実は文字列をダブルクォーテーションで初期化する際、  
末尾に勝手に一文字追加される。  
この文字の事を<span style ="color: red;">**ヌル文字**</span>と呼ぶ。

なぜそんな事をするのかというと、  
<span style ="color: red;">**文字列の終端がどこになるのかを明確にする為に必要**</span>だから。

先ほどおかしな表示になったプログラムは、  
ダブルクォーテーションで初期化していない。  
各添え字を指定して文字を入れただけなので、  
どこにもヌル文字が入っていない事になる。

正しく表示させたい場合はどうすれば良いかというと、  
単純に文字の最後にヌル文字を入れるだけ。

```c++
// 文字列の表示
//char str[10] = "abc";

char str[10];
str[0] = 'a';
str[1] = 'b';
str[2] = 'c';
str[3] = '\0'; // ヌル文字

printf("%s\n", str);
```

これで正しい表示になるはず。

文字列を正しく扱うには、末尾に必ずヌル文字が必要という事を  
忘れないようにしてほしい。

文字列は文字を配列化しただけに見えるが、  
ヌル文字の存在や初期化方法の違いなどから、  
文字型とは全く違う型として認識してもらえればと思う。

## **文字を表す数値**

コンピュータは全ての情報を 0 と 1 の塊で保持すると話した。  
文字の場合であっても例外ではない。

`a` や `b` といった文字は全て「数値」の形で扱われる。

試しにこの様にコーディングしてみてほしい。

```c++
// 文字列の表示    
char str[10];
str[0] = 97;
str[1] = 0;
printf("%s\n", str);

// ちなみにこれでも [a] と表示される
printf("%c\n", 97);
```

`a` と表示されるはず。  
97 という数値を文字として扱うと、アルファベットの半角小文字 `a` になるという事。

各文字に番号がふられている、という認識で問題ない。  
アルファベットの半角小文字 `a` は 97 番、といった具合。

## **日本語の文字列**

文字列で少々厄介なのがひらがな、カタカナ、漢字。  
先ほどの例の `abc` を `あいう` に変えてみてほしい。

```c++
// 文字列の表示
char str[10];
str[0] = 'あ';
str[1] = 'い';
str[2] = 'う';
str[3] = '\0'; // ヌル文字

printf("%s\n", str);
```

ちゃんと表示されない事がわかるだろうか？  
では次にこの様にしてみてほしい。

```c++
// 文字列の表示
char str[10] = "あいう";
printf("%s\n", str);
```

こうすると、皆さんの環境ではちゃんと表示されると思う。

ではこの状態で `str[0]` だけ表示させると、どうなるだろうか？

```c++
// 文字列の表示
char str[10] = "あいう";
// 一文字目だけ表示する
printf("%c\n", str[0]);
```

ちゃんと出ない。

全ての文字は番号で表されるとお話ししたが、  
実は「あ」というひらがなは `char` 型一つだけでは表せない。

`char` 型は 1 バイト。  
1 バイトで表現できる数値の最大は 255 となる。

半角小文字の「a」を指す 97 番なら 1 バイトで十分。  
では「あ」の番号はいくらだろうか？  
ShiftJIS という文字コードでは 33440 番になる。

255 までしか入らないのに、こんな番号は入るわけがない。  
そのため、ひらがなやカタカナ、漢字は `char` 型を 2 つ使う。  
`str[0]` と `str[1]` の 2 つを利用して、「あ」という一文字を保持している。

`str` の配列は要素数が 10 あるが、`あいう` と 3 文字保持するだけで  
6 つ使ってしまう事になる。  
正しく表示する場合は最後にヌル文字が必要なので合計 7 つ使う事になる。

## **文字コード**

この文字を表す番号を<span style="color: red;">**文字コード**</span>と呼ぶ。  
半角アルファベットや半角数字、半角の記号は  
**アスキーコード** と呼ばれる文字コードで表される。  
アスキーコードは 8 ビット、0 から 255 までで文字が割り振られている。  
つまり、アスキーコードであれば `char` 型一つで一文字を表す事ができる。

ひらがな、カタカナ、漢字、全角記号は結構ややこしい。  
先程ひらがなの「あ」は ShiftJIS では 33440 と述べたが、  
文字コードの種類によって番号が異なる。

- UTF-16 の場合 : 12354 番
- UTF-8 の場合  : 14942594 番

全部ひらがなの「あ」を指す番号。  
文字は同じでも、それを表す番号が文字コードの種類分存在している。

「文字化け」という言葉を聞いた事があるかもしれないが、  
あれは異なる文字コードで文字を表示しようとした際に起こる現象。
