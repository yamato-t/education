# **構造体とメモリ**

---

## **構造体型の変数が宣言された際のメモリ状況**


構造体は複数の変数を一つに纏めている物と紹介した。  

当然、複数の変数が纏まっている為、  
<span style="color: red;">**構造体型の変数が一つ宣言されるだけで、そのメンバー分のメモリが確保される事になる**</span>。

今回用意した ```Parameter 型```でいうと

- int 型が 4 つ
- float 型が 1 つ
- char 型が １つ

実際に、各変数が利用しているメモリサイズを取得してみる。

```c++
int i;
int int_size = sizeof(i);
printf("int型変数のサイズは %d", int_size);

float f;
int float_size = sizeof(f);
printf("float型変数のサイズは %d", float_size);

char c;
int c_size = sizeof(c);
printf("char型変数のサイズは %d", c_size);

```

int 型のバイト数は 4  
float 型のバイト数は 4  
char 型のバイト数は 1   

なので ```Parameter 型```の変数を一つ宣言しただけで、  
**最低でも (4 * 4) + 4 + 1 = 21 バイト分のメモリが利用される事になる**。


--- 
## **構造体型の変数のサイズ**

では実際に確認してみる。

```c++
Parameter p;
int parameter_size = sizeof(p);
printf("Parameter型変数のサイズは %d", parameter_size);
```

今回用意した ```Parameter 型```の変数のサイズは「24」 になる。

メンバー変数の合計のメモリサイズは 21 で正しい。  
だが実際に構造体型の変数のサイズを調べると 「24」。  

実はコンパイル時にとあるルールに基づいて、  
 <span style ="color: red;">**構造体のどこかに空白のデータが自動的に追加**</span>される。

これにはちゃんと理由がある。  
**メモリにはコンピュータがアクセスしやすい区切りが存在する為**。

手元のプログラムは CPU と呼ばれる部品で動く。  
変数が宣言されると、変数の為にメモリが確保されるが、  
その確保されたアドレスに値を書き込んだり、  
あるいはそのアドレスから値を読み取ったりするのは CPU が行っている。    

一連の動作をメモリアクセスと呼ぶが、こういったメモリアクセスの際、  
 <span style ="color: red;">**CPU にとってアクセスするのに都合のいいアドレスやサイズが存在する**</span>  。

- このアドレスから始まっていると CPU としてはありがたい。  
- アクセス範囲がこのサイズになっていると CPU としてはありがたい。  

という感じ。

構造体はその影響をもろに受ける。  
その為 CPU がアクセスしやすいサイズになるように、  
コンパイル時に空白データが自動で挿入される。

こういった「コンパイラが行うメンバーの配置調整（＝サイズ変化）」を <span style ="color: red;">**アライメント**</span> と呼ぶ。  
そしてアライメントの結果、埋められた空白のデータを <span style ="color: red;">**パディング**</span> と呼ぶ。

「どういう基準でどこにパディングが入るのか」は別の機会に話す。  
まずは「そういった物がある」と留意しておいてほしい。

## **構造体が引数で渡された際のメモリ状況**

関数の引数で変数が渡されると、受け取ったときに変数が複製される。  
それは構造体も例外ではない。

```c++
// ---------------------------------------------------------
/**
 * @brief	各パラメータを表示する関数
 * @param	parameter 各パラメータ
 */
void DispParameter(Parameter parameter)
{
    parameter.hp = 10;

    printf("HP:%d\n", parameter.hp);
    printf("MP:%d\n", parameter.mp);
    printf("素早さ:%c\n", parameter.speed);
    printf("攻撃力:%d\n", parameter.attack);
    printf("防御力:%d\n", parameter.defense);
    printf("魔力:%2.1f\n", parameter.magic);
}
```

引数で受け取った構造体型変数のメンバー変数の内容を一部変更した。  
この時、この printf で表示される内容はどうなるか？
```c++
printf("HP:%d\n", parameter.hp);
```

では、こうなった時どう表示されるか？

```c++
DispParameter(player_parameter);
printf("HP:%d\n", player_parameter.hp);
```

関数の中で変更しても `player_parameter` の内容は変わらない。  

関数の引数で変数を受け取ると、関数内で利用する為の変数として複製される。  
渡された変数の内容と全く同じ別物が作られる。

複製や別物というのが意識しにくい方は、メモリの図を思い浮かべるといい。  

<span style="color: red;">**同じサイズ、同じ内容の物が、別のメモリ領域に追加される**</span>と考えてもらうとわかりやすい。
