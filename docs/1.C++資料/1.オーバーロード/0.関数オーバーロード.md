# **関数オーバーロード**

---

同じ関数名でも、引数の数や型が違う場合に、  
別の関数として定義する事が出来る。  
これを<span style="color: red;">**関数のオーバーロード**</span>と呼ぶ。  

---

## **関数オーバーロードの利用方法**

通常の関数の宣言や定義と、  
同じようにするだけでいい。

```c++
// 関数名は全て同じだが、引数の数や型が違う為、別物として宣言、定義できる

void functionA();
void functionA(int i);
void functionA(char c, int i);

// 引数なしの functionA を定義
void functionA()
{
	std::cout << "none" << std::endl;
}

// 引数が整数一つの functionA を定義
void functionA(int i)
{
	std::cout << i << std::endl;
}

// 引数が文字一つと整数一つの functionA を定義
void functionA(char c, int i)
{
	std::cout << c << " and " << i << std::endl;
}

```

呼び出しの際も、通常の関数と同じやり方で問題ない。  
引数の差異によって処理される関数が変わる。

```c++
// それぞれの関数を呼び出す
functionA();
functionA(1);
functionA('a', 2);

```

関数オーバーロードは、クラスのメンバ関数でも利用できる。

---

## **関数オーバーロードが出来ないパターン**

引数の数や型違いでオーバーロードが可能になるが、  
<span style="color: red;">**戻り値の違いのみではオーバーロード出来ない**</span>。


```c++

// 戻り値の違いのみでオーバーロードする事は出来ない
// 下記コードはコンパイルエラーになる

// 戻り値のみが違う関数を宣言、定義する
void functionA();
int  functionA();

// 引数なしの functionA を定義
void functionA()
{
	std::cout << "none" << std::endl;
}

// 戻り値が整数で、引数なしの functionA を定義
int functionA()
{
	return 1;
}

```

---

## **関数オーバーロードの利用意義**

関数オーバーロードは、  
<span style="color: red;">**機能は同じだが使用する型が違う場合**</span>に利用する事が多い。

例えば整数同士を加算する関数と、浮動小数同士を加算する関数を作るとする。  
当然引数の型が違うので、Cでは下記のように、  
型も関数名に含める等する必要があった。

```c++
// 整数同士の加算
int addInt(int l, int r);
// 浮動小数同士の加算
float addFloat(float l, float r);

```

これらの関数が「加算する」という機能を持つ事は同じである為、  
関数オーバーロードを利用すると、下記の様に宣言できる。

```c++
// 整数同士の加算
int add(int l, int r);
// 浮動小数同士の加算
float add(float l, float r);

```

こうすると何が良いかというと、**使う側が使いやすくなる**。  
使う側にとっては、整数だろうが浮動小数だろうが、とにかく**加算したい**だけであって、  
型によって関数を使い分けないといけないのは面倒。

「加算する」という機能を持つ関数名を統一しておけば、  
その関数だけ覚えて利用するだけで、型に依存せず望んだ結果を得る事が出来る。  

---

## **型に依存しない事の重要性**

なお、この様に**型に依存しない処理**は**抽象化された処理**と呼ばれる。  
**抽象化された処理**を作成し、利用する事は、  
C++において**オブジェクト指向と同様に重要な要素**となっている。
