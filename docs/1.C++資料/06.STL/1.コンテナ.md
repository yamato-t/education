# **コンテナ**

コンテナの中でも特に利用機会の多いであろう物を挙げる。

---
## **vector**

複数のオブジェクトを配列のような形で保持できるコンテナ。  
要素が順序を持ち、配列の様に連続しているコンテナは<span style="color: red;">**シーケンスコンテナ**</span>と呼ばれる。

### **vector の使い方** 
下記は vector の利用例になる。

```c++
#include <iostream>
//　vector を利用する
#include <vector>

// エントリー関数
int main()
{
	// int を扱う vector を宣言する
	std::vector<int> nums;

	// 追加方法の例
	{
		// push_back を使って末尾に追加
		nums.push_back(1);
		nums.push_back(3);
		nums.push_back(2);

		// emplace_back を使って末尾に追加
		nums.emplace_back(8);
		nums.emplace_back(11);
	}

	// 取得方法の例
	{
		// [] で添え字で取得する
		for (int i = 0; i < nums.size(); ++i)
		{
			// 添え字を使って要素を取得する
			int num = nums[i];
			std::cout << num << std::endl;
		}

		// at 関数でインデックス(○○番目)を指定して取得する
		int num = nums.at(0);
		std::cout << num << std::endl;
	}

	// 更新方法の例
	{
		// [] の添え字で更新する
		nums[1] = 15;

		// at 関数の戻り値で更新する（戻り値が参照である為可能）
		nums.at(0) = 5;
	}	

	// 削除方法の例
	{
		// 末尾を削除する
		nums.pop_back();

		// 全部削除する
		nums.clear();	
	}	
}
```

追加方法

- ```push_back``` 関数で末尾に追加 
- ```emplace_back``` 関数で末尾に追加

取得方法

- ```[]``` で添え字を指定して取得
- ```at``` 関数で格納されている番号を渡して取得

更新方法

- ```[]``` で添え字を指定して更新
- ```at``` 関数の戻り値（参照）で更新

削除方法

- ```pop_back``` 関数で末尾を削除
- ```clear``` 関数ですべて削除

## **map**

キーとなる情報と、ある値をペアで保持できるコンテナ。  
順序に依存せず、キーに基づいて値を保持するコンテナは<span style="color: red;">**連想コンテナ**</span>と呼ばれる。  
map の要素はキーに合わせて自動的にソートされる。

### **map の使い方**
下記は map の利用例になる。

```c++
#include <iostream>
//　map を利用する
#include <map>

// エントリー関数
int main()
{
	// キーがchar、値が float を扱う map を宣言する
	std::map<char, float> map;

	// 追加方法の例
	{
		// 'a' をキーに値 1.5f を追加する
		map['a'] = 1.5f;

		// emplace 関数 を利用する
		// 'c' をキーに値 3.1f を追加する
		map.emplace('c', 3.1f);

		// std::pair と insert 関数を利用する
		// 'b' をキーに値 2.4f を追加する
		map.insert(std::make_pair('b', 2.4f));	
	}

	// 取得方法の例
	{
		// [] でキーを指定して値を取得する
		float f1 = map['a'];
		std::cout << f1 << std::endl;

		// at 関数でキーを指定して値を取得する
		float f2 = map.at('c');
		std::cout << f2 << std::endl;
	}

	// 更新方法の例
	{
		// [] にキーを指定して更新する
		map['a'] = 0.3f;

		// at 関数の戻り値で更新する（戻り値が参照である為可能）
		map.at('c') = 100.5f;
	}
	
	// 削除方法の例
	{
		// キーを指定して削除する
		map.erase('c');

		// 全部削除する
		map.clear();
	}
}
```

追加方法

- ```[]``` で追加 
- ```std::pair``` と ```insert``` 関数で追加
- ```emplace``` 関数で追加

取得方法

- ```[]``` にキーを指定して取得
- ```at``` 関数でキーを渡して取得

更新方法

- ```[]``` にキーを指定して更新
- ```at``` 関数の戻り値（参照）で更新

削除方法

- ```erase``` 関数でキーを指定して削除
- ```clear``` 関数ですべて削除


[] で存在しないキーを指定して取得しようとした場合、  
自動でキーと、キーに対応する値の初期値が追加されるという事が起こる。  

また map は要素が追加された際、キーを元に自動的にソート(昇順)が行われる。  
上記の例では、追加順に関わらず「a」「b」「c」の順番で格納される事になる。

map の場合、キーを重複して追加することはできない。
```c++
// 'a' をキーに値 1.5f を追加する
map['a'] = 1.5f;

// 1.5f が 2.5f に更新される
map['a'] = 2.5f;

// 'a' のキーがすでに追加済みである為、無視される
map.insert(std::make_pair('a', 2.4f));

// 'a' のキーがすでに追加済みである為、無視される
map.emplace('a', 3.1f);
```  

### **multimap と unordered_map**
stlの連想コンテナには map とよく似た物として、  
**multimap** と **unordered_map** がある。

**multimap** は、キーの重複を認める map になっている。  
同一のキーで複数の値を扱いたい場合に利用できる。

**unordered_map** は、キーをハッシュという値に変換し、  
ハッシュと値を紐づけて管理している。  
要素がソートされていない代わりに、パフォーマンスが map より優れている。
