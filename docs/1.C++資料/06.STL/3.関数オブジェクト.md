# **関数オブジェクト**

---
## **stl 関数オブジェクト**

下記は stl に用意されている、  
```std::plus``` と ```std::greater``` という関数オブジェクトの利用例になる。

```c++
#include <iostream>
//　pulsとgreater を利用する
#include <functional>

// エントリー関数
int main()
{
	// float 同士の足し算を行う関数オブジェクト
	std::plus<float>	fPlus;
	// int 同士の大きさ比較を行う関数オブジェクト
	std::greater<int>   iGreater;

	// 変数だが関数の様に利用できる

	// fには合計が入る
	float	f = fPlus(2.3f, 4.1f);
	// 第一引数が第二引数より大きければ true 
	bool	i = iGreater(2, 4);

}
```

上記を見ると、```fPlus``` や ```iGreater``` という変数が宣言されている。  
だが、それを利用している箇所を見ると、```()``` を付けて関数の様に扱っている。

```fPlus``` や ```iGreater``` が関数オブジェクトとなる。  

()演算子オーバーロードが各構造体テンプレートに用意されており、  
構造体が実体化する事で、実引数の型に合わせた足し算や比較を行えるようになる。  

単純な仕組みである為、stl 関係なく、  
任意の挙動を行う関数オブジェクトを自作する事も容易。

昨今では、関数オブジェクトはラムダ式という形で記述されることが多い。  
是非調べてみてほしい。

## **関数を変数の形で保持する**

関数オブジェクトに限った話ではないが、  
自身が作成した関数を変数の形で扱う例を挙げておく。

ある関数を別の関数内で呼び出し、その結果を利用したい時、  
下記のような書き方をすると思う。

```c++
#include <iostream>

// 関数A
char A()
{
	return 'A';
}

// 関数
void function()
{
	// 関数Aを呼び出す
	char c = A();
	std::cout << "結果は " << c << std::endl;
}

// エントリー関数
int main()
{
	function();
}
```

ここで、条件によって呼び出したい関数が変わるとする

```c++
// 関数A
char A()
{
	return 'A';
}

// 関数B
char B()
{
	return 'B';
}

// 関数
void function(int type)
{
	char c = {};
	if (type == 1)
	{
		c = A();
	}
	else
	{
		c = B();
	}
	std::cout << "結果は " << c << std::endl;
}

// エントリー関数は割愛

```

この形でも確かに問題ない。  
だが呼び出す可能性がある関数が増えていけばどうなるか。

非常に保守しにくい形になっていくと思う。  
そんな時、関数を変数の形で保持するとシンプルな形にする事ができる。  
以下はその一例になる。

```c++
#include <iostream>

// function を扱う
#include <functional>

// 関数A
char A()
{
	return 'A';
}

// 関数B
char B()
{
	return 'B';
}

// 関数
// 引数に関数を渡せるようにする
void function(std::function<char()> f)
{
	// 渡された関数を呼び出す
	char c = f();
	std::cout << "結果は " << c << std::endl;
}

// エントリー関数
int main()
{
	// 引数に関数そのものを渡す
	function(B);
}
```

```function``` の引数に、```std::function<char()>```という型が指定されている。  
```std::function``` が関数を保持するクラステンプレートであり、  
実引数が ```char()``` となっているが、この実引数は保存する関数のフォーマットを指している。  
```char``` が戻り値で ```()の中``` が引数を表している。   
ただし、今回保持させたい関数 (関数A や 関数B) には引数がないので空になっている。

関数の内側で条件分岐して関数を呼び出すのではなく、  
呼び出したい関数そのものを渡せると、シンプルな形に変えられる。

また、渡した関数を保持し続け、特定のタイミングで呼び出す事で、  
他の制御から任意のタイミングで処理してもらう関数(**コールバック関数**)を実現する事ができる。
