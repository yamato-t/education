## **vector のイテレータ**

先ほどの vector の例で
 ```main``` 関数の中身を抜粋している。  
 ```for``` に注目してもらいたい

```c++
// エントリー関数
int main()
{
    ・・・省略

	// vector に入っている整数をイテレータを利用して出力する
	for (std::vector<int>::iterator it = nums.begin(); it != nums.end(); ++it)
	{
		// 取得方法の例
		{
			// イテレータはポインタの様に利用する
			int num = *it;
			std::cout << num << std::endl;
		}

		// 更新方法の例
		{
			// 要素の内容を「１００」に更新する
			*it = 100;	
			int num = *it;
			std::cout << num << std::endl;	
		}
	}

    ・・・省略 
}
```
```for``` の () を分解すると下記の様になっている
```c++
// std::vector<int>に対応するイテレータ型で it を宣言する
std::vector<int>::iterator it

// it に nums の先頭位置を代入する
it = nums.begin()

// it が nums の末端位置ではない事を確認する
it != nums.end()

// 次の位置へ進む
++it
```

イテレータはコンテナの位置とその要素を保持し、  
アスタリスクを付ける事で、その要素にアクセスする事が可能になる。  
また、イテレータを数値の様にインクリメントする事で、次の位置に移動する事ができる。

一点、注意点として、  
 ```begin``` は実際に要素が入っている先頭の位置を指すイテレータが取得できるが、  
 ```end``` は **最後に追加された要素の次** 、つまり **無効な位置** を指しているイテレータを取得する。

上記の ```for``` は、イテレータを利用して、  
コンテナの末端（無効な位置）になるまで要素を出力する、  
といった処理になる。

### **イテレータを利用した追加と削除**
vector ではイテレータを利用する事で、  
指定の位置に新しく要素を追加（挿入）する事と、  
指定の位置の要素を削除する事ができる。

```c++
// エントリー関数
int main()
{
	// 整数を扱う vector を宣言する
	std::vector<int> nums;

	{
		nums.push_back(1);
		nums.push_back(3);
		nums.push_back(2);
	}

	// 挿入
	{
		// 先頭から２進めた位置、つまり「２」が入っている位置を取得する
		std::vector<int>::iterator position = nums.begin();
		position += 2;

		// 「２」の位置に「５」を追加（挿入）する
		nums.insert(position, 5);

		// nums の要素を出力する
		・・・省略
	}

	// 削除
	{
		// 先頭から１進めた位置、つまり「３」が入っている位置を取得する
		std::vector<int>::iterator position = nums.begin();
		position += 1;

		// 「３」の位置を削除する
		nums.erase(position);

		// nums の要素を出力する
		・・・省略
	}
}

```

<span style="color: red;">**イテレータを利用してforループで繰り返して削除する際は注意が必要**</span>。  
下記コードはクラッシュする。

```c++
// for ループで削除する場合
for (std::vector<int>::iterator it = nums.begin(); it != nums.end(); ++it)
{
	nums.erase(it);
}
```
```erace``` 関数はイテレータが指している位置そのものを削除する。  
その為 ```erase``` 関数が処理された直後、引数で指定した ```it``` が指している位置は、  
 **すでに不正な物になってしまっている** 。  
この状態から、繰り返しでインクリメントするとクラッシュを起こす。

下記は回避方法の一例となる。
```c++
// for ループで削除する場合
for (std::vector<int>::iterator it = nums.begin(); it != nums.end(); /*++it ここのインクリメントをやめる*/)
{
	// erase の戻り値で、削除した次の位置を取得する
	it = nums.erase(it);
}
```
まず、 ```for``` の () でインクリメントをやめる。  
その上で、```erace``` 関数の戻り値でイテレータを上書きする。  
```erace``` 関数の戻り値は、削除した次の位置を指すイテレータを返す為、  
削除する度にイテレータを受け取ると、正しく順番に削除する事ができる。

## **map のイテレータ**

基本的な使い方は vector と同じと思ってもらっていいが、  
連想コンテナのイテレータである為、一部違いがある。

```c++
// エントリー関数
int main()
{
    ・・・省略

	// vector と同じようにイテレータで開始位置を取得して、終了位置までループする
	for (std::map<char, float>::iterator it = map.begin(); it != map.end(); ++it)
	{
		// 取得方法の例
		{
			// it の位置にある要素はキーと値のペアになっている為、それぞれ取得できる
			char	key = it->first;
			float	value = it->second;
			std::cout << key << " " << value << std::endl;
		}

		// 更新方法の例
		{
			// 変更が可能なのは値だけ
			it->second = 100.0f;
			char	key = it->first;
			float	value = it->second;
			std::cout << key << " " << value << std::endl;
		}
	}

	・・・省略
}
```

イテレータを取得し、インクリメントする事で次の位置にある内容にアクセスできる事は同じ。  
違いは、**イテレータがキーと値のペア**になっている事。  
```it->first``` にはキーが、```it->second``` には値が入っている。  
キーは値を取得する絶対的な目印である為、変更はできない。  
値は変更する事ができる。

もう一点、直接数値を指定して位置を移動させる事はできない。  
下記の様なコードはコンパイルエラーになる。
```c++
// コンパイルエラーになる
std::map<char, float>::iterator position = map.begin() + 3;
```

### **イテレータを利用した追加と削除**

vector と同じく、イテレータを利用して追加と削除を行う事ができる。

```c++
// エントリー関数
int main()
{
	・・・省略

	// 挿入
	{
		// 位置を指定して追加する（ただしソートされる為意味がない）
		std::map<char, float>::const_iterator position = map.begin();
		map.insert(position, std::make_pair('x', 100.0f));
	}

	// 削除
	{
		for (std::map<char, float>::iterator it = map.begin(); it != map.end(); ++it)
		{
			// 指定のキーだったらその位置を指定して削除する
			char key = it->first;
			if (key == 'b')
			{
				map.erase(it);
				break;
			}
		}
	}

	・・・省略

}
```
**イテレータを利用してforループで繰り返して削除する際は注意が必要** 。  
```erase``` 関数で指定したイテレータの指す位置が、  
削除後に不正になる点は vector と同じである為、  
繰り返し削除する場合は、イテレータを戻り値で上書きしていく必要がある。
