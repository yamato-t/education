# テンプレート

---
## **ジェネリックプログラミング**

<strong><span style="color: red;">型に依存しないプログラミング手法</span></strong>を指す。

ある値同士を合計したい関数を用意する際に、  
整数だろうと小数だろうと、合計する事は同じである為、  
関数を利用する側からすると、同じ呼び出しで利用出来る方が便利。  

そのような便利な形を実現する手法を **ジェネリックプログラミング** と呼ぶ。

---
## **C++ におけるジェネリックプログラミング**
今回紹介するテンプレートは、  
C++ で **ジェネリックプログラミング** を実現する為の **機能** になる。  
**機能** である為、まずは簡単な例で書き方を覚えて貰いたい。  

興味が出てきたり、より使ってみたいといった事になれば、  
改めて仕組みを調べてもらえればいいかと思う。

テンプレートは奥が深く、傍から見ると魔法と思えるような事ができる。  
半面、デバッグでコードが追いにくくなる等、デメリットもある。  
また、使いこなそうとすると、  
テンプレートがコンパイル時にどの様に扱われるかを知る必要が出てきたり、  
記述している内容から実装が素直に理解できなくなってくるなど、  
難度が一気に高くなる。
