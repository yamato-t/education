# **関数テンプレート**

まずは最も使われるであろう<span style="color: red;">**関数のテンプレート**</span>を紹介する。

---
## **同じ機能で扱う型のみが違う関数**

下記のコードを見てもらいたい。

```c++
#include <iostream>

// int 型の数値を表示する関数
void disp(int value)
{
    std::cout << value << std::endl;
}

// float 型の数値を表示する関数
void disp(float value)
{
    std::cout << value << std::endl;
}

// エントリー関数
int main()
{
    // int 型の表示を行う
    disp(2);

    // float 型の表示を行う
    disp(1.5f);

    return 0;
}

```

上記 ```disp``` 関数は、引数で扱う型が違う事による **関数オーバーロード** で二つ定義されている。  
使う側としては、大変扱いやすい物になる。  

そこで、今度は作る側が新しく追加しなければならない時の事を考える。  
仮に ```char``` 型の ```disp``` 関数が必要になれば、当然新しく宣言、定義しなければならない。  
どうせなら **disp 関数の定義も一つの記述で実現したい** 。

そんな時に**関数テンプレート**が利用できる。


---
## **関数テンプレートの宣言と定義**
先にコードを紹介する。

```c++
#include <iostream>

// T 型の数値を表示する関数テンプレート
template<class T> 
void disp(T value)
{
    std::cout << value << std::endl;
}

```

 ```disp``` 関数が大きく変わっている。  
 まず ```template<class T>``` という記述が関数の前に追加されている。  
 ```template``` がテンプレートを示すキーワードであり、  
 ```class T``` がテンプレートで扱う仮の型を示している。  
ここで ```T``` は **テンプレート仮引数** と呼ばれる。

この記述がある事で、その後に宣言、定義する関数が **関数テンプレート** である示す。

そのあとの関数定義を見てみると、引数の型が ```T``` 型になっている。  
これは上記にある **テンプレート仮引数** の名称によって決まる。  

**とりあえず仮の型である T 型で、disp 関数の内容を作っておく** 、  
といった認識でいてもらえればいい。

内容は今までと同じ。

---
## **関数テンプレートの利用**

```c++

// エントリー関数
int main()
{
    // int 型の表示を行う
    disp<int>(2);

    // float 型の表示を行う
    disp<float>(1.5f);

    return 0;
}
```

利用する側を見ると、 ```<int>```、 ```<float>``` という記述がついている。  
これによって、関数テンプレートの ```T``` 型を具体的な型に置き換えられる。  
ここで当て込まれる具体的な引数型は **テンプレート実引数** と呼ばれる。

これらを行う事で **置き換えた引数型に対応する関数が処理される** といった挙動をとるようになる。  

なお **テンプレート実引数は省略することができる** 。  
この場合「```T``` に何が入るのか」が呼び出され方から推論され、  
対応する型の関数が処理されるようになる。

さらに**実引数の辻褄が合えば、何でも放り込める**為、  
下記の様に文字列を入れる事も可能。
 
```c++
// エントリー関数
int main()
{
    // int 型の表示を行う
    disp(2);

    // float 型の表示を行う
    disp(1.5f);

    // disp 関数の内容は T が  const char* に置き換えられてもコンパイルエラーにならない
    // その為文字列を渡しても問題なく表示される
    disp("文字を表示します");

    return 0;
}

```

---
## **関数テンプレートの注意点**
関数テンプレートは特殊なキーワードが付いた関数のような記述だが、  
実は<span style="color: red;">**宣言と定義の時点では利用できる関数として完成していない**</span>。

関数テンプレートそれだけでは意味がなく、  
**使われて初めて、型に合致した関数が生成される**という認識でいてほしい。  
関数の生成は<span style="color: red;">**コンパイル時に行われる**</span>。

```c++
Ｔ型の値を表示する関数テンプレートが定義されている
template<class T> 
void disp(T value)
{
    std::cout << value << std::endl;
}
            ↓
            ↓
            ↓

T を int 型として利用している物が見つかった
関数テンプレートを元にして、T を int 型に置き換えた関数が自動的に作られる
disp(2);
            ↓
            ↓
            ↓

int 型の数値を表示する関数が作られた
これが実際に呼び出される
void disp(int value)
{
    std::cout << value << std::endl;
}

```

関数テンプレートを元に自動的に作られる事を**実体化**と呼ぶ。  

**利用できる関数として完成していない** というのは、  
下記の様に宣言、定義、関数を利用、といった形でファイルを分けてみるとわかる。
 

template.h
```c++
#pragma once 

// T 型の数値を表示する関数テンプレートの宣言
template<class T> 
void disp(T value);
``` 

template.cpp
```c++
#include <iostream>
#include "template.h"

// T 型の数値を表示する関数テンプレートの定義
template<class T> 
void disp(T value)
{
    std::cout << value << std::endl;
}
``` 

main.cpp
```c++
#include "template.h"

// エントリー関数
int main()
{
    // int 型の表示を行う
    disp(2);

    return 0;
}
```

コンパイルしてみると **リンクエラー** というものが出る。  

ちゃんとインクルードしているので、問題なさそうに見えるが、  
<span style="color: red;">**関数テンプレートは宣言だけが見えていても、定義が別ファイルにある場合、  
型に合わせて実体化する事ができなくなり、使う事ができない**</span> 。

ではこの場合どうすればいいかという事だが、明示的に実体化させればいい。

template.cpp
```c++
#include <iostream>
#include "template.h"

// T 型の数値を表示する関数の定義
template<class T> 
void disp(T value)
{
    std::cout << value << std::endl;
}

// int 型で利用する事を予め宣言しておく
template void disp<int>(int value);
``` 

一番最後の行が、具体的な型で明示的に実体化している箇所になる。  
```template``` のキーワードの後に、 ```T``` を ```int``` に置き換えた ```disp``` 関数が宣言されている。  
明示的に実体化する場合は宣言だけでいい。

ただこの場合、予め利用する型が何なのか把握しておく必要がある。  
その為<span style="color: red;">**関数テンプレートの定義はヘッダーに書く事が多い**</span>。
