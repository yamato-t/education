# 課題

---
## **下記の仕様に合わせて設計を考えプログラムを作成してください**
味方陣営と敵陣営に分かれて戦うゲームを作る。  

### 概要

 １. 味方陣営から一人選び、行動を設定する  
 ２. 敵陣営からランダムに一人選ばれ、ランダムに行動が設定される  
 ３. 上記の二名が設定された行動を取り、ルールに基いて陣営の「戦力」を削りあう  
 ４. ３までを「1ターン」とし、ターンを繰り返して、先に相手陣営の「戦力」をゼロにした方が勝ち

### 基本ルール

 - 各陣営には「剣士」「魔法使い」「召喚士」の3人が所属している
 - 各陣営は「戦力」というステータスを持つ
 - 各キャラクターは「攻撃力」というステータスを持つ
 - 各キャラクターは「攻撃する」「防御する」という行動を取る事が出来る
 - 「攻撃する」場合、相手の陣営の戦力を攻撃力分削る事が出来る
 - 「防御する」場合、相手の攻撃で戦力が削られる事を防ぐ事が出来る
- お互いの陣営が攻撃した場合、攻撃力の高い方が相手陣営の戦力を削る事が出来る
- お互いの攻撃力が一致している場合、ドロー（何も起こらない）。
- 攻撃と防御が分かれた場合は下記の様な結果になる


|  防御→<br>攻撃↓ |  剣士  |  魔法使い  |  召喚士  |
| :----: | :----: | :----: | :----: |
|  剣士 |  ドロー<br>(何も起こらない)  | カウンター発生<br>(攻撃側の戦力を攻撃側の攻撃力x2で削る) | クリティカル攻撃<br>(防御側の戦力をx2で削る) |
|  魔法使い  | クリティカル攻撃<br>(防御側の戦力をx2で削る) | ドロー<br>(何も起こらない) | カウンター発生<br>(攻撃側の戦力を攻撃側の攻撃力x2で削る) |
|  召喚士  |  カウンター発生<br>(攻撃側の戦力を攻撃側の攻撃力x2で削る)  | クリティカル攻撃<br>(防御側の戦力をx2で削る) | ドロー<br>(何も起こらない) |
 


### キャラクタ毎の特殊ルール
各キャラクタは、決められた条件を満たすと特殊な能力が一度だけ発動する

|   |  条件  |  能力  |
| :----: | :----: | :----: 
|  剣士 | 味方陣営が累計6回攻撃を受ける | 自身（剣士）が次にクリティカル攻撃を発生させた際、攻撃力がx6になる |
|  魔法使い | 味方陣営がクリティカル攻撃を3回受ける | 自身（魔法使い）が攻撃した回数x10戦力を回復させる |
|  召喚士 | 味方陣営がカウンター発生を3回成功させる | 自身（召喚士）が次に行動した際、結果を必ずドローにする |


※  
コンソールプロジェクトで作成してください。  
陣営が持つ初期戦力は「300」としてください。  
三人の攻撃力の内訳は自由ですが、合計が「100」になる様にしてください。  
各ターンで「何が起こり」「どんな結果になったか」を可能な限りコンソールに表示してください。  
味方陣営から人や行動を選ぶ方法は、どの様な形でも構いません。
