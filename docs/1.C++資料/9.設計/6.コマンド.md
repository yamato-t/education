# **コマンドパターン**

---

## **コマンドパターンとは**

コマンドパターンとは、「操作」そのものをオブジェクトとして表現する  
設計パターン。  

通常は関数を直接呼び出して終わりだが、  
コマンドパターンでは操作をクラスにカプセル化することで、  
操作の記録・取り消し・再実行・遅延実行が可能になる。  

ゲーム開発では入力バッファ・Undo/Redo・  
リプレイ機能の実装に広く使われる。  

---

## **なぜコマンドパターンが必要か**

### **直接呼び出しの限界**

入力に応じてキャラクターを操作する処理を素直に書くと次のようになる。  

```c++
void InputHandler::Update()
{
    if (input.IsPressed(KEY_UP))    player.MoveForward();
    if (input.IsPressed(KEY_DOWN))  player.MoveBack();
    if (input.IsPressed(KEY_A))     player.Jump();
    if (input.IsPressed(KEY_B))     player.Attack();
}
```

この書き方では以下の要件に対応できない。  

- キーコンフィグ（ボタン割り当ての変更）
- 操作の Undo（一手前に戻る）
- リプレイ（過去の操作を再現する）
- AI による操作（プレイヤーと同じ行動系統を使い回す）

いずれも「操作をデータとして保存・再利用する」必要があるが、  
直接呼び出しではそれができない。  

---

## **コマンドパターンの仕組み**

### **二つの要素**

- <span style="color: red;">**Command インターフェース**</span>
    - すべての操作に共通する `Execute()` を定義する。  
    - Undo が必要な場合は `Undo()` も定義する。  
- <span style="color: red;">**具体的な Command クラス**</span>
    - 「移動」「ジャンプ」「攻撃」など、各操作をクラスとして実装。  
    - 実行に必要な情報（対象・パラメータ）をメンバ変数に保持する。  

---

## **実装例**

### **基本的な実装：キーコンフィグへの応用**

```c++
// Command インターフェース
class Command
{
public:
    virtual ~Command() {}
    virtual void Execute() = 0;
};

// 具体的なコマンド
class MoveForwardCommand : public Command
{
public:
    MoveForwardCommand(Player& player) : player(player) {}
    void Execute() override { player.MoveForward(); }
private:
    Player& player;
};

class JumpCommand : public Command
{
public:
    JumpCommand(Player& player) : player(player) {}
    void Execute() override { player.Jump(); }
private:
    Player& player;
};
```

```c++
// InputHandler がコマンドのポインタを持つ
class InputHandler
{
public:
    // ボタンとコマンドを紐付ける（キーコンフィグ）
    void SetCommand(int key, Command* command)
    {
        bindings[key] = command;
    }

    void Update()
    {
        for (auto& [key, command] : bindings)
        {
            if (input.IsPressed(key))
            {
                command->Execute();
            }
        }
    }

private:
    std::unordered_map<int, Command*> bindings;
};
```

```c++
// ボタン割り当てを自由に差し替えられる
InputHandler handler;
handler.SetCommand(KEY_UP, new MoveForwardCommand(player));
handler.SetCommand(KEY_A,  new JumpCommand(player));

// キーコンフィグの変更も差し替えるだけ
handler.SetCommand(KEY_A, new MoveForwardCommand(player));
```

### **発展的な実装：Undo / Redo への応用**

Undo に対応するには、実行前の状態を保存して  
`Undo()` で元に戻せるようにする。  

```c++
class Command
{
public:
    virtual ~Command() {}
    virtual void Execute() = 0;
    virtual void Undo()    = 0;
};

class MoveCommand : public Command
{
public:
    MoveCommand(Player& player, float dx, float dy)
        : player(player), dx(dx), dy(dy) {}

    void Execute() override
    {
        // 実行前の位置を保存しておく
        prevX = player.GetX();
        prevY = player.GetY();
        player.Move(dx, dy);
    }

    void Undo() override
    {
        // 保存しておいた位置に戻す
        player.SetPosition(prevX, prevY);
    }

private:
    Player& player;
    float dx, dy;
    float prevX = 0.0f, prevY = 0.0f;
};
```

```c++
// コマンド履歴を管理するクラス
class CommandHistory
{
public:
    void Execute(Command* command)
    {
        command->Execute();
        history.push_back(command);
    }

    void Undo()
    {
        if (history.empty()) return;

        history.back()->Undo();
        history.pop_back();
    }

private:
    std::vector<Command*> history;
};
```

```c++
CommandHistory history;

// 操作を実行しながら履歴に積む
history.Execute(new MoveCommand(player,  1.0f, 0.0f));
history.Execute(new MoveCommand(player,  0.0f, 1.0f));

// 一手前に戻る
history.Undo();
```

### **発展的な実装：リプレイへの応用**

コマンドを時刻とセットで保存しておくと、  
後から同じ順序で `Execute()` を呼ぶだけでリプレイが実現できる。  

```c++
struct TimedCommand
{
    float    time;    // 実行したゲーム時刻
    Command* command;
};

// 録画：操作のたびに記録する
std::vector<TimedCommand> recording;
recording.push_back({ currentTime, new MoveCommand(player, 1.0f, 0.0f) });

// 再生：記録した順に再実行するだけ
for (auto& timed : recording)
{
    timed.command->Execute();
}
```

## **直接呼び出しとの比較**

| 比較項目 | 直接呼び出し | コマンドパターン |
|---|---|---|
| キーコンフィグ | 条件分岐を書き換える必要がある | コマンドを差し替えるだけ |
| Undo / Redo | 別途状態管理が必要で複雑になる | 履歴スタックと `Undo()` で実現 |
| リプレイ | 操作を記録する仕組みがない | コマンドリストを再生するだけ |
| AI 操作 | プレイヤーとは別の仕組みが必要 | 同じコマンドを AI が発行できる |

## **注意点**

### **コマンドクラスが増えすぎる場合**

操作の種類が多いと、コマンドクラスの数も比例して増える。  
ラムダ式でシンプルに書く方法も有効。  

```c++
class LambdaCommand : public Command
{
public:
    LambdaCommand(std::function<void()> exec,
                  std::function<void()> undo)
        : execFunc(exec), undoFunc(undo) {}

    void Execute() override { execFunc(); }
    void Undo()    override { undoFunc(); }

private:
    std::function<void()> execFunc;
    std::function<void()> undoFunc;
};

// 個別クラスを作らずにその場で定義できる
auto cmd = new LambdaCommand(
    [&]() { player.Jump(); },
    [&]() { player.CancelJump(); }
);
```

## **まとめ**

| ポイント | 内容 |
|---|---|
| 目的 | 操作をオブジェクト化し、記録・取り消し・再実行を可能にする |
| 構成要素 | Command インターフェース ＋ 具体コマンド ＋ 履歴管理 |
| 主な用途 | キーコンフィグ・Undo/Redo・リプレイ・AI操作 |
| 発展形 | ラムダ式との組み合わせでクラス数を削減できる |

コマンドパターンの本質は、  
<span style="color: red;">**「操作を呼び出しから切り離し、データとして扱えるようにする」**</span>こと。  
「後から操作を再利用したい」という要件が出てきたとき、  
導入を検討するパターン。
