# **new によるインスタンス化**

実装したクラスは**インスタンス化**する事で利用できるようになる。  
インスタンス化は簡単で、単にクラスを型として変数宣言すればいいだけ。

ただ <span style="color: red;">**インスタンス化の方法は一つではない**</span>。  
前述した変数宣言によるインスタンス化の他に、  
<span style="color: red;">**メモリ確保を行ってアドレスを受け取る**</span>という方法もある。

ここではクラスのインスタンス化に焦点を当てているが、  
この**アドレスを受け取る方法**は、クラスインスタンスだけではなく、  
<span style="color: red;">**あらゆるオブジェクトの生成、つまり int 型などでも利用できる**</span>。

どういう時にどういう方法でのインスタンス化が好ましいのかを把握し、  
状況に合わせて、使い分けていく必要が出てくる。


---

## **new キーワード**

インスタンス化に利用するキーワード。  
<span style="color: red;">**メモリ上にインスタンス用の領域を確保し、そのアドレスをポインタで受け取る**</span>。

インスタンスを直接宣言する方法とは違い、  
<span style="color: red;">**スコープを抜けてもインスタンスが破棄されない**</span>。

---

## **new によるインスタンス化**

通常の変数宣言によるインスタンス化の他に、  
下記の通り `new` キーワードを利用したインスタンス化が可能。

```c++
#include <string>

// 動物クラス
class Animal final
{
public:
        std::string	name_	= {}; // 名前
        int		footNum_	= 0;  // 足の数
};

// エントリー関数
int main()
{
	// インスタンス化①
	{
		// {}内（スコープ内)で利用するインスタンスを作成する

		Animal animal; // インスタンス作成
		animal.name_    = "動物";
		animal.footNum_ = 4;
	}
	// スコープを抜けると animal (インスタンス) が破棄され、メモリ上からも消える


	// インスタンス化②
	{
		// {}内（スコープ内)で利用するインスタンスを new を使って作成する

		// new キーワードの後にインスタンス化したい型を指定
		// 確保したメモリ領域の開始アドレスを取得する
		Animal* pAnimal = new Animal;
		pAnimal->name_ = "動物";
		pAnimal->footNum_ = 4;
	}
	// スコープを抜けても pAnimal が指す先の領域（インスタンス）は破棄されず、メモリ上に残りっぱなしになる
}
```

### **コンストラクタの呼び出し**

クラスの場合、インスタンス化の際にコンストラクタが呼び出されるが、  
new によるインスタンス化であってもちゃんと処理される。  

```c++
// インスタンス化
{
	// new で指定する型のみの場合は通常のコンストラクタが呼ばれる
	Animal* pAnimal1 = new Animal;

	delete pAnimal1;
	pAnimal1 = nullptr;

	// new で指定する型の後ろに引数の形で指定すると、引数付きコンストラクタが呼ばれる
	Animal* pAnimal2 = new Animal("怪物", 3);

	delete pAnimal2;
	pAnimal2 = nullptr;
}
```

## **複数を一度にインスタンス化**

配列の形にする事で、複数を一度にインスタンス化する事が出来る。  
`new` キーワードを利用する場合も同じ様に、配列の形で指定する。    
ただし、取得できるものが開始アドレスである事は変わらない為、  
アクセスする際は注意が必要。


```c++
// インスタンス化①
{
	// 配列の形で複数をインスタンス化する
	Animal animals[3]; 
		
	for (int i = 0; i < 3; ++i)
	{
		animals[i].name_ = "動物";
		animals[i].footNum_ = i;
	}
}


// インスタンス化②
{
	// 配列の形で複数をインスタンス化する
	// new の後の型が配列になっている為、配列の数だけメモリ領域が確保されている
	// ただし、確保したメモリ領域の開始アドレスを取得する事は同じなので、
	// アクセスする際は -> ではなく . になる事に注意
	Animal* pAnimals = new Animal[3];
		
	for (int i = 0; i < 3; ++i)
	{
		pAnimals[i].name_ = "動物";
		pAnimals[i].footNum_ = i;
	}
}

```

---
## **new によるインスタンス化の注意点**

同じインスタンス化でも、new を使ってインスタンス化した場合、  
「確保したメモリ領域の破棄の挙動」に違いが出てくる。

上記、インスタンス化②のコードで取得したインスタンスは、  
確保したアドレスを知っているポインタ変数が、  
スコープを抜けて破棄されてしまった事によって、  
明示的なメモリ領域の破棄が出来なくなり、  
結果メモリに残り続けてしまう。

これは**メモリリーク**と呼ばれ、  
必ず修正しなければならない不具合の一つになる。  

<strong><span style="color: red;">new で確保したメモリ領域</span></strong>は、  
必要がなくなれば必ず<strong><span style="color: red;">明示的に破棄しなければならない</span></strong>。
