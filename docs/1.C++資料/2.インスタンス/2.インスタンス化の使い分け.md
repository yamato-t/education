# **インスタンス化の使い分け**

---

## **二つの方法による違いをイメージする**

同じインスタンス化ではあるが、  
<span style="color: red;">二つの方法には結果に大きな違いがある</span>。

この違いをしっかりイメージ出来るようにしておかなければならない。

---

## **変数宣言による直接のインスタンス化と破棄**

### **変数自体がインスタンスになる**

変数を宣言すると、その変数自体がインスタンスになる。  
「変数 ＝ インスタンス」であり、変数のスコープが終わると  
インスタンスもそのまま自動で消える。

```c++
class Enemy {
public:
    int hp_;
    Enemy() : hp_(100) {}
};

void someFunction() {
    Enemy enemy;     // enemy という変数が実体そのもの
    enemy.hp_ = 50;
}   // ← スコープ終了。enemy（インスタンス）が自動で消える
```

<span style="color: red;">**変数とインスタンスが一体になっており**</span>  
変数が生きている間だけインスタンスは存在し、  
スコープを抜ける（変数の寿命）と自動でメモリが解放される。



---

## **new / delete を使ったインスタンス化と破棄**

### **ポインタ変数と実体は別物**

`new` を使うと、<span style="color: red;">**インスタンスはヒープ領域という別の場所に作られる**</span>。  
受け取るポインタ変数にはその<span style="color: red;">**アドレスだけ**</span>が入っている。

つまり<span style="color: red;">**変数 ≠ インスタンス**</span>であり、  
変数はインスタンスへの地図のようなものになっている。

```c++
class Enemy {
public:
    int hp_;
    Enemy() : hp_(100) {}
};

void someFunction() {
    Enemy* p = new Enemy();  // インスタンスはヒープ上に作られる
                             // p はその場所を示すポインタ変数
    p->hp_ = 50;

    delete p;    // 実体を明示的に破棄
    p = nullptr;
}   // ← p（ポインタ変数）は消えるが、
    //   delete しなければ実体はヒープに残り続ける
```

<span style="color: red;">**ポインタ変数とインスタンス自体は別物**</span>であり、  
ポインタ変数のスコープが終わってもインスタンスは消えず、  
インスタンスを消すには明示的に `delete` を呼ぶ必要がある。

## **使い分け**

### **変数宣言では使わない分も実体を持ち続ける**

たとえば最大 100 体の敵が出るゲームで、  
変数宣言でインスタンス化しようとすると下記のようになる。

```c++
void GameScene::init() {
    Enemy enemies[100];  // 最大数の数だけ、最初から全員分のインスタンスが確保される
}
```

現在画面に 3 体しかいなくても、100 体の実体が  
常にメモリを占有し続ける。  
使っていない 97 体のメモリは無駄になる。

### **new / delete なら必要な分だけ確保・解放できる**

`new` を使えば、実際に必要になったタイミングで実体を作り、  
不要になったタイミングで確実に消せる。

```c++
class GameScene {
    Enemy* enemies_[100]{};  // ポインタだけを管理する配列
    int enemyCount_ = 0;

public:
    // 敵が出現するときだけインスタンスを生成
    void spawnEnemy() {
        if (enemyCount_ >= 100) return;
        enemies_[enemyCount_++] = new Enemy();
    }

    // 敵が倒されたらインスタンスを破棄
    void killEnemy(int index) {
        delete enemies_[index];
        enemies_[index] = enemies_[--enemyCount_];
        enemies_[enemyCount_] = nullptr;
    }

    // シーン終了時に残ったインスタンスをすべて解放
    void cleanup() {
        for (int i = 0; i < enemyCount_; i++) {
            delete enemies_[i];
            enemies_[i] = nullptr;
        }
        enemyCount_ = 0;
    }
};
```

- <span style="color: red;">**メモリを節約できる**</span>
    - 「出現した敵」ぶんのインスタンスしかメモリに存在しない
    - 倒した瞬間にインスタンスを破棄でき、メモリをすぐに返せる
- <span style="color: red;">**ポインタ変数のサイズはインスタンスそのものより小さく済む**</span>
    - ポインタ変数は「アドレス」しか持たないため、  
      環境によらずほとんどの場合 8 バイト（64 bit 環境）で固定
    - 一方インスタンスはメンバ変数をすべて含むサイズになる
    - 100 要素の配列でも、ポインタ配列なら 800 バイト程度で済む
- 全体が必要なくなれば、必ず全インスタンスを解放する

同じ 100 個持っているように見えて、  
インスタンスそのものを 100 個持つのと、  
ポインタ変数を 100 個持つのとでは、  
利用メモリサイズに大きな差が出てくる。

常に存在するものではない and 数が変動する and 大量に発生するものは、  
基本 new によるインスタンス化と delete による破棄を利用するべき。

逆に、常に存在する and 数が固定 and 少ない数しか発生しないものは、  
変数宣言によるインスタンス化が向いている。

この違いをしっかり理解できるようになっていてほしい。
