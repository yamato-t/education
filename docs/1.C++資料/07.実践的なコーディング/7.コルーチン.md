# **コルーチン**

---

## **コルーチンとは**

<span style ="color: red;">**コルーチン**</span> は、  
「処理の途中で一時停止し、後から続きを再開できる関数」の仕組みを指す。  
C++20 から標準機能として追加された。

---

## **通常の関数との違い**

まず通常の関数の動きを確認する。

```c++
#include <iostream>

void NormalFunc()
{
    std::cout << "ステップ1" << std::endl;
    std::cout << "ステップ2" << std::endl;
    std::cout << "ステップ3" << std::endl;
}

int main()
{
    NormalFunc(); // ステップ1、2、3 が一気に出力される
    return 0;
}
```

通常の関数は呼び出されると、処理が全て終わるまで呼び出し元に戻ってこない。

コルーチンは異なる動きをする。  
`co_await` というキーワードを使うと、その箇所で処理を中断して  
呼び出し元に制御を返すことができる。

```c++
// Resumable はコルーチンを扱う為に自分で定義する構造体（後述）
Resumable CoroutineFunc()
{
    std::cout << "ステップ1" << std::endl;
    co_await std::suspend_always{}; // ここで中断して呼び出し元へ戻る

    std::cout << "ステップ2" << std::endl;
    co_await std::suspend_always{}; // ここで中断

    std::cout << "ステップ3" << std::endl;
}
```

呼び出し側が「続きを実行してくれ」と伝えるたびに、  
コルーチンは中断した箇所から再開してステップを一つずつ進める。


---

## **コルーチンを使う三つのキーワード**

C++20 のコルーチンは以下の三つのキーワードで動作を制御する。

| キーワード | 役割 |
|---|---|
| `co_await` | 条件が満たされるまで中断して待機する |
| `co_yield` | 値を返しながら中断する |
| `co_return` | コルーチンを終了する |

この三つのうち一つでも関数内に書くと、  
その関数はコルーチンとして扱われる。

---

## **簡易サンプル：co_await でステップ実行**

`co_await` を使う場合、比較的シンプルな補助構造体だけで動作する。  
下記の `Resumable` 構造体を用意すれば、  
手動で再開できるコルーチンを作れる。

```c++
#include <iostream>
#include <coroutine>

// 手動で再開できるコルーチン用の構造体
struct Resumable
{
    struct promise_type
    {
        Resumable get_return_object()
        {
            return Resumable{ std::coroutine_handle<promise_type>::from_promise(*this) };
        }
        std::suspend_always initial_suspend() { return {}; }
        std::suspend_never  final_suspend() noexcept { return {}; }
        void return_void() {}
        void unhandled_exception() { std::terminate(); }
    };

    std::coroutine_handle<promise_type> handle;

    explicit Resumable(std::coroutine_handle<promise_type> h) : handle(h) {}

    // コルーチンを次の中断点まで進める
    void resume()
    {
        if (!handle.done()) handle.resume();
    }
};
```

この `Resumable` を戻り値型に使って、イベントシーケンスを書いてみる。

```c++
// イベントシーケンスをコルーチンで表現する
Resumable EventSequence()
{
    std::cout << "[ステップ1] キャラクターが歩いてくる" << std::endl;
    co_await std::suspend_always{};  // ここで中断して呼び出し元に制御を返す

    std::cout << "[ステップ2] 台詞を表示する" << std::endl;
    co_await std::suspend_always{};  // ここで中断

    std::cout << "[ステップ3] シーン終了" << std::endl;
}

int main()
{
    // コルーチンを作成（まだ実行しない）
    auto seq = EventSequence();

    std::cout << "-- Aボタンを押した --" << std::endl;
    seq.resume(); // ステップ1 まで進む

    std::cout << "-- Aボタンを押した --" << std::endl;
    seq.resume(); // ステップ2 まで進む

    std::cout << "-- Aボタンを押した --" << std::endl;
    seq.resume(); // ステップ3 まで進む

    return 0;
}
```

出力結果：

```
-- Aボタンを押した --
[ステップ1] キャラクターが歩いてくる
-- Aボタンを押した --
[ステップ2] 台詞を表示する
-- Aボタンを押した --
[ステップ3] シーン終了
```

`resume()` を呼ぶたびに、中断した直後から次の `co_await` まで処理が進む。  
コールバック関数や `switch` による状態管理を使わずに、  
演出の流れを一本の関数として分かりやすく書ける点が利点となる。

## **自作ジェネレータを利用する：co_yield**

`co_await` が「中断して待つ」のに対し、  
`co_yield` は<span style ="color: red;">**値を返しながら中断する**</span>キーワードとなる。  
コルーチンから値を逐次取り出す仕組みを  
<span style ="color: red;">**ジェネレータ**</span> と呼ぶ。

`co_yield` を使う為には、`Resumable` より少し複雑な  
`Generator` 構造体が必要となる。  
中身は難しく見えるが、「値の受け渡しに必要なおまじない」として  
まずはそのまま使い方を覚えればいい。

### **Generator 構造体の用意**

```c++
#include <coroutine>
#include <optional>

// コルーチンの状態と値を管理する構造体
template<typename T>
struct Generator
{
    struct promise_type
    {
        std::optional<T> current_value;

        Generator get_return_object()
        {
            return Generator{ std::coroutine_handle<promise_type>::from_promise(*this) };
        }
        std::suspend_always initial_suspend() { return {}; }
        std::suspend_always final_suspend() noexcept { return {}; }

        std::suspend_always yield_value(T value)
        {
            current_value = value;
            return {};
        }
        void return_void() { current_value = std::nullopt; }
        void unhandled_exception() { std::terminate(); }
    };

    std::coroutine_handle<promise_type> handle;

    explicit Generator(std::coroutine_handle<promise_type> h) : handle(h) {}
    ~Generator() { if (handle) handle.destroy(); }

    // コピーを禁止する（ハンドルの二重破棄を防ぐ）
    Generator(const Generator&) = delete;
    Generator& operator=(const Generator&) = delete;

    // 次の値まで進める。値がある場合は true を返す
    bool next()
    {
        if (handle.done()) return false;
        handle.resume();
        return !handle.done();
    }

    T value() const { return handle.promise().current_value.value(); }
};
```

### **シンプルなカウンタ**

`Generator` を使って 1 から 3 までを一つずつ返すコルーチンを書く。

```c++
#include <iostream>

Generator<int> Count()
{
    co_yield 1;  // 1 を返して中断
    co_yield 2;  // 再開後、2 を返して中断
    co_yield 3;  // 再開後、3 を返して中断
}

int main()
{
    auto gen = Count();

    while (gen.next())                         // 次の値まで進める
    {
        std::cout << gen.value() << std::endl; // 値を取得して表示
    }

    return 0;
}
```

出力結果：

```
1
2
3
```

処理の流れを一つずつ確認する。

1. `Count()` を呼ぶ → コルーチンが作られるが、まだ実行しない
2. `gen.next()` → コルーチンが動き始め `co_yield 1` に到達して中断。値 1 が保存される
3. `gen.value()` → 保存されている値 1 を取得して表示
4. `gen.next()` → 中断した直後から再開し `co_yield 2` に到達して中断。値 2 が保存される
5. 以降も同様に続く

## **co_return を使う**

`co_return` はコルーチンを終了させるキーワードとなる。  
通常の `return` に相当するが、コルーチン関数内では `co_return` を使う。

```c++
Generator<int> Limited()
{
    co_yield 10;
    co_yield 20;
    co_return;  // ここでコルーチンを終了する（省略も可能）
}
```

戻り値が `void` の場合は `co_return;` と書くか省略もできる。  
関数の末尾まで到達した場合も自動的に終了となる。

## **ゲーム開発での活用場面**

### **演出・イベントシーケンス**

上記の `co_await` の例のように、  
「3秒待つ → 台詞を表示 → 次のシーンへ」という演出の流れを  
一つの関数として書ける。  
`enum` による状態変数や複数のコールバック登録が不要になる。

### **ローディング処理**

ファイルやテクスチャの読み込みなど、時間がかかる処理を待ちながら、  
メインスレッドの描画を止めずに進められる。

### **敵AIの行動パターン**

「2秒待つ → 攻撃 → 1秒待つ → 移動 → 繰り返す」といった  
時間に依存したAI行動パターンを、自然な流れのコードで書ける。
