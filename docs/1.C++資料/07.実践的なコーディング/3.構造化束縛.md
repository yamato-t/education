# **構造化束縛**

---

## **複数の変数に分解する**

配列や構造体、```std::map<T,U>``` で出てきた ```std::pair<T,U>``` といった情報を、  
取得する時点で分解してしまう事が出来る。  
これは C++ バージョン 17 以降で使える機能。

```c++
// パラメータ構造体
struct Parameter {
	std::string name	= {};	// 名前
	int			attack	= {};	// 攻撃力
	float		speed	= {};	// スピード

};

// パラメータ構造体型の変数を宣言して初期化する
Parameter p{ "hero", 10, 5.3f };

// 構造体の内容を予め分解して取得する
auto [n, a, s] = p;
std::cout << n << std::endl;	// 名前
std::cout << a << std::endl;	// 攻撃力
std::cout << s << std::endl;	// スピード
```

```auto [n, a, s]``` が該当の部分になる。  
構造体を構造化束縛した際、[]の中身は、  
構造体のメンバ変数の宣言順に代入される。  
当然、メンバ変数の数を超える宣言はできない。

また、上記の場合、```n, a, s``` は全て ```p``` の各メンバ変数を複製する形で取得している。  
その為、それぞれに値を代入しても ```p``` の内容は上書きされない。  
編集可能にするためには、参照にする必要がある。

```c++
// 参照で各要素を取得する
auto& [n, a, s] = p;
n = "aaa";  // p が持つ name の内容が変わる 
```


```std::map<T,U>``` 等の場合、先程の範囲 for 文 と合わせてこの様な記述が可能
```c++
// キーが文字列、値が整数のマップを宣言して初期化する
std::map<std::string, int> m{
  { "aaa", 15 },
  { "bbb", 9 },
  { "ccc", 72 },
  { "ddd", 64 }
};

// 構造化束縛 + 範囲 for 文で走査する
for (auto [ key, value ] : m) {
	std::cout << key << std::endl;
	std::cout << value << std::endl;
}
```
```m``` からは ```std::pair<std::string, int>``` の値が取得できる。  
```std::pair<T,U>``` は構造化束縛で分解して取得する事が可能である為、  
このような記述ができる。

上記も同じく、取得した際に各変数が複製されている。  
内容を編集したい際は参照にする必要があるが、  
```std::map``` のキーは更新できない(const変数)ので、  
```value``` のみ書き換える事が可能になっている。

## **関数の戻り値との相性**

構造化束縛は関数の戻り値で特に便利な物になっている。

```c++
std::string name	= "hero";
int			attack	= 10;

// 複数の値を戻り値に pair の形で詰める
std::pair<int, std::string> f()
{
	return { attack, name };
}

// 複数の戻り値を取得出来る
auto [a, n] = f();
std::cout << a << std::endl;
std::cout << n << std::endl;
```

この様に複数の戻り値を簡潔に受け取る事が出来る。  
また、戻り値の型を ```std::pair<int&, std::string>``` 等に変えると、  
特定の値（```attack```）だけ参照で返すといった事も可能。
