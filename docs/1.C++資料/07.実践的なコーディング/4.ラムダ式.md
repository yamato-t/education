# **ラムダ式**
---

## **関数オブジェクトの記述方法**

関数オブジェクトの話の中で、関数を変数の形で保持して利用する事が出来るとお話しした。  
**ラムダ式** という言葉が出てきた事を覚えているか？

ラムダ式というのは、関数定義方法の一つ。  
下記は以前のコードを少し変更したもの。

```c++
#include <iostream>
#include <functional>

// 関数A
char A()
{
	return 'A';
}

// エントリー関数
int main()
{
	// 関数を変数で保持する
	std::function<char()> f = A;

	// 変数から関数を呼び出す
	std::cout << "結果は " << f() << std::endl;
}
```

ここでは ```A``` という関数を変数 ```f``` に代入している。  
これをラムダ式を利用した形に変更してみる。

```c++
#include <iostream>
#include <functional>

// エントリー関数
int main()
{
	// 関数を変数で保持する
	std::function<char()> f = []() -> char { return 'A'; };

	// 変数から関数を呼び出す
	std::cout << "結果は " << f() << std::endl;
}

```

下記の記述がラムダ式と呼ばれる。

```c++
[]() -> char { return 'A'; };
```

今まで見たことがない記述だが、落ち着いて一つ一つの項目を見ると、  

* 関数名がない代わりに [] が付いている
* 戻り値型が後ろにある形の関数定義と同じ

である事が分かると思う。  
この記述をそのまま ```std::function``` 型の変数に代入すると、  
ラムダ式によって名前のない関数が生成されて変数に代入される。   

「int 型の引数を受け取って二倍した値を返す関数をラムダ式で定義する」となった場合は下記のようになる。

```c++
// 関数を変数で保持する
std::function<int(int)> f = [](int val) -> int { return val * 2; };

// 変数から関数を呼び出す
std::cout << "結果は " << f(5) << std::endl;
```

ここからさらに簡潔な物にしていくとこうなる。

```c++
// 関数を変数で保持する
auto f = [](int val) { return val * 2; };
```

さらにラムダ式の場合、 **引数にも auto が利用できる** 。

```c++
// 関数を変数で保持する
auto f = [](auto val) { return val * 2; };
```

## **キャプチャ**

ラムダ式の関数の代わりにある `[]` には  **キャプチャ**　という特殊な役割がある。  
キャプチャとは、ラムダ式の外の変数をラムダ式で利用できる様にする機能を指す。

例えば、ラムダ式内の係数を外側で宣言されている変数から取得したいとする。

```c++
// 係数をラムダ式の外で決める
auto coef = 2;

// 関数を変数で保持する
auto f = [](auto val) { return val * coef; };
```

この場合 ```coef``` でコンパイルエラーが出る。  
外側の変数を利用したい場合、[] 内に外側の変数を利用する指定を行わなければならない。  
指定の方法には幾つかある。

```c++
// コピーするパターン
auto f1 = [=](auto val) { return val * coef; };		// 必要な変数を全てコピーして利用できるようにする	
auto f2 = [coef](auto val) { return val * coef; };	// coef 変数をコピーして利用できるようにする

// 参照するパターン	
auto f3 = [&](auto val) { return val * coef; };		// 必要な変数を全て参照で利用できるようにする	
auto f4 = [&coef](auto val) { return val * coef; };	// coef 変数を参照で利用できるようにする

```

必要な変数を全部コピー or 参照、  
あるいは、特定の変数をコピー or 参照といった用途別に指定方法は変わる。

注意するべき点として、

* コピーの場合  
**別アドレスが確保されて内容が複製される事**  
別インスタンスが生成される為、  
大量のメモリを利用するクラスや初期化に時間がかかるクラスは向かない。
* 参照の場合  
**キャプチャした参照元の変数が生き続けていなければならない事**  
実際にラムダ式の関数が処理される時に、  
変数の寿命が切れていると解放済みアドレスを参照する事になる。

が挙げられる。

参照に関して補足すると、下記のような状況が挙げられる。

```c++
std::function<int(int)> f = {};
{
	auto coef = 3;
	f = [&coef](auto val) { return val * coef; };

	// coef の寿命はここで切れる
}
// ラムダ式関数はここで呼び出され、関数内で coef が利用されているが寿命は切れている
// 上手く処理できている様に見えていても、解放済みのアドレスを参照しているので何が起こるかは分からない
std::cout << "結果は " << f(5) << std::endl;
```
