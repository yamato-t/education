# **純粋仮想関数**
---

## **具体的にさせる事を強要する**

自動車は「運ぶ」という特性を持っているが、  
内容が曖昧で、運ぶ対象は「何か」としか言えない。  
逆に、自動車の特性を引き継いだ救急車の場合は、  
用途に応じた特徴が付加されている事もあり、  
**運ぶ対象は必ず具体的になっていないといけない** 。

プログラムでもこういった形にしたい場合がよくある。  
「 ```Car``` クラスを継承するなら、必ず ```carry()``` をオーバーライドして中身を具体的に実装させる」という形。  
そんな時に利用するのが<span style="color: red;">**純粋仮想関数**</span>になる。

---
## **純粋仮想関数の宣言と定義**

純粋仮想関数の宣言は、仮想関数宣言の最後に記述を追加するのみ。  
定義は必要ない（できないわけではない）。

```c++
// 自動車クラス
class Car
{
public: 
    Car() {}
    ~Car() {}

    // （何かを）運ぶ特性を純粋仮想関数として宣言する
    virtual void carry() = 0;
};
```
注目するのは下記の部分。
```c++
virtual void carry() = 0;
```
```virtual``` は同じだが最後に ```= 0;``` がついている。  
これだけで純粋仮想関数になる。

---
### **派生クラスで再定義**

```c++
// 救急車クラス
class Ambulance : public Car
{
public:
    Ambulance() {}
    ~Ambulance() {}

    // （患者を）運ぶ
    void carry() override
    {
        std::cout << "患者を運ぶ" << std::endl;
    }
};
```
オーバーライドの方法は、通常の仮想関数の時と同じ。  

---
## **純粋仮想関数に関する注意点**

純粋仮想関数を利用する場合、いくつか注意点がある。  

<span style="color: red;">**純粋仮想関数を持つクラスはインスタンス化ができなくなる**</span>。

```c++
// エントリー関数
int main()
{
    Car* car1 = new Car;   // Car はインスタンス化できない為コンパイルエラーになる
    Car car2;              // Car はインスタンス化できない為コンパイルエラーになる

    Car* car3 = new Ambulance;  // Ambulance インスタンスのアドレスをアップキャストで受け取っているだけなので問題ない

}
```

<span style="color: red;">**派生クラスで純粋仮想関数をオーバーライドしないと、インスタンス化の際にコンパイルエラーになる**</span>。


```c++
// ダンプカークラス
class Dump : public Car
{
public:
    Dump() {}
    ~Dump() {}
};

// エントリー関数
int main()
{
    Dump dump; // carry() がオーバーライドされていない為、コンパイルエラーになる
}
```

---
## **抽象クラスとインターフェイスクラス**

純粋仮想関数を持つクラスには特別な呼ばれ方がある。

C++では一般的に、  
純粋仮想関数を一つでも持つクラスを **抽象クラス**  
純粋仮想関数と仮想デストラクタしかないクラスを **インターフェイスクラス**   
と呼んでいる。
