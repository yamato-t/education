# **仮想関数**
---

## **キャスト時の問題点**

基底クラスと派生クラスの項目で**アップキャスト** について挙げたが、  
利用した際に下記の様な問題点があった。

- 派生クラスで再定義した関数が呼び出せない。

上記は **ポインタ変数の型** によって呼び出される内容が決まる事によって発生している。  
これは正しいといえば正しいが、派生クラスのインスタンスのアドレスを、  
基底クラスのポインタ型に変換（アップキャスト）できるのであれば、  
<span style="color: red;">基底クラスのポインタ変数から派生クラスで再定義した関数を呼び出したい</span>という状況も出てくるはず。

上記を解決するのが<strong><span style="color: red;">仮想関数</span></strong>になる。

---
## **仮想関数の宣言と定義**

基底クラスで関数宣言時にキーワードを追加する。

```c++
// 自動車クラス
class Car
{
public: 
    Car() {}
    ~Car() {}

    // （何かを）運ぶ
    virtual void carry();
};

// （何かを）運ぶ
void Car::carry()
{
    std::cout << "何かを運ぶ" << std::endl;
}
```

注目するのは下記の部分。
```c++
virtual void carry();
```
関数宣言時、戻り値の前に ```virtual``` という記述をつける。  
それにより、この関数を**派生クラスで再定義される前提の関数**とする事ができる。

---
## **派生クラスで再定義**

次に派生クラス側で再定義する際にもキーワードをつける。

```c++
// タンクローリークラス
class Tank : public Car
{
public:
    Tank() {}
    ~Tank() {}

    // （燃料を）運ぶ
    void carry() override;
};

//  （燃料を）運ぶ
void Tank::carry()
{
    std::cout << "燃料を運ぶ" << std::endl;
}
```
関数宣言時、最後に ```override``` という記述をつける。  
それにより、この関数が**基底クラスの仮想関数を明確に再定義した事**を示す様になる。  
```override``` は省略も可能だが、意図を明確にするのと、  
記述間違い等を回避する為にも付けておいた方がいい。

このように基底クラスの仮想関数を派生クラスで再定義する事を<span style="color: red;">**オーバーライド**</span>と呼ぶ。  
ちなみに単なる再定義は**隠蔽**と呼ばれる。

---
## **仮想関数の利用**

前の項目で正しく呼び出せなかったサンプルを実行すると、  
正しく処理されるようになっている事が確認できる。

```c++
// エントリー関数
int main()
{
    Tank* tank = new Tank;
    // Tank のアドレスを Car ポインタで受け取る
    Car* car = tank;

    // Tank の carry() が呼び出される様になった
    car->carry();

    delete tank;
}
```
```
燃料を運ぶ
```

### **派生クラスでオーバーライドしなかった場合**
基底クラスで ```virtual``` をつけて仮想関数を定義したが、  
派生クラスでオーバーライドしないという事もある。

この場合、単に基底クラスの仮想関数の処理が呼ばれるだけになる。  
コンパイルエラーにはならない。

---
## **仮想デストラクタ**
**アップキャスト** の件でもう一つ、  
<span style="color: red;">**delete すると派生クラスのデストラクタが呼ばれない**</span>という問題があった。

この問題も**基底クラスのデストラクタを仮想関数化**する事で解決できる。

```C++
// 自動車クラス
class Car
{
・・・省略
    virtual ~Car() {
        std::cout << "Car のデストラクタが処理されました" << std::endl;
    }
・・・省略 
};

// タンクローリークラス
class Tank : public Car
{
・・・省略
    ~Tank() {
        std::cout << "Tank のデストラクタが処理されました" << std::endl;
    }
・・・省略 
};

```
```
Car のコンストラクタが処理されました
Tank のコンストラクタが処理されました
Tank のデストラクタが処理されました
Car のデストラクタが処理されました
```

通常の仮想関数と違い、<span style="color: red;">**デストラクタは派生クラスも基底クラスも両方処理される**</span>。  
実際、派生クラスのデストラクタだけが呼ばれてしまうと、  
インスタンスを破棄した際の後始末が正しく行えない可能性がある為、  
両方処理される事は正しい挙動といえる。
