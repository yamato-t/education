# **名前空間**

---

クラス実装を進める上で、クラス整理に役立つものを合わせて紹介しておく。  

<span style="color: red;">**名前空間**</span>というものを利用する事で、  
クラスを更にグループ分けし、整理した形にしていく事が出来る。

なおクラスだけではなく、  
グローバル変数やグローバル関数をまとめる用途にも利用できる。  

---

## **名前空間の定義**
名前空間の最低限の定義は下記の通り。

```c++

// 名前空間「nameA」の定義
namespace nameA
{
}

```

一つではなく、複数を入れ子の形で定義する事も可能。

```c++
// 名前空間「nameA」の定義
namespace nameA
{
	// 名前空間「type1」の定義
	namespace type1
	{
	}
}
```

また C++ バージョンによっては、  
入れ子の際に下記の様な記述も出来る。

```c++
// 名前空間「nameA」「type1」の定義
namespace nameA::type1
{
}
```

---

## **名前空間の利用**

下記には「MyClass」クラスと「function」関数が複数定義されており、  
それぞれが違う名前空間に属している。

```c++

// 名前空間「nameA」の定義
namespace nameA
{
	// クラス「MyClass」の定義
	class MyClass
	{
	public:
		int value_ = 0;
	};

	// 関数「function」の定義
	void function() {
		std::cout << "nameA::function" << std::endl;
	}
}

// 名前空間「nameB」の定義
namespace nameB
{
	// クラス「MyClass」の定義
	class MyClass
	{
	public:
		int value_ = 0;
	};

	// 関数「function」の定義
	void function() {
		std::cout << "nameB::function" << std::endl;
	}
}
```

通常、同名のクラスや同名の関数が定義されるとコンパイルエラーとなるが、  
<span style="color: red;">それぞれ別の名前空間に属している為、コンパイルエラーとはならない</span>。

その代わり、<span style="color: red;">**クラスインスタンス作成や関数呼び出しで名前空間を指定する必要がある**</span>。


```c++

// 「nameA」の MyClass のインスタンスを作る
nameA::MyClass classA;
// 「nameB」の MyClass のインスタンスを作る
nameB::MyClass classB;

// 「nameA」の function を呼び出す
nameA::function();
// 「nameB」の function を呼び出す
nameB::function();

```


---

## **名前空間の注意点**

名前空間の指定は`using`キーワードで省略する事が出来る。  
下記は「nameA」の指定を省略している。


```c++

// 名前空間「nameA」を利用する事を明記する
// 以降、スコープ内で「nameA::」を指定する必要が無くなる
using namespace nameA;

// 「nameA」の MyClass のオブジェクトを作る
MyClass classA;
// 「nameB」の MyClass のオブジェクトを作る
nameB::MyClass classB;

// 「nameA」の function を呼び出す
function();
// 「nameB」の function を呼び出す
nameB::function();

```

ただし、この省略は基本的に避けた方が無難。  
理由は「`using namespace nameB;`」を追加するとわかる。

同名のクラスや関数が各名前空間に所属しているので、両方省略しようとすると、  
どの名前空間に所属しているクラスなのか判断できなくなる為、コンパイルエラーになる。

---

## **無名空間**

名前を省略した名前空間を<span style ="color: red;">**無名空間**</span>と呼ぶ。

```c++
namespace  // 名前を省略（無名空間）
{
	void function()
	{
		std::cout << "function" << std::endl;
	}
}
```

無名空間の中に定義した関数・変数は、  
<span style ="color: red;">**定義があるファイル内部でのみ有効**</span>になる。  
他のファイルから参照する事はできない。

この挙動は関数や変数に `static` を付けた場合と同等になる。

<span style ="color: red;">**無名空間は基本的に CPP ファイル内での利用が一般的**</span>。  
外部に公開しない内部のヘルパー関数などを  
まとめる目的で使われる事が多い。

```c++
// cpp 内での利用
namespace
{
	// このファイル内でしか使わない関数として定義
	int helperCalc(int x)
	{
		return x * 2;
	}
}
```

### **その他**

ヘッダで関数を定義し複数の .cpp でインクルードすると、  
通常は多重定義エラーが発生するが、無名空間で囲む事で回避できる。  
ただし<span style ="color: red;">**この回避は inline キーワードでの回避とは別物**</span>である為、  
区別して認識しておいてほしい。

また、ヘッダの内容を無名空間で囲む事はそもそも推奨されない方法になるので、  
<span style ="color: red;">**無名空間は CPP でしか使わない**</span> という認識で利用してもらった方が良い。
