# **メンバ関数**

---

各クラスは、それぞれが動作を持つ事が出来る。  
クラスが行う振る舞い（動作）は<span style ="color: red;">**メンバ関数**</span>として用意する

---

## **メンバ関数の宣言と定義**

メンバ関数の宣言方法は下記の通り。  
クラス内に通常の関数と同じように宣言する。  

```c++
// クラス「MyClass」の定義
class MyClass
{
	void functionA(); // メンバ関数「functionA」を宣言する

	int valueA = 0;
};
```

関数は定義も必要になる。  
下記の様にクラスの外に定義する。

```c++

// メンバ関数「functionA」を定義する
void MyClass::functionA()
{
	// メンバ変数「valueA」の値を更新する、という振る舞いをする
	valueA = 1;
}

```

これで「MyClass」クラスは、「valueA」というデータを更新する、  
「functionA」という機能を持った事になった。


## **メンバ関数を宣言と同時に定義する**

関数宣言時に一緒に定義する事も可能。  
その場合、クラス内の宣言が定義に置き換わる。  
クラスの定義をヘッダで行っていた場合、  
C の関数定義は多重定義エラーが発生するが、  
クラスのメンバ関数の場合は多重定義と扱われず発生しない。

但し、宣言と定義を分けて記述している場合は、
C の関数定義と同様に多重定義が発生する。

```c++
// myclass.h の内容

// クラス「MyClass」の定義
class MyClass
{
	// クラス内でメンバ関数を宣言と同時に定義する
	// 関数の定義を h に用意した場合でも多重定義は起きない
	void functionA()
	{
		// メンバ変数「valueA」の値を更新する、という振る舞いをする
		valueA = 1;
	}

	// メンバ関数を宣言のみ行い定義は行わない
	void NG();

	int valueA = 0;
};

// h でメンバ関数を続けて定義すると多重定義が発生
void MyClass::NG()
{
}

```

クラス内でメンバ関数を「宣言と同時に定義する」場合だけ多重定義が回避できるが、  
これは<span style ="color: red;">**その記述でのみ、暗黙的にインライン展開の対象になる**</span>おかげ。


### **インライン展開**

メンバ関数を宣言と同時に定義した場合、  
コンパイラは自動的に<span style ="color: red;">**インライン展開**</span>の対象として扱う。

インライン展開とは、関数呼び出しの箇所に関数の処理を直接埋め込む形への変換を指す。

通常の関数呼び出しでは次の手順が発生する。

1. 呼び出し元の状態を保存する
2. 関数の場所へジャンプする
3. 処理を実行する
4. 元の場所に戻る

インライン展開が行われると、これら手順がなくなる。  
処理の少ない関数を頻繁に呼び出す場合に効果的な最適化となる。  
ただし対象になっても実際に展開するかどうかはコンパイラが判断する為、  
必ず展開されるわけではない。

関数をインライン展開の対象にしたい場合は `inline` キーワードを付ける。

```c++
// inline キーワードで展開の対象にする
inline void MyClass::functionA()
{
	valueA = 1;
}

// 通常の関数でも対象にできる
inline void globalFunction()
{
	// ・・・・
}

```

`inline` キーワードには<span style ="color: red;">**多重定義を許可する意味合いもある**</span>。

「クラス内で宣言と同時に定義されたメンバ関数」は、  
<span style ="color: red;">**暗黙的にインライン展開の対象になる**</span>事で `inline` キーワード付きと同じ解釈が行われる。  
その為**キーワードをわざわざ付けなくても多重定義にはならない**。


## **メンバ関数の呼び出し**

メンバ関数を呼び出したい場合、  
メンバ変数同様に**基本的**にクラスのインスタンスが作られている必要がある。  

```c++
// 「オブジェクト.メンバ関数」で呼び出せる
MyClass myclass;
myclass.functionA();

// クラス型のポインタの場合、
// 「ポインタ->メンバ関数」で呼び出せる
MyClass* myclassPtr = &myclass;
myclassPtr->functionA();
```

## **静的メンバ関数**

静的メンバ変数と同じように、静的メンバ関数という物もある。  
クラスのインスタンスを作らずにメンバ関数を利用する事が出来る。


```c++
// クラス「MyClass」の定義
class MyClass
{
	static void functionB(); // 静的メンバ関数「functionB」を宣言する
};


// 静的メンバ関数「functionB」を定義する
void MyClass::functionB()
{
	std::cout << "Call functionB" << std::endl;
}
```

利用する際は、下記の様な形になる。  

```c++
// 「クラス型::メンバ関数」で呼び出せる
MyClass::functionB();
```

注意するべき点として <span style ="color: red;">**静的メンバ関数では通常のメンバ変数を利用する事が出来ない**</span>。  
関数は静的な物、つまりクラスからインスタンスを作らなくても呼び出せる為、  
インスタンスを作らないと利用できない通常のメンバ変数は利用できない。  
逆に、インスタンスを作らなくても利用できる<span style ="color: red;">**静的メンバ変数であれば利用できる**</span>。

### **const メンバ関数**

C++ ではクラスのメンバ関数に `const` をつける事が出来る。  
const メンバ関数にした場合、**その関数内でメンバ変数を変更する事が出来なくなる**。

```c++
// クラス「MyClass」の定義
class MyClass
{
public:
	// const メンバ関数「functionA」の定義
	void functionA(int arg) const {
		// メンバ変数「value_」を更新する
		value_ = 10; // const メンバ関数内でメンバ変数を変更しているのでコンパイルエラーになる

		// 引数「arg」を変更する
		arg = 0;    // こちらは問題ない
	}

	// メンバ関数「functionB」の定義
	void functionB() {
		value_ = 10; // 問題ない
	}

public:
	int value_ = 0; // メンバ変数「value」を宣言
};

```

さらにクラスのインスタンスに `const` が付いた場合、  
<span style ="color: red;">**const メンバ関数しか呼び出せなくなる**</span>。


```c++
// const インスタンスを作成する
const MyClass constMyclass;

constMyclass.functionA(1); // 問題ない
constMyclass.functionB();  // const メンバ関数ではない為コンパイルエラー
```

メンバ関数に const を付ける事で、  
そのメンバ関数によってメンバ変数が更新されない事を保障する事が出来る。  
<span style ="color: red;">**使う側が作った側の意図を理解しやすく、各メンバ関数を安全に使う事ができる**</span>。
