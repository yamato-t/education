# **クラス**

---

## **プログラム言語「C++」**
 
C言語に <span style="color: red;">**オブジェクト指向的拡張やジェネリクス的拡張**</span>を加えた言語。  
開発規模の拡大に伴い、プロジェクトに携わる人員が多くなっても、  
開発しやすい言語として登場した。

メモリなどのコンピュータの低レベルな部分を柔軟に扱う事ができる為、  
高パフォーマンスな半面、問題のあるコードを埋め込みやすいなどの特徴がある。

C++を最初に学ぶ際、<span style="color: red;">**クラス**</span>を介して**オブジェクト指向**から入る場合が多い。  
実際それが C からの特に大きな変化であり、C++ における基本的な概念となっている。


---

## **オブジェクト指向**

ここでいう**オブジェクト**とは、  
<span style="color: red;">**すべての『物』**</span>と捉えてもらえるといい。

**オブジェクト指向**とは、  
<span style="color: red;">**一つ一つを「物」として捉え、それらを組み合わせて大きな「物」を作る**</span>。  

C++において大抵の場合、最初に学ぶのは**クラス**になるが、  
この**クラス**が C++ をオブジェクト指向を実現しやすい言語として成り立たせている。  

クラスには「パラメータとその制御」が纏められている。  
クラスをざっくりいうと <span style="color: red;">設計図</span>と捉えられる。  

クラスを元に作られたオブジェクトは<span style="color: red;">**インスタンス**</span>と呼ばれる。  

オブジェクトとインスタンスは混合されがちだが少し違う。  
「インスタンスはオブジェクト」と言えるが、  
「オブジェクトはインスタンス」とは言えない。

C++では、クラスを元に作られたインスタンスをはじめ、  
様々なオブジェクトを組み合わせる事で、  
アプリケーション全体の動作を組み立てていく。

---

## **オブジェクト指向プログラミングの三大要素** ###

オブジェクト指向に沿ったプログラムを作成する上で、  
クラス設計（作り方を考える）時に重視されている基礎的な概念がある。  
下記三つの言葉は覚えていてもらいたい。

① <span style="color: red;">**カプセル化**</span>  
② <span style="color: red;">**継承**</span>  
③ <span style="color: red;">**多態性**</span>

最初はまずクラスを通じて**カプセル化**のイメージをつかめる事を目標として頂くのが良い。

---

## **クラスを利用する**

C++ でオブジェクト指向を実現しやすくするための仕組み。  
扱うデータやパラメータとその制御を纏める形で宣言、定義し、  
必要に応じてオブジェクトを作成（インスタンス化）し利用する。  
Cから学んできた場合、よく「構造体とほぼ同じ物」という紹介をされる。  

以降、クラスから作られたオブジェクトは**インスタンス**と呼ぶ。

---

## **クラスの定義**
クラスを定義する最低限のコードは以下の通り。  

```c++
// クラス「MyClass」の定義
class MyClass
{
};
```

Cにおける**構造体**の定義と似ており、  
先頭が **class** というキーワードに変わるだけで、  
クラスが定義できると思ってもらっていい。

上記は何も持たず、何の機能もない、  
空の`MyClass`クラスを定義している。

---
## **定義したクラスの利用**

クラスを利用する為には、基本的にインスタンスを作る必要がある。  
以下の様に<span style="color: red;">クラス型の変数を宣言する事でインスタンス化する</span>。  
こちらも構造体型の変数を宣言する形と同じ。

```c++
// 「MyClass」を元にインスタンスを作る
MyClass myclass;
```

---

## **構造体とクラスの違い**

C++において、構造体にも様々な機能が追加された。  
結果、構造体とクラスの違いは**ほぼ無いと言って等しい**状態になっている。  
とは言え全く無い訳では無い。
