# **メンバ**

---

各クラスは、それぞれが取り扱うデータやパラメータ、動作を持つ事が出来る。  
それらは総称して <span style="color: red;">**クラスのメンバ**</span> と呼ばれる。

メンバには下記二種類が存在する。

- クラスが持つデータやパラメータを指す<span style="color: red;">**メンバ変数**</span>
- クラスが行う振る舞い（動作）を指す<span style="color: red;">**メンバ関数**</span>

---

## **メンバ変数**

### **メンバ変数の宣言**

メンバ変数の宣言方法は下記の通り。

```c++
// クラス「MyClass」の定義
class MyClass
{
	int valueA = 0; // メンバ変数「valueA」を宣言し、ゼロで初期化する
};
```
こちらも構造体内の変数宣言と同じ記述になる。

これで「MyClass」クラスは、  
「valueA」というデータを持つ事になった。

### **宣言したメンバ変数の利用**

メンバ変数を利用したい場合、  
こちらも**基本的**にクラスのインスタンスが作られている必要がある。  

```c++
// 「オブジェクト.メンバ変数」で利用できる
MyClass myclass;
myclass.valueA = 1;

// クラス型のポインタの場合、
// 「ポインタ->メンバ変数」で利用できる
MyClass* myclassPtr = &myclass;
myclassPtr->valueA = 2;
```

### **静的メンバ変数**

特殊な例として、静的なメンバ変数がある。  
クラスのインスタンスを作らずにメンバ変数を利用する事が出来る。

注意する点として、通常のメンバ変数宣言とは違い、  
<span style="color: red;">クラスの外側でも再宣言が必要になる</span>。  
※ただし、C++バージョンによっては特定のキーワードを付ける事で再宣言を省略可能。

```c++
// クラス「MyClass」の定義
class MyClass
{
	static int valueB; // 静的メンバ変数「valueB」を宣言する（初期化はしない）
};

// クラスの定義外で再宣言する（ここで初期化する）
int MyClass::valueB = 0;

```

利用する際は、下記の様な形になる。  

```c++
// 「クラス型::静的メンバ変数」 で利用する
MyClass::valueB = 0; 

```
なお、通常のメンバ変数と同じようにインスタンスから利用する事も可能。


---

## **メンバ関数**

### **メンバ関数の宣言と定義**

メンバ関数の宣言方法は下記の通り。  
クラス内に通常の関数と同じように宣言する。  

```c++
// クラス「MyClass」の定義
class MyClass
{
	void functionA(); // メンバ関数「functionA」を宣言する

	int valueA = 0;
};
```

関数は定義も必要になる。  
下記の様にクラスの外に定義する。

```c++

// メンバ関数「functionA」を定義する
void MyClass::functionA()
{
	// メンバ変数「valueA」の値を更新する、という振る舞いをする
	valueA = 1;
}

```

なお宣言時に定義する事も可能。  
その場合、クラス内の宣言が定義に置き換わる。


```c++

// クラス「MyClass」の定義
class MyClass
{
	// メンバ関数「functionA」を定義する
	void functionA()
	{
		// メンバ変数「valueA」の値を更新する、という振る舞いをする
		valueA = 1;
	}

	int valueA = 0;
};

```

これで「MyClass」クラスは、「valueA」というデータを更新する、  
「functionA」という機能を持った事になった。


### **宣言、定義したメンバ関数の利用**

メンバ関数を利用したい場合、  
やはり**基本的**にクラスのインスタンスが作られている必要がある。  

```c++
// 「オブジェクト.メンバ関数」で呼び出せる
MyClass myclass;
myclass.functionA();

// クラス型のポインタの場合、
// 「ポインタ->メンバ関数」で呼び出せる
MyClass* myclassPtr = &myclass;
myclassPtr->functionA();
```

### **静的メンバ関数**

静的メンバ変数と同じように、静的メンバ関数という物もある。  
クラスのインスタンスを作らずにメンバ関数を利用する事が出来る。


```c++
// クラス「MyClass」の定義
class MyClass
{
	static void functionB(); // 静的メンバ関数「functionB」を宣言する
};


// 静的メンバ関数「functionB」を定義する
void MyClass::functionB()
{
	std::cout << "Call functionB" << std::endl;
}

```

利用する際は、下記の様な形になる。  

```c++
// 「クラス型::メンバ関数」で呼び出せる
MyClass::functionB();

```

注意するべき点として <span style="color: red;">**静的メンバ関数では通常のメンバ変数を利用する事が出来ない**</span>。  
関数は静的な物、つまりクラスからインスタンスを作らなくても呼び出せる為、  
インスタンスを作らないと利用できない通常のメンバ変数は利用できない。  
逆に、インスタンスを作らなくても利用できる<span style="color: red;">**静的メンバ変数であれば利用できる**</span>。

### **const メンバ関数**

C++ ではクラスのメンバ関数に `const` をつける事が出来る。  
const メンバ関数にした場合、**その関数内でメンバ変数を変更する事が出来なくなる**。

```c++
// クラス「MyClass」の定義
class MyClass
{
public:
	// const メンバ関数「functionA」の定義
	void functionA(int arg) const {
		// メンバ変数「value_」を更新する
		value_ = 10; // const メンバ関数内でメンバ変数を変更しているのでコンパイルエラーになる

		// 引数「arg」を変更する
		arg = 0;    // こちらは問題ない
	}

	// メンバ関数「functionB」の定義
	void functionB() {
		value_ = 10; // 問題ない
	}

public:
	int value_ = 0; // メンバ変数「value」を宣言
};

```

さらにクラスのインスタンスに `const` が付いた場合、  
<span style="color: red;">**const メンバ関数しか呼び出せなくなる**</span>。


```c++
// const インスタンスを作成する
const MyClass constMyclass;

constMyclass.functionA(1); // 問題ない
constMyclass.functionB();  // const メンバ関数ではない為コンパイルエラー
```

メンバ関数に const を付ける事で、  
そのメンバ関数によってメンバ変数が更新されない事を保障する事が出来る。  
<span style="color: red;">**使う側が作った側の意図を理解しやすく、各メンバ関数を安全に使う事ができる**</span>。
