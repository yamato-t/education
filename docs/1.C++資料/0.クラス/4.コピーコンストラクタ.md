# **コピーコンストラクタ**

---
コンストラクタには<span style="color: red;">**コピーコンストラクタ、ムーブコンストラクタ**</span>といった、  
通常とは違う記述で呼び出される**特殊なコンストラクタ**がある。

ここでは、上記のうちの<span style="color: red;">**コピーコンストラクタ**</span>に焦点をあてる。  

---

## **コピーコンストラクタの宣言と定義**

コピーコンストラクタも特殊な**関数**なので、  
関数のように宣言、定義する。  

```c++
// 動物クラス
class Animal final
{
public:
	// コピーコンストラクタの宣言
	Animal(const Animal& src);

public:
	std::string		name_		= {};
	int				footNum_	= 0;
};

// コピーコンストラクタの宣言
Animal::Animal(const Animal& src)
{
	std::cout << "コピーコンストラクタが呼ばれました" << std::endl;

	name_ = src.name_;
	footNum_ = src.footNum_;

}
```

コピーコンストラクタは上記の様に、  
**同型のconst参照を引数で受け取るコンストラクタ**を指す。  

---
## **コピーコンストラクタの呼び出し方**

コピーコンストラクタは下記の状況で呼び出される。  
- **インスタンスの宣言時にイコールで別インスタンスを代入した場合**  
- **同型のconst参照を引数で渡す形で宣言した場合**

```c++
// 引数付きコンストラクタが呼ばれる
Animal animal1("アニマル", 2);

// 宣言と同時に代入するとコピーコンストラクタが呼ばれる
Animal animal2 = animal1;
// 同型のインスタンスを引数で渡す場合もコピーコンストラクタが呼ばれる
//Animal animal2(animal1);

// この場合はコンストラクタもコピーコンストラクタも呼び出されない（後述）
animal1 = animal2;
```
上記の例だと、中央の行だけコピーコンストラクタが呼び出される。  

---
## **コピーコンストラクタの存在理由**

一見すると単に複製するだけなので、  
何故わざわざコピーコンストラクタといった形で、  
独自に用意できるのか分かりにくいかもしれないが、  
<span style="color: red;">**メンバ変数でポインタ変数を持っている場合などで**</span>、  
コピーコンストラクタの存在が重要になってくる。

---
## **コピーコンストラクタは省略可能**

コピーコンストラクタを省略した場合も、コンパイルエラーにはならない。  
省略した場合、メンバ変数の内容を全て複製する処理が自動で定義される。  
が、**複製されるメンバ変数が何なのかを良く理解しておく必要がある**。  

特定のクラスの複製を禁止する為に、  
コピーコンストラクタが処理されるようなコードを書くと、  
コンパイルエラーになる様に実装する事も珍しくない。
