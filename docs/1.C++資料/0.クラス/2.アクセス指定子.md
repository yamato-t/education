# **アクセス指定子**

---

オブジェクト指向の三大要素として<span style="color: red;">**カプセル化**</span>という要素がある。  
アクセス指定子はそのカプセル化の基本となる仕組み。  
クラスが持つメンバ（データや振る舞い）の<span style="color: red;">**公開範囲**</span>を指定する。  
また C++ では**構造体にもアクセス指定子を記述する事が出来る**。

---

## **アクセス指定子の種類**

|名称|説明|
|----|----|
|public| 完全に公開され、何処でも利用できる |
|private| 非公開とし、自身のクラス内でのみ利用できる |
|protected| 非公開とし、自身と、自身を継承したクラス内でのみ利用できる |

クラスが持つメンバ変数やメンバ関数、その他の定義は、  
全て上記の何れかに属している。

---
## **アクセス指定子の利用方法**

各メンバの宣言や定義前に、アクセス指定子を記述する事によって、  
それぞれの公開範囲を指定する事が出来る。  
次のアクセス指定子が記述されるまで公開範囲は更新されない。

```c++
// クラス「MyClass」の定義
class MyClass
{
public: // アクセス指定子の記述により、ここから
	void functionA();
	void functionB();
	int valueA = 0;
        // ここまで公開されている

private: // 新しいアクセス指定子の記述により、ここから
	int valueB = 0;
         // ここまで非公開となり、自クラス内でのみ利用出来る
};
```

自クラスのメンバ関数内（クラス内）であれば、  
アクセス指定子関係なく、全て利用する事が出来る。  


```c++

// 公開メンバ関数「functionA」を定義する
void MyClass::functionA()
{
	// valueAは公開されているので、クラス内外関係なく利用できる
	valueA = 1;
	std::cout << valueA << std::endl;
}

// 公開メンバ関数「functionB」を定義する
void MyClass::functionB()
{
	// valueBは非公開だが、メンバ関数内（自クラス内）なので利用できる
	valueB = 2;
	std::cout << valueB << std::endl;
}

```


<span style="color: red;">各メンバをクラス外で利用する際はアクセス指定子による公開設定の影響を受ける</span>。



```c++
// エントリー関数
int main()
{
	MyClass myclass;

	myclass.functionA(); // ← 問題ない
	myclass.functionB(); // ← 問題ない
	myclass.valueA = 10; // ← 問題ない

	myclass.valueB = 20; // ← クラス外で非公開のメンバを利用している為、コンパイルエラーになる

	return 0;
}
```

---
## **アクセス指定子を記述しない場合**
アクセス指定子は記述を省略出来る。  
省略した場合はデフォルトの指定が行われるが、  
クラスは<span style="color: red;">**private扱い**</span>に、  
構造体は<span style="color: red;">**public扱い**</span>になる。


---
## **アクセス指定子の重要性**

クラスの設計者がアクセス指定子を正しく記述する事で、  
「利用していい物」と「利用されたくない物」の分別が付きやすくなる。  
つまり<span style="color: red;">**他人が間違った使い方をしにくくなる**</span>事に繋がる。  

クラスの持つデータや振る舞いの役割をハッキリさせる上でも、  
アクセス指定子は意識して指定してほしい。
