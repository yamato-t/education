# **nullptr**

---

初期化時等、無効なアドレスを持つポインタ変数に指定する `NULL` というキーワードがある。  
`NULL` は、C言語において「0」という値を持っており、  
それを「無効なアドレス」と把握しやすいように `NULL` を利用している。

ただ、プログラムの広義において **「0」と「NULL」は全く違う物として認識しないといけない**。  

- 「0」　 ・・・ ゼロという値が存在する
- 「NULL」・・・ **存在がない**

C++ ではこの違いを明確にする為 ```nullptr``` というキーワードが追加された。

---

## **NULL を利用する事による問題**

下記のコードを確認してほしい。  
実行するとどの様な結果になるか。

```c++
#include <iostream>

// 引数がポインタの関数
void func(int* p) {
	std::cout << "引数がポインタ" << std::endl;
}

// 引数がポインタではない関数
void func(int p) {
	std::cout << "引数がポインタではない" << std::endl;
}

// メイン関数
int main()
{
	func(0);    // 「引数がポインタではない」と表示される
	func(NULL); // 「引数がポインタ」と表示される？

	return 0;
}
```

関数オーバーロードを利用して、二つ関数を用意した。  
片方はポインタ変数を引数に持つ為、  
```NULL``` を渡せば適切に呼び出されると考えるかもしれないが、  
そうはならない。

なぜなら **NULL は「0」である為**。  
つまり、`NULL` を渡すと int 型の引数の関数が呼び出される。

この場合、両方とも「引数はポインタではない」と表示される。

---

## **nullptr を利用**

この様な問題を回避する際、```nullptr``` が利用できる。

```c++
// メイン関数
int main()
{
	func(0);       // 「引数がポインタではない」と表示される
	func(nullptr); // 「引数がポインタ」と表示される

	return 0;
}
```

`nullptr` は「無効なアドレスを持つポインタ」である事**明確にする**キーワードとなる。  
`NULL` とは違い、曖昧な扱い方をされない物となる。  

### **nullptr 利用のメリット**

`nullptr` が入っているポインタ変数を `delete` で指定した場合は、  
破棄する必要が無いと判断され、何も行われない。

```c++
{
	Animal* pAnimal = new Animal;
	pAnimal->name_ = "動物";
	pAnimal->footNum_ = 4;

	// まず破棄する
	delete pAnimal;
	// 直後にもう一度同じポインタ変数に対して delete するとクラッシュする
	//delete pAnimal;

	// すでに破棄（解放）済みである為、nullptr でポインタ変数を初期化する
	pAnimal = nullptr;

	// nullptr で初期化されているので、問題は起こらない
	delete pAnimal;
}
```
