# **継承**

---

## **「継承する」という言葉の意味**
以前**あるオブジェクトの持つ機能を引き継いだ新しいオブジェクトをつくる**という説明をした。  
その言葉についてもう少し理解を深くする。

### **拡張する**

一旦プログラムは忘れ、まずは下記の共通点を考えて貰いたい。

- タクシー
- バス
- 救急車
- タンクローリー
- 消防車
- ダンプカー

これらが全て**自動車**である事は理解出来るかと思う。  
つまり全て**タイヤがあり、エンジンによって動く、何かを運ぶ為に地上を移動する機械**を指している。

上記で挙げた物は全て**自動車**の特性を基本として持っており、  
そこに<span style="color: red; ">**用途に応じた特徴を付加した物**</span>と言える。

- タクシー　　　：少人数の運搬に特化している
- バス　　　　　：大人数の運搬に特化している
- 救急車　　　　：医療設備を搭載した救急運搬に特化している
- タンクローリー：燃料の輸送に特化している
- 消防車　　　　：火災や災害対策の設備の輸送に特化している
- ダンプカー　　：土砂や砕石の輸送に特化している

例えば**タンクローリー**であれば、  
**自動車に「燃料を輸送する」機能拡張した物**である事は理解できると思う。  

### **具体的にする**

**自動車**と**タンクローリー**は、それぞれ「**何かを運ぶ**」機械だが、  
ここで「**運ぶ物**」に注目してみる。

- 自動車　　　　：何かを運ぶ
- タンクローリー：液体・気体といった燃料を運ぶ

上記は一般的に常識であり、「タンクローリーが運んでる」という事を聞いて、  
チョコレートを連想する人はまぁまずいないはず。

つまり「**運ぶ**」という言葉は同じだが、  
**タンクローリー**は**自動車**に比べて、  
<span style ="color: red;">**内容がより具体的になっている**</span>事がわかるかと思う。

---

上記で挙げた様な事をプログラムで実現する際、  
今回のテーマである <span style ="color: red;">**継承**</span> が用いられる。

<span style="color: red; ">**継承の利用はプログラム設計に特に大きな影響を与える**</span>為、  
どの様に利用するかをしっかり検討する必要がある。
