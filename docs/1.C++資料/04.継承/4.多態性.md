# **多態性**
---

## **仮想関数や純粋仮想関数を使う理由**

端的に言えば<span style="color: red;">**多態性（ポリモーフィズム）**</span>を実現する為。  
**多態性** とは<span style="color: red;">****同じ呼び出しでも、オブジェクトの型によって適切な処理が行われる性質**</span> をさす。  


---
## **多態性がないコード**


```c++
#include <iostream>

 コンストラクタ、デストラクタは省略

// 自動車クラス
class Car
{
public:
    void carCarry() {
        std::cout << "何かを運ぶ" << std::endl;
    }
};

// タンクローリークラス
class Tank : public Car
{
public:
    void tankCarry() {
        std::cout << "燃料を運ぶ" << std::endl;
    }
};

// ダンプカークラス
class Dump : public Car
{
public:
    void dumpCarry() {
        std::cout << "土砂を運ぶ" << std::endl;
    }
};

// エントリー関数
int main()
{
    Car*  car = new Car;
    Tank* tank = new Tank;
    Dump* dump = new Dump;

    car->carCarry();
    tank->tankCarry();
    dump->dumpCarry();

    delete car;
    delete tank;
    delete dump;
}
```

それぞれ「運ぶ」機能を持っているが、それぞれが独自の関数名で宣言、定義されている為、  
利用する側は **型と型における機能の関数名** を把握しておく必要がある。

## **少し改善**

```c++
#include <iostream>

 コンストラクタ、デストラクタは省略

// 自動車クラス
class Car
{
public:
    void carry() {
        std::cout << "何かを運ぶ" << std::endl;
    }
};

// タンクローリークラス
class Tank : public Car
{
public:
    void carry() {
        std::cout << "燃料を運ぶ" << std::endl;
    }
};

// ダンプカークラス
class Dump : public Car
{
public:
    void carry() {
        std::cout << "土砂を運ぶ" << std::endl;
    }
};

// エントリー関数
int main()
{
    Car*  car = new Car;
    Tank* tank = new Tank;
    Dump* dump = new Dump;

    car->carry();
    tank->carry();
    dump->carry();

    delete car;
    delete tank;
    delete dump;
}

```

関数をそれぞれのクラスで再定義し、利用する際に同名で呼び出せるようになった。  
これでもよさそうな印象があるかもしれないが、**呼び出しているポインタ変数のクラス型に依存してしまっている** 。

```c++
Car*  car = new Tank;
car->carry();
```
こうすると ```Car``` の ```carry()``` が呼び出される。
その為、これでも十分ではない。

---

## **さらに改善**
```c++
#include <iostream>

 コンストラクタ、デストラクタは省略

// 自動車クラス
class Car
{
public:
    virtual void carry() {
        std::cout << "何かを運ぶ" << std::endl;
    }
};

// タンクローリークラス
class Tank : public Car
{
public:
    void carry() override {
        std::cout << "燃料を運ぶ" << std::endl;
    }
};

// ダンプカークラス
class Dump : public Car
{
public:
    void carry() override {
        std::cout << "土砂を運ぶ" << std::endl;
    }
};

// エントリー関数
int main()
{
    Car*  car = new Car;
    Tank* tank = new Tank;
    Dump* dump = new Dump;

    car->carry();
    tank->carry();
    dump->carry();

    delete car;
    delete tank;
    delete dump;
}

```
```virtual``` と ```override``` をつけた。   
これによって、ポインタ変数の型ではなく、  
**実際のインスタンスの型で関数が呼び分けられるようになった**。 
```c++
Car*  car = new Tank;
car->carry();
```
上記も ```Tank``` の ```carry()``` が呼び出される。  
これにより、**一応多態性は実現できている** 。

## **多態性をうまく使う**

```c++
#include <iostream>

 コンストラクタ、デストラクタは省略

// 自動車クラス
class Car
{
public:
    virtual void carry() {
        std::cout << "何かを運ぶ" << std::endl;
    }
};

// タンクローリークラス
class Tank : public Car
{
public:
    void carry() override {
        std::cout << "燃料を運ぶ" << std::endl;
    }
};

// ダンプカークラス
class Dump : public Car
{
public:
    void carry() override {
        std::cout << "土砂を運ぶ" << std::endl;
    }
};

// エントリー関数
int main()
{
    Car* cars[] = { new Car, new Tank, new Dump };

    for (int i = 0; i < 3; ++i)
    {
        cars[i]->carry();
    }

    for (int i = 0; i < 3; ++i)
    {
        delete cars[i];
        cars[i] = nullptr;
    }
}

```
```main()```の内容が大きく変わっている。  
まず、インスタンス化の際に全て ```Car``` ポインタの配列に格納している。  
その後、forループで同じ関数を呼び出している。  
仮想関数をオーバーライドしている為、  
**インスタンスの型に合わせて carry() が呼び出されている** 。
