# **スレッドとは**

---

## **プログラムの実行単位**

プログラムを実行すると、OS はそのプログラムの為に  
<span style ="color: red;">**プロセス**</span> と呼ばれる実行環境を用意する。  
プロセスにはメモリ空間やファイルハンドルなど、  
プログラムが動くために必要なリソースが割り当てられる。

そのプロセスの中で、実際に命令を順番に実行していく流れを  
<span style ="color: red;">**スレッド**</span> と呼ぶ。  
通常のプログラムはスレッドを一本だけ持ち、  
命令を先頭から順番に一つずつ実行していく。  
これを <span style ="color: red;">**シングルスレッド**</span> という。

---

## **マルチスレッドとは**

一つのプロセスの中に複数のスレッドを持たせることを  
<span style ="color: red;">**マルチスレッド**</span> という。  
複数のスレッドはそれぞれ独立して命令を実行できる為、  
複数の処理を同時並行で進める事ができる。

現代のCPUはコアを複数持っており、  
コアが複数ある場合はスレッドを物理的に同時実行できる。  
これを <span style ="color: red;">**並列処理**</span> という。

コアが足りない場合でも、OS がスレッドを短い時間単位で切り替えながら  
実行することで擬似的に同時に動いているように見せる。  
これを <span style ="color: red;">**並行処理**</span> という。

---

## **マルチスレッドのメリット**

### **処理速度の向上**

重い計算処理を複数のスレッドに分割して実行することで、  
シングルスレッドより速く処理を完了できる場合がある。

例えば大量の画像を変換する際、  
シングルスレッドでは一枚ずつ順番に処理するしかない。  
マルチスレッドにすれば複数枚を同時に処理できる。

### **応答性の維持**

重い処理をメインスレッドとは別のスレッドで行うことで、  
メイン側の処理を止めずに済む。

ゲームでいえば、ファイルの読み込み中でも  
ロード画面のアニメーションが動き続けるのはこのおかげ。

---

## **マルチスレッドの危険性**

マルチスレッドは便利な反面、特有の問題を引き起こす。

### **データ競合**

複数のスレッドが同じメモリ領域に対して、  
同時に読み書きを行おうとした場合に問題が起きる。  
これを <span style ="color: red;">**データ競合**</span> という。

データ競合が起きると、予測不能な値の破壊やプログラムのクラッシュが発生する。  
また、環境によって再現したりしなかったりする為、  
原因の特定が非常に難しいバグになりやすい。

### **デッドロック**

複数のスレッドが互いに相手の処理の完了を待ち続け、  
どちらも先に進めなくなる状態を  
<span style ="color: red;">**デッドロック**</span> という。

例えば、スレッドAがリソース1を確保した後にリソース2を要求し、  
スレッドBがリソース2を確保した後にリソース1を要求した場合、  
互いに相手のリソースを永遠に待ち続ける事になる。

### **デバッグの困難さ**

マルチスレッドのバグは、スレッドの実行順序が毎回変わる為、  
再現条件が安定しない事が多い。  
デバッガで止めた瞬間に問題が消えてしまうケースもある。  
シングルスレッドのバグより遥かに発見・修正が難しい。


マルチスレッドは強力な機能だが、正しく扱わなければ  
深刻なバグを生む。  
メリットと危険性を理解した上で使う事が重要となる。
