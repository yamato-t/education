# **delete による破棄**

単に変数を宣言するのではなく、`new` によるインスタンス化の方法を話したが、  
`new` で確保したメモリ領域は<span style="color: red;">**変数宣言時とは違い、自動で解放されない**</span>。 

その為 `new` で確保したあらゆるオブジェクトは、  
<span style="color: red;">**対になる専用の方法で明示的に破棄しなければならない**</span>。

---

## **delete キーワード**

必要なくなったインスタンスを明示的に破棄するキーワード。  
これによって、インスタンスが利用していたメモリ領域を解放する事が出来る。

`new` の項目で「明示的に破棄する必要がある」と述べたが、  
`delete` はそれを実現する。

---

## **delete によるインスタンス破棄**

`new` でインスタンスが利用するメモリ領域を確保した場合に限り、  
`delete` でそのメモリ領域を解放する事が出来る。

```c++
// 動物クラス
class Animal final
{
public:
        std::string	name_		= {}; // 名前
        int		footNum_	= 0;  // 足の数
};

// エントリー関数
int main()
{
	// delete によるインスタンス破棄
	{
		// new によるインスタンス化
		Animal* pAnimal = new Animal;
		pAnimal->name_ = "動物";
		pAnimal->footNum_ = 4;

		// delete の後に破棄したいインスタンスのポインタ変数を指定する
		// new によって確保したメモリ領域を明示的に解放する
		delete pAnimal;
	}
}
```

なお、通常の変数宣言でインスタンス化した場合は、  
スコープを抜けた際に自動的に破棄される為、  
`delete` してはいけないという認識でいてもらえればいい。

---
## **デストラクタの呼び出し**

デストラクタは「**インスタンスが破棄される際に処理される物**」なので、  
`delete` によるクラスインスタンス破棄の際も呼び出される

```c++

// インスタンス化と破棄
{
	Animal* pAnimal = new Animal;

	delete pAnimal;	// new によるインスタンスが破棄されるのは delete した時、
		            // つまりここで pAnimal が指すメモリ領域にあるインスタンスのデストラクタが呼ばれる
	pAnimal = nullptr;

	std::cout << "スコープを抜ける" << std::endl;
}
```


---
## **複数を一度に破棄**

`new` で配列の形で複数を一度にインスタンス化した場合、  
`delete` も複数を一度に破棄する記述にしなければならない。

```c++
// 複数を一度にインスタンス化した場合の delete
{
	Animal* pAnimals = new Animal[3];
		
	for (int i = 0; i < 3; ++i)
	{
		pAnimals[i].name_ = "動物";
		pAnimals[i].footNum_ = i;
	}

	// delete の後ろに[]を付ける事で、配列で new した場合に正しく破棄できる
	delete[] pAnimals;
}
```

---
## **delete を利用する場合の注意点**

`delete` で指定するアドレスが指すメモリ領域は「破棄していいメモリ領域」である必要がある。  
分かりやすい例だと `delete` した直後に同じアドレスに対してもう一度 `delete` するとクラッシュする。  

一度 `delete` した事で、そのメモリ領域は「未使用」になる。  
何らかのインスタンスによって同じ領域が利用されるといった事がない限り、  
再度 `delete` してはならない。

その為 `delete` した後に指定したポインタ変数を `NULL` で初期化しておくのが良い。


```c++
{
	Animal* pAnimal = new Animal;
	pAnimal->name_ = "動物";
	pAnimal->footNum_ = 4;

	// まず破棄する
	//delete pAnimal;
	// もう一度同じポインタ変数に対して delete するとクラッシュする
	//delete pAnimal;

	// NULL じゃない時だけ delete する
	if(pAnimal != NULL)
	{
		delete pAnimal;
		// 破棄したので NULL でポインタ変数を初期化する
		pAnimal = NULL;
	}
}

```

---
## **借りた物は正しく返す**

<strong><span style="color: red;">new した物は必ず delete しなければならない</span></strong>。  
これによって、new の項目で挙げたメモリリークが回避できる。

パソコンや各ゲームハードで利用できるメモリ総数には必ず限界がある。  
メモリリークは、利用できるメモリを徐々に少なくし、  
最後にはゲームが正しく動かなくなるといった問題に繋がる為、気を付ける必要がある。

なお、そんなメモリに関するトラブルを少しでも回避する為、  
C++では**スマートポインタ**と呼ばれる仕組みが提供されている為、ぜひ使ってほしい。
