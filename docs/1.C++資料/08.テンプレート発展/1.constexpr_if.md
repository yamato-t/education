# **constexpr if**

## **テンプレート実引数の型による分岐**

テンプレートを作成し利用していると、  
**実引数の型に合わせて処理を分岐させたい場合が出てくる**。  

以前紹介した際、そのような場合は特殊化などを行って対応するとお話しした。  
例えば下記の様なコード

```c++
#include <iostream>

// 昨今の C++ に合わせた文字列の扱い方を利用する
#include <string_view>

// 引数 value がゼロかどうかをチェックする関数
template <class T>
std::string_view checkZero(T value) {
    if (value != 0) {
        return "ゼロではありません";
    }

    return "ゼロです";
}

// 引数 の型が float の場合の特殊化関数
template <>
std::string_view checkZero(float value) {
    if (value != 0) {

	    // 0.x だった場合は特殊な判定を行う
        if (0 < value && value < 1.0f) {
            return "小数部はゼロではありませんが、整数部はゼロです";
        }

        return "ゼロではありません";
    }

    return "ゼロです";
}

// エントリー関数
int main() {
    int   i  = 5;
    char  c  = 0;
    float f1 = 0.3f;
    float f2 = 1.5f;
    float f3 = 0.f;

    // 各変数がゼロかどうかをチェックする
    std::cout << "i は" << checkZero(i) << std::endl;
    std::cout << "c は" << checkZero(c) << std::endl;
    std::cout << "f1 は" << checkZero(f1) << std::endl;
    std::cout << "f2 は" << checkZero(f2) << std::endl;
    std::cout << "f3 は" << checkZero(f3) << std::endl;

    return 0;
}

```

上記でも問題はないが、昨今の C++ バージョンでは  
**テンプレート関数の中で特殊な分岐を行わせる方法で対応出来る**。

上記のコードを昨今の記述に変化させると下記のようになる。

```c++

#include <iostream>

// 昨今の C++ に合わせた文字列の扱い方を利用する
#include <string_view>

// テンプレート内で T の型をチェックする為に必要になる
#include <type_traits>

// 引数 value がゼロかどうかをチェックする関数
template <class T>
std::string_view checkZero(T value) {
    if (value != 0) {
        // 引数 value(T) の型が float 型だった場合の特殊判定を追加する
        if constexpr (std::is_same_v<decltype(value), float>) {

            // 0.x だった場合は特殊な判定を行う
            if (0 < value && value < 1.0f) {
                return "小数部はゼロではありませんが、整数部はゼロです";
            }
        }

        return "ゼロではありません";
    }

    return "ゼロです";
}

// エントリー関数
int main() {
    int   i  = 5;
    char  c  = 0;
    float f1 = 0.3f;
    float f2 = 1.5f;
    float f3 = 0.f;

    // 各変数がゼロかどうかをチェックする
    std::cout << "i は" << checkZero(i) << std::endl;
    std::cout << "c は" << checkZero(c) << std::endl;
    std::cout << "f1 は" << checkZero(f1) << std::endl;
    std::cout << "f2 は" << checkZero(f2) << std::endl;
    std::cout << "f3 は" << checkZero(f3) << std::endl;

    return 0;
}

```

見比べて見ると下記の大きな違いがある。

* float 用の特殊化関数が削除されている
* テンプレート関数内に `if constexpr (std::is_same_v<T, float>)` の分岐が増えている

型が何か？を一つのテンプレート関数の中で判定してしまうことで、  
一つの関数で型に依存した個別の処理を行う事が出来る。

## **constexpr if 文**

テンプレート関数の中で、引数の型による分岐を行っている点に関して、  
`std::is_same_v` という機能を利用して型を比べているだけになるので、  
特に難しい事は無いと思う。  
問題なのは if の後ろについている `constexpr` になる。

`constexpr` には、「コンパイル時の定数化」という意味があり、  
様々な状況で利用されるので、是非自分でも調べて見てほしい。

今回は if 文と絡んだ場合の説明を行う。  
実は上記のコードは `constexpr` を削除しても動作する。

通常の if 文は、条件を満たす分岐の処理も、満たさない分岐の処理も全て**有効な処理として存在している**。  
だからこそプログラムが実際に動いている時に、変数の内容を判定して処理を分岐させる事が出来る。

ところが `constexpr` のキーワードを if につけると、  
**if 文の条件をチェックし、条件を満たさない場合はその分岐の処理をコンパイル時に無効とする**
という働きをする。

さらに、テンプレート関数内の if 文にこのキーワードを付ける事によって、  
**テンプレート関数が実引数に合わせて実体化する時に、条件を満たさない部分を実体化させずに無効とする**  
という事が可能になる。

今回の例で端的に述べると、テンプレート関数が実体化する際に  

* `if constexpr (std::is_same_v<T, float>)`  の場合  
 T が float 型で実体化しない場合 if 文の中身の処理が**存在しなくなる**
* `if (std::is_same_v<T, float>)`  の場合  
 T が float 型で実体化しない場合**でも** if 文の中身の処理が**存在している**

という違いがある。

この「存在しなくなる」「存在している」の区別が後ほど重要になってくる。
