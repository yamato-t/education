# **std::optional**

---

## **std::optional とは**

std::optional は C++17 で導入された、<strong>値が存在するかしないか</strong> を表現できるテンプレートクラス。  
従来はポインタや特殊な値（-1 や nullptr など）で「値がない」状態を表現していたが、  
std::optional を使うことでより安全で明示的なコードが書ける。  
ヘッダファイル `<optional>` をインクルードすることで使用可能になる。

### **基本的な使い方**

std::optional は「空の状態」を取ることができ、  
has_value() や value() などの関数で状態を確認できる。

```c++
#include <optional>
#include <iostream>

std::optional<int> getValue(bool flag) {
    if (flag) {
        return 42;  // 値を持つ optional を返す
    }
    return std::nullopt;  // 空の optional を返す
}

int main() {
    auto result = getValue(true);
    if (result.has_value()) {
        std::cout << "値: " << result.value() << std::endl;
    } else {
        std::cout << "値なし" << std::endl;
    }
}
```

---

## **std::optional の主なメンバ関数**

| メンバ関数 | 説明 | 例 |
|-----------|------|-----|
| has_value() | 値を持つか確認（bool を返す） | `if (opt.has_value())` |
| value() | 値を取得（値がない場合は例外） | `int x = opt.value();` |
| value_or(default) | 値を取得（値がない場合はデフォルト値） | `int x = opt.value_or(0);` |
| reset() | 値を破棄して空にする | `opt.reset();` |

### **安全な値の取得方法**

<span style="color: red;">**value() を直接呼ぶと値がない場合に std::bad_optional_access 例外が投げられる**</span>ため、  
必ず has_value() で確認するか、value_or() を使用する。

```c++
std::optional<int> opt = std::nullopt;

// 推奨される方法
int safe_value = opt.value_or(0);  // 値がなければ 0 を返す

// bool 変換を使った方法
if (opt) {
    int value = *opt;  // * 演算子でも値を取得できる
}
```

## **使用例**

std::optional は関数の戻り値やエラーハンドリングで特に有用。

### **検索関数での利用**

配列やコンテナから要素を検索する際、見つからない場合を明示的に表現できる。

```c++
#include <optional>
#include <vector>

std::optional<int> findElement(const std::vector<int>& vec, int target) {
    for (int i = 0; i < vec.size(); ++i) {
        if (vec[i] == target) {
            return i;  // インデックスを返す
        }
    }
    return std::nullopt;  // 見つからなかった
}

int main() {
    std::vector<int> numbers = {10, 20, 30, 40};
    auto index = findElement(numbers, 30);
    
    if (index) {
        std::cout << "見つかった位置: " << *index << std::endl;
    } else {
        std::cout << "要素が見つかりません" << std::endl;
    }
}
```

### **設定値の表現**

オプション的な設定項目を表す際にも有効。

```c++
// 通信接続に関するコンフィグ情報
struct Config {
    std::optional<int> port;      // ポート番号（未設定も可）
    std::optional<std::string> host;  // ホスト名（未設定も可）
};

// 接続関数
void connect(const Config& config) {
    
    // // 設定されていればその値を、未設定であればデフォルト値 (8080) を利用する
    int port = config.port.value_or(8080);

    // 設定されていればその値を、未設定であればデフォルト値 (localhost) を利用する
    std::string host = config.host.value_or("localhost");  

    // 接続処理...
}
```

## **optional の利点**

これまでは nullptr や特定の値を使って「無効な状態かどうか」を判別する事が多かったと思う。  

ただ実装が進むにつれ「null = 無効ではない扱い」になったり、  
「無効を表す特定値がいつの間にか別用途で使われる様になった」という事も起こりえる。

もしそうなった場合、これまでの実装と辻褄が合わなくなり、  
どうやっても大きな変更や不具合の発生に繋がってしまう。  

<strong>optional</strong> の利用は、そういった自体を未然に防ぐのに大変有用。   
<span style="color: red;">明確な無効値</span> を用意する事によって、  
<strong><span style="color: red;">無効を表すと思っていた値が、無効の意味ではなくなっていた</span></strong>という状況を起こりえなくする事が出来る。
