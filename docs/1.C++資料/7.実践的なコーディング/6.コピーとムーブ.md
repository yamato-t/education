# **ムーブとコピー**
---

各オブジェクトを別のオブジェクトに代入する事を当たり前の様に行っていると思う。
```c++
auto xx;
auto yy = xx;

auto zz;
auto yy = zz;
xx = yy;
```

ある入れ物の値を、違う入れ物に入れる、という事をするのだが、  
厳密に言うと代入には<span style="color: red;">**ムーブ or コピー**</span>の二つの方法が存在している。

---

## **左辺値と右辺値**

ムーブやコピーの前に、  
幾つか把握しておかなければならない事がある。  

変数の宣言と初期化を行う際に下記の様に記述すると思う。

```c++
// int 型の変数を宣言する
int value = 100;
```

実は ``` = ``` を挟んだ左右の記述にはそれぞれ呼び方がある。  

* ```value```　→　 <span style="color: red;">**左辺値**</span>  
* ```100```　　→　 <span style="color: red;">**右辺値**</span>  

と呼ばれる。

左辺値は「 **実体が存在している名前を持つオブジェクト** 」  
右辺値は「 **その一行（式）のみで有効になっている一時的なオブジェクト** 」  
を指す。

この左辺値や右辺値の把握は   
<span style="color: red;">**処理負荷を軽減する意味で**</span>とても重要な意味を持つ。


それぞれの違いを例に挙げると
```c++
// int 型の変数を宣言する
int value;

value = 0;    // コンパイルエラーにならない. value は 0 を代入出来る
100 = 0;      // コンパイルエラーになる. 100 は 0 を代入出来ない
```
**参照を返す以外の** 関数の戻り値も右辺値になる。  
参照を返す関数の戻り値は左辺値になる。

```c++
// int 型の値を返す関数
int func1() {
    return 1;
};

// int 型の値を返す関数
int value = 0;
int& func2() {
    return value;
};

func1() = 10; // コンパイルエラーになる. 関数の戻り値は右辺値
func2() = 10; // コンパイルエラーにならない. 関数の戻り値は参照である為左辺値
```

下記のどれが左辺値で、どれが右辺値かわかるか
```c++
int value1;
value1 = 0;

int value2 = value1 + 1;
value2 = 20;

int value3 = value2;
```
この場合、右辺値は「0」「value + 1」「20」の三つ  
左辺値は必ず左側にあるとは限らない。

## **左辺値参照と右辺値参照**

左辺値と右辺値は、その参照も区別されている。  
変数に「&」をつける事で参照を取得出来、  
そのアドレスの内容を書き換えられる事が出来る事は以前お話しした。

「実体の参照」という事は、つまり「左辺値」を取得する為の参照になる。  
この参照型を<span style="color: red;">**左辺値参照**</span>と呼ぶ。

```c++
// int 型の変数を宣言する
int value = 0;

// 左辺値の参照を取得する
int& ref = value;
```

左辺値参照と明確に区別できるように、右辺値用の参照型もある。  
この参照型を<span style="color: red;">**右辺値参照**</span>と呼ぶ。  
これは C++ バージョン 11 で追加された。

```c++
// 右辺値の参照を取得する
int&& ref = 0; 
```

本来、右辺値はその式の中でのみ有効な物だが、  
右辺値参照で受け取ると寿命を伸ばす事ができる。

```c++
// 右辺値の参照を取得する
int&& ref = 0; 

// 右辺値参照で寿命が伸びているので上書き出来る
ref = 1;
```

右辺値参照は「右辺値の参照」である為、  
左辺値の参照を取得できない。

```c++
int value = 0;
int&& ref = value; // コンパイルエラー. 左辺値を右辺値参照型で受け取れない
```

逆に左辺値参照は右辺値の参照を受け取る事は出来ない。  
ただし、const 左辺値参照 の場合は右辺値でも受け取れる。
```c++
// 右辺値の参照を取得する
int& ref        = 10; // コンパイルエラー. 右辺値を左辺値参照型で受け取れない
const int& ref  = 10; // エラーにならない

// const 左辺値参照で寿命が伸びているので利用できる
int value = ref; 
```

## **ムーブ**

右辺値や右辺値参照が何の為にあるか、だが、  
<span style="color: red;">**無駄な一時オブジェクトの生成を回避する為**</span>というのが大きい。


まず下記のプログラムを確認してもらいたい。
```c++
#include <iostream>

// 整数型のデータを扱うクラス
class MyClass {
public:
	// コンストラクタ
	MyClass() = default;

	// 引数付きコンストラクタ
	MyClass(int size) {
		data_ = new int[size];
		dataSize_ = size;
	}

	// コピーコンストラクタ
	MyClass(const MyClass& src) {

		if (data_) {
			delete[] data_;
		}

		// インスタンスを別に再生成
		data_		= new int[src.dataSize_];
		dataSize_	= src.dataSize_;

		// 内容を全部設定する
		for (auto i = 0; i < dataSize_; ++i) {
			data_[i] = src.data_[i];
		}

        std::cout << "コピーコンストラクタ" << std::endl; 
	}

	// デストラクタ
	~MyClass() {
		if (data_) {
			delete[] data_;
		}
		data_ = {};
	}

private:
	int*	data_		= {}; // データポインタ
	int		dataSize_	= {}; // データサイズ
};


// エントリー関数
int main()
{
	MyClass src(1000);
	// temp に src の内容を移す
	MyClass temp = src;

	// 以降 temp は色々操作するが src は一切利用しない
    {
        // ループ処理等
    }

	return 0;
}

```

このプログラムを動かすと、まず ```src``` のインスタンス化処理が行われる。  
次に ```src``` を ```temp``` を代入しているが、ここでコピーコンストラクタが処理される。  
つまりインスタンスの **複製（コピー）** が行われるコードになっている。  

複製である為、ここでは新しいインスタンスへ代入される度に、

* int 型 1000 個分のメモリが毎回確保される 
* int 型 1000 個分のメモリの内容が毎回初期化される  

という事が起こる。  

ただ、ここで重要なのは ```temp に移した後 src は一切利用しない``` という点。  
それを踏まえると、複製する挙動では効率が悪いのがわかるか。  
<span style="color: red;">**メモリ確保や初期化に時間がかかるのに、それらを代入の度に処理する為**</span>。 

ではどう改善する？という事を考えると、  
```src``` で生成した内容をそのまま ```temp``` に渡すのが一番効率的。  
要するにこんな事をすればいい。
```c++
MyClass src(1000);
MyClass temp;

// private 変数なのでエラーになるが、
// src が持っているデータのアドレスとサイズのみを代入するのが一番早い
temp.data_      =  src.data_;
temp.dataSize_  =  src.dataSize_;
```

ただしこの方法には色々問題が出る。  
**誰がそのメモリ領域の所有権を持っているのかも分かりにくくなり、  
同じアドレスの二重解放も多発しやすくなる**。  
実際このコードは ```src``` も ```temp``` も同じアドレスを指すポインタを持っており、  
デストラクタの際に両方 delete しているので問題が起こる。

その為<span style="color: red;">**値を所有権ごと他のオブジェクトに移行する**</span>という事が必要になる。  
このような動作を **ムーブ** と呼ぶ。

何かを何かに代入する、という動作には、  
「コピー」と「ムーブ」の二種類があるという事を覚えておいてほしい。

---
##  **ムーブの特徴**

所有権ごと値を移行するムーブだが、コピーとは大きく違うある特徴がある。  
それは<span style="color: red;">**ムーブ後のオブジェクトが持つ値は無効な物になっているべき**</span>という事。

ムーブされた後も同じオブジェクトを使えてしまうと、  
前述の多重解放といった問題が起こってしまう為、  
無効な物になっていなければならない。  
それを踏まえてコードを変更すると下記の様になる

```c++
MyClass src(1000);
MyClass temp;

// private 変数なのでエラーになるが、
// src が持っているデータのアドレスとサイズのみを代入するのが一番早い
temp.data_      =  src.data_;
temp.dataSize_  =  src.dataSize_;

// src の内容を temp に移行したので、src の内容を無効な物にする
src.data_       = nullptr;
src.dataSize_   = 0;
```
ここまでやってようやくムーブが成立する。

## **ムーブと右辺値参照**

**無駄なメモリ確保や処理負荷を発生させない**という意味で、  
ムーブは非常に有用である事がイメージできたと思う。

ただ、ムーブは端的に言ってとても分かりにくい。  

コードを一見するだけでは「ムーブしている」という事が把握しにくく混乱しやすい。  
実装する側も気を使って実装しなければならない。

そういう経緯から、ムーブを少しでも簡潔に扱えるようにする為に、  
明確に**ムーブに利用できる型を示す必要がある**。  
そこで登場するのが **右辺値参照** になる。  

右辺値は「 **その一行（式）のみで有効になっている一時的なオブジェクト** 」である為、  
その参照である右辺値参照とムーブの挙動は、意味的にマッチしている。  

右辺値を利用したムーブを行うコードは下記の通り。

```c++
#include <iostream>

// 整数型のデータを扱うクラス
class MyClass {
public:
	// コンストラクタ
	MyClass() = default;

	// 引数付きコンストラクタ
	MyClass(int size) {
		data_ = new int[size];
		dataSize_ = size;
	}

	// コピーコンストラクタ
	MyClass(const MyClass& src) {

		if (data_) {
			delete[] data_;
		}

		// インスタンスを別に再生成
		data_		= new int[src.dataSize_];
		dataSize_	= src.dataSize_;

		// 内容を全部設定する
		for (auto i = 0; i < dataSize_; ++i) {
			data_[i] = src.data_[i];
		}

		std::cout << "コピーコンストラクタ" << std::endl;
	}

	// ムーブコンストラクタ
	MyClass(MyClass&& src) noexcept {

		if (data_) {
			delete[] data_;
		}

		// 再生成ではなく、src のdata_が指すアドレスをそのまま取得する
		data_	  = src.data_;
		dataSize_ = src.dataSize_;

		// src が破棄された際にデストラクタで delete されると問題なので初期化しておく
		src.data_ = {};
		src.dataSize_ = {};

        std::cout << "ムーブコンストラクタ" << std::endl; 
	}


	// デストラクタ
	~MyClass() {
		if (data_) {
			delete[] data_;
		}
		data_ = {};
	}

private:
	int*	data_		= {}; // データポインタ
	int		dataSize_	= {}; // データサイズ
};


// エントリー関数
int main()
{

	MyClass src(1000);
	
	// src をムーブ可能な形（右辺値）にキャストして temp に代入する
	// これによって、コピーコンストラクタではなくムーブコンストラクタが処理される
	MyClass temp = std::move(src);

	// 以降 src は data_ が nullptr になっている為利用できない
	{
		// ループ処理等
	}

	return 0;
}
```

まず、 **引数が右辺値参照になっている**<span style="color: red;">**ムーブコンストラクタ**</span>というコンストラクタが増えている。  
```std::move()``` という関数があるが、これは「左辺値を右辺値にキャストする」という事を行う。  
<span style="color: red;">**実際にムーブしている訳ではない。ただキャストしているだけ**</span>。

そして、変換した物をそのまま ```temp``` に代入している。  
これによって、今までコピーコンストラクタだったのが、ムーブコンストラクタで処理されるようになる。

ムーブコンストラクタでは、新しく int 型配列のメモリを確保しているわけではなく、  
```src``` が持っているデータのアドレスを取得している。  
これによって ```src``` で生成したメモリ領域を、所有権ごと ```temp``` で引き継ぐ事になる。  

さらに ```src``` が持っているポインタを初期化して所有権を破棄し、  
```delete```が処理されないようにしておく。

代入の際に右辺値にキャストし、実際に右辺値に対応する処理に流れてくる事から、  
明確にムーブしようとしている事がここから見て取れる。

ムーブはコンストラクタだけではなく、代入演算子のオーバーロードにも用意されている。

```c++
// コピー代入演算子
MyClass& operator=(const MyClass& src) {
	if (data_) {
		delete[] data_;
	}

	// インスタンスを別に再生成
	data_ = new int[src.dataSize_];
	dataSize_ = src.dataSize_;

	// 内容を全部設定する
	for (auto i = 0; i < dataSize_; ++i) {
		data_[i] = src.data_[i];
	}

	std::cout << "コピー代入演算子" << std::endl;
	return *this;
}

// ムーブ代入演算子
MyClass& operator=(MyClass&& src) noexcept {
	if (data_) {
		delete[] data_;
	}

	// 再生成ではなく、src のdata_が指すアドレスをそのまま取得する
	data_ = src.data_;
	dataSize_ = src.dataSize_;

	// src が破棄された際にデストラクタで delete されると問題なので初期化しておく
	src.data_ = {};
	src.dataSize_ = {};
	
    std::cout << "ムーブ代入演算子" << std::endl;
	return *this;
}
```

ムーブ代入演算子を利用した際は、下記のような main 関数で挙動を確認できる。
```c++
// エントリー関数
int main()
{
	MyClass temp1;
	{
		MyClass src(1000);
        MyClass temp2;

		// src を temp2 にコピーする
        temp2 = src;

		// src を temp1 にムーブする
		temp1 = std::move(src);        

		// 以降 src は 寿命が切れる
	}

	{
		// ループ処理等で temp1 が利用される
	}
	return 0;
}

```

## **stlコンテナ や スマートポインタのムーブ** 

以前スマートポインタの話をした際に「譲渡」と「共有」に関して挙げたのを覚えているか？

```unique_ptr``` は唯一の所有権を持ち、  
「共有」はできないが「譲渡」はできると紹介した。  
この**所有権の譲渡でムーブが利用される**。

```c++
// メモリを確保し、所有権を得る
std::unique_ptr<int> ptr = std::make_unique<int>();
*ptr = 100;

std::unique_ptr<int> temp1 = ptr;	            // コンパイルエラー	
std::unique_ptr<int> temp2 = std::move(ptr);	// コンパイルエラーにはならない. temp2 に ptr の内容と所有権が譲渡され、これ以降 ptr は利用できなくなる

```

stl のコンテナ等でもムーブ出来る。    
<span style="color: red;">下記の様な大量の要素を持つコンテナの場合は効果が非常に大きい</span>。

```c++
std::vector<int> v(10000);
// コピー 　新しく 4 x 10000 バイトのメモリ確保が行われてしまう
std::vector<int> temp1 = v;
// ムーブ 　メモリは増えない
std::vector<int> temp2 = std::move(v);
```

## **右辺値と左辺値の区別**

右辺値と左辺値の区別に関して、気を付けなければならない事がある。  
下記の様に変更してみる。

```c++
// 右辺値に変換したものを一旦別の変数に入れる
auto rRef = std::move(src);
// ムーブ？
MyClass temp = rRef;
```
これはうまくいかない。  
というのも、1 行目ですでにムーブコンストラクタが呼ばれてしまい、  
```rRef``` が実体を持つインスタンスになる為、  
2 行目でコピーコンストラクタが処理されてしまう事になる。

1 行目でコンストラクタが呼ばれるのは、参照の形で受け取れていないから。  
そこれ ```MyClass&& ``` に型に変更する。

```c++
// 右辺値に変換したものを右辺値参照で受け取る
MyClass&& rRef = std::move(src);
// ムーブ？
MyClass temp = rRef;
```

実はこれでも問題がある。  
1 行目は確かに右辺値参照で受け取れているのだが、  
右辺値参照を受け取る変数 ```rRef``` は、実は **左辺値** になっている。  
その為、2 行目は変わらずコピーコンストラクタが処理される。  

結局もう一度右辺値に変更する必要がある。  
代入時に ```std::move``` すれば解決する。
```c++
// 右辺値に変換したものを参照で受け取る
MyClass&& rRef = std::move(src);
// ムーブ出来る
MyClass temp = std::move(rRef);
```

## **ユニバーサル参照**

このコードで ```MyClass&&``` と書かれている部分を ```auto&&``` に変更する

```c++
// 右辺値に変換したものを参照の型で受け取る
auto&& rRef = std::move(src);
```

ここで、 ```rRef``` に代入する物を左辺値にしてみる
```c++
// 左辺値をそのまま代入する
auto&& rRef = src;
```

右辺値参照型で左辺値を受け取る事は出来ないと前述したが、  
この場合コンパイルエラーは発生しない。

実は<span style="color: red;">**auto や テンプレート仮引数(T等) に && を付けた時は特殊な解釈が行われる**</span>。  
<span style="color: red;">**右辺値が渡された時は右辺値参照、左辺値が渡された時は左辺値参照に変化する**</span>。 

```c++
// この場合は ref が右辺値参照になる
auto&& ref = std::move(src);

// この場合は ref が左辺値参照になる
auto&& ref = src;
```

```auto&&``` や ```T&&``` といった参照は<span style="color: red;">**ユニバーサル参照**</span>と呼ばれる。

## **完全転送** 

ユニバーサル参照の事を踏まえて、下記のような実装を考える。

```c++
// MyClass を保持するクラス
class MyClassHolder {
public:
	// ユニバーサル参照 t を受け取り、
	// 型が右辺値参照の場合はムーブ、左辺値参照の場合はコピーする関数
	template<class T>
	void set(T&& t)
	{
		instance = t;
	}

public:
	MyClass instance;
};

// エントリー関数
int main()
{
	MyClass src(1000);

	MyClassHolder temp1;
    // コピーされるはず
	temp1.set(src);

	MyClassHolder temp2;
	// ムーブされるはず
	temp2.set(std::move(src));
}
```

```MyClassHolder::set``` 関数はテンプレート関数でユニバーサル参照を受け取っており、  
インスタンスに代入している。  

これによって、関数の引数に

* 左辺値が渡されていればコピー代入演算子が処理される
* 右辺値が渡されていればムーブ代入演算子が処理される

という事が期待できる。
ところが結果は両方コピーされてしまう。

何故かというと、```MyClassHolder::set``` に渡されてきた引数は、  
<span style="color: red;">**右辺値参照型であろうと、左辺値参照型であろうと、引数自体は左辺値になる為**</span>。  
何かの変数に入った時点で**必ず左辺値になってしまう**。

だからと言って、``` instance = std::move(t); ``` とすると、  
今度はコピーが行えなくなる。

その為 ```t``` が  

* 左辺値参照なら左辺値として instance に代入される
* 右辺値参照なら右辺値として instance に代入される

事が実現できればいい。

これを実現する方法として<span style="color: red;">**完全転送**</span>という仕組みがある。  
完全転送とは、<span style="color: red;">**値だけでなく、参照の型自体もそのまま他の処理に引き渡す働き**</span>を指す。

方法は簡単で ``` std::forward() ``` を利用するのみ。


```c++
// ユニバーサル参照 t を受け取り、
// 型が右辺値参照の場合はムーブ、左辺値参照の場合はコピーする関数
template<class T>
void set(T&& t)
{
	// ユニバーサル参照 T の型情報に合わせて代入する
	instance = std::forward<T>(t);
}
```
こうする事で、右辺値参照であればムーブ、  
左辺値参照であればコピーと振る舞いを変える事が出来る。

## **ユニバーサル参照に関する余談**

ユニバーサル参照は範囲 for 文で利用されている事も多い。  
例えば下記のコード。


```c++
std::vector<bool> v{ true, false, true, false };
for (auto& b : v) {
	b = true;
}
```

これはコンパイルエラーになる。  
```b``` に右辺値を入れようとしている為。

```std::vector<bool>``` は特殊な仕様になっており、  
要素にアクセスする際、参照する為の別オブジェクトを一時的に生成する。  
つまり右辺値になっている為、上記の様に左辺値参照型では取得できない。

そういった経緯からユニバーサル参照で受け取るコードになっている例も多い。

```c++
std::vector<bool> v{ true, false, true, false };
for (auto&& b : v) {
	b = true;
}
```

コンテナの内容を書き換える必要がない場合は ```const auto& ```  
コンテナの内容を書き換える場合は ```auto& または auto&&```  
といった形を覚えておくのがいいかと思う。
