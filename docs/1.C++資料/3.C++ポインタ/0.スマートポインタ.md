# **スマートポインタ**

---

## **ポインタを扱う危険性**

`new` や `delete` を利用した動的なメモリの利用に関して、  
メモリリークや不正なメモリアドレスを参照するなど、  
色々と危険な問題を起こしやすい事は理解して頂けたと思う。

とはいえゲームを作る上では、`new / delete` の利用は必須。  

出来るだけ気を付けて使う様にしなければならないが、  
どんなに気を付けてコーディングしていても、  
```int* ○○``` といったポインタ変数をそのまま利用している限り、  
問題を起こす可能性は低くない。

そこで「アドレスによる情報の取り扱い」を少しでも安全に行えるように、様々な手法が考案されている。  
その一つが<strong><span style="color: red;">スマートポインタ</span></strong>。

---

## **スマートポインタ** 

**スマートポインタ**は**必要なくなったメモリを自動的に解放する**仕組みを持つポインタ。  
動的にメモリを扱う際は必ずと言って良いほど利用されている。

また**確保したメモリの利用意図を人に把握させる**事にも役に立つ為、  
積極的に利用していってほしい。  

ちなみに ```int* ○○``` といったポインタ変数は「生ポインタ」と呼ばれ、  
スマートポインタとは区別される。

---
## **所有権**
スマートポインタを知る前に<span style="color: red;">**所有権**</span>という言葉を把握する必要がある。  
**所有権**とは一般的な意味で考えてもらって問題ない。  

物の管理はそれを所持している人が行うべき事で、  
他人が好きにして良いはずがない。  
それはコーディングにおいても同じ事が言える。  

```c++
#include <iostream>

int main()
{
	int* p1 = nullptr;
	int* p2 = new int;
	p1 = p2;

	*p1 = 10;

	// どれを delete するか？
	
	return 0;
}
```

この場合  ```p1``` と ```p2``` のどちらを `delete` してもかまわない。  
だが二つとも `delete` すると当然問題が発生する。   
つまり ```p1``` と ```p2```を使う側がしっかりと内容を把握した上で利用しないといけない。  

この様な問題を回避する為には、動的に確保したメモリに対して  
<span style="color: red;">**そのメモリを管理しているのが誰なのかを明確にしておく事が必要**</span> 。

**管理している誰か**が確保したメモリにアクセスする権利と、  
そのメモリを解放する権利を持っている。  
こういった権利を<span style="color: red;">**リソース（メモリ）の所有権**</span>と呼ぶ。

所有権は**譲渡**や**共有**を行う事が出来る。  

- **譲渡**  
    - 人に渡す事  
    - 所有権が相手に移り、自分は失う
- **共有**  
    - 人と一緒に持つ事  
    - 相手も自分も同じ所有権を持つ

スマートポインタの利用には、  
所有権がどこにあるのかを明確にする意図もある。  

---

## **スマートポインタの基礎的な仕組み**
所有権の明確化や、必要なくなったメモリの解放を、  
クラスを利用する事で実現させる事が出来る。

所有権を持つ簡易的なスマートポインタは、  
実は今までの知識で自作出来る様になっている。  

```c++
#include <iostream>

// int 型のスマートポインタクラス
class IntPtr {
public:
	IntPtr(int* ptr) {
		// new されたアドレスを保持する
		ptr_ = ptr;
	}

	~IntPtr() {
		// デストラクタで破棄（メモリ解放）
		delete ptr_;
	}

	void set(int value) {
		*ptr_ = value;
	}

	int get() {
		return *ptr_;
	}

private:
	// 外部からアクセスできない形でポインタ変数を宣言
	int* ptr_ = nullptr;
};


// メイン関数
int main()
{
	{
		// int 型スマートポインタをインスタンス化し、
        // コンストラクタの引数で確保したメモリアドレスを渡す
		IntPtr ptr(new int);

		// new したメモリの内容を設定
		ptr.set(3);

		// new したメモリの内容を取得
		int a = ptr.get();
	}
	// ptr の寿命が尽きてデストラクタが呼び出される
	// その時にポインタが delete される
	// 使う側は delete していないがメモリリークは回避できる
	
	return 0;
}

```

メイン関数で `new` で生成した int型 オブジェクトのアドレスを、  
そのまま引数付きコンストラクタに渡している。  

メイン関数では `delete` を行っていないが、  
```IntPtr``` のデストラクタで確保したアドレスに対して `delete` を行っている為、  
```ptr``` の破棄と同時にメモリが解放され、メモリリークは発生しない。

`new` したアドレスを知っているのは ```IntPtr の private メンバ変数``` だけであり、  
値の設定も取得もメモリ解放も ```ptr``` を介して行われている為、外から好き勝手出来ない。  
```ptr``` は <span style="color: red;">**new で確保したメモリの所有権を持つオブジェクト**</span>である事が明確になっている。

確保したメモリ（new で生成したインスタンスやオブジェクト）が途中で必要なくなる場合もあり得る。  
その場合、```ptr``` は残りつつ、**所有しているメモリは解放する** という形になる。

「所有権を持つオブジェクトが破棄されたからメモリを解放する」ではなく、  
「オブジェクトが所有権を放棄したからメモリを解放する」という言い方の方が正しい。


注意点として、今回の例はあくまで簡易的な実装である為、  
実用する為には、様々な用途に対応する必要がある。  
そういった点から、C++ が用意しているスマートポインタを利用した方が好ましい。
