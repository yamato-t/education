# **移動**

---
## **座標系**

ゲーム画面で物の位置を動かす際、<strong><span style ="color: #ff0000;"> 座標 </span></strong>という概念が必要になってくる。  
大抵の場合 x, y, z といった各軸の値の組み合わせで座標が表される。  
このような軸毎の値を組み合わせて位置を決める座標系の事を<strong><span style ="color: #ff0000;"> 直交座標系 </span></strong> と呼ぶ。

まずは下記二つの座標系をイメージ出来るようになってほしい。

- 2D 座標
- 3D 座標

### **2D 座標**
二つの軸 x, y で表される座標で二次元の位置を表す。  
PCのモニタも「スクリーン座標」という2D座標系で表す事が出来る。  
スクリーン座標の場合 y は下に向かうにつれて大きい値になる。

### **3D 座標**
二つの軸 x, y, z で表される座標で三次元の位置を表す。  
3D ゲームを作る際は画面の中に、横と縦と奥行の三次元の仮想空間を作る必要があり、  
その仮想空間内では、当然三次元を扱える座標系を利用する。

## **ベクトルの利用**
ゲームにおいて座標の計算にはベクトルが用いられる。  
ベクトルとは**向き**と**大きさ**を持つ量を表す。
  
現在の座標にベクトルを加算、あるいは減算する事で、  
座標を変化させていくのが最も基本的な座標更新処理になる。

ゲームプログラムにおいて、  
座標とベクトルは大概の場合両方とも `(x, y, z)` の形で扱われる事が多い。

座標も <strong><span style ="color: #ff0000;"> 原点からのベクトル（位置ベクトル） </span></strong> という意味で扱う事ができる。

ベクトルの表し方はこの様に書かれることが多い。

$A=(x,y)$

![translation3.png](https://bitbucket.org/repo/bx54aGe/images/349216013-translation3.png)

高校では $\vec{A}=(x,y)$ の様に矢印を入れている場合が多いと思う。

## **座標を移動するプログラム**
座標を踏まえた上で移動を考える。

ベクトルは各要素別に値が独立している為、   
例えば横の移動だけしたい場合は x を  
縦の移動だけしたい場合は y を変化させれば良い。

---
## **Sample 1**

今試したサンプルは x と y を分けて座標を変化させていた。  
つまり、二次元の位置を一次元の各要素に分けて計算していた。  
次に <strong><span style ="color: #ff0000;"> ベクトルで座標を更新する </span></strong>事を考えてみる。  
ある位置からあるベクトルを加える事で、座標を更新する。  
便宜上、あるベクトルを**移動ベクトル**と呼ぶようにする。

---
## **Sample 2**

座標に対して必ず移動ベクトルを加算し移動させている。  
移動ベクトル自体にマイナスの値が入ってる為これで問題ない。  

ベクトル A(Ax, Ay) とベクトル B(Bx, By) があった場合、  
その加減算の計算式は下記の通り  

### **加算**
$A + B = (Ax + Bx, Ay + By)$

### **減算** 
$A - B = (Ax - Bx, Ay - By)$   
$A - B = (Ax + -Bx, Ay + -By)$

これをコーディングしている。

## **Sample 3**

移動ベクトルそのものを計算から割り出す方法を考える。

クリックした座標から現在の座標を減算する。  
これで「クリックした座標までの移動ベクトル」が計算できる。  

ゆっくり動かす為に移動ベクトルを 30 で割って 30 回加算を繰り返す。  
つまり 30 フレームで目的地に到着するように計算している。

移動ベクトルの移動量を変化させる計算をしている。  
 1/30 しているが、これはベクトルの大きさに倍率を乗算している。  
ベクトル A (Ax, Ay) を n 倍する場合の計算式は下記の通り  

$nA=n(Ax,Ay)=(n\times Ax,n\times Ay)$

![translation5.png](https://bitbucket.org/repo/bx54aGe/images/3453216037-translation5.png)

この計算の場合、必ず 30 回加算を繰り返す事でクリックした座標に到着してしまう。  
つまり移動速度が一定ではない。

## **Sample 4**

これを特定の速度で動かす様に考えてみる。

**Sample 3**では、1/30 していた為、  
どんな位置であっても到着するまでの時間は同じ 30 フレームだった。  
今回は移動量を固定する事で到着するまでの時間が一定ではない。  
その為、予め移動量（速さ）と距離から移動時間（加算回数）を計算する。

その加算回数でクリックした座標までの移動ベクトルを分割する。  
この時、分割された移動ベクトルの移動量は、  
決められた移動量とほぼ同じになるはず。

### **二点間の距離の計算**

`vector.magnitude` で行っている事は<strong> 二つの点の距離を出す </strong> 事。  
これを自前で計算する。

<strong><span style ="color: #ff0000;"> 三平方の定理（ピタゴラスの定理） </span></strong>が使える。  
直角三角形の場合、斜辺を C 、その他二辺を A、 B とすると  
$C^{2}=A^{2}+B^{2}$

が成り立つという法則。

![translation6.png](https://bitbucket.org/repo/bx54aGe/images/4066584422-translation6.png)

二点の位置関係を考えると、必ず直角を含める三角形になる。  
その為 x、y 要素の差を自乗して加算した結果の平方根が今回必要な移動量になる。

```c++
float x = (targetPosition.x - currentPosition.x);
float y = (targetPosition.y - currentPosition.y);
float length = Mathf.Sqrt((x * x) + (y * y));
```

### **ベクトルの正規化**

ベクトルの正規化とは<span style="color: red;">**ベクトルの大きさを「1」にする事**</span>。  
大きさが「1」のベクトルは<span style="color: red;">**単位ベクトル**</span>と呼ばれる。

ベクトルの大きさを意図したの値にしたい場合、   
1. まずベクトルを単位ベクトルにする
2. 大きさに倍率をかける

という方法が一般的。  

ベクトル A(Ax, Ay) の単位ベクトル $\hat{A}$ の計算方法は以下の通り。  
$\hat{A}=\frac{1}{\|\boldsymbol{A}\|}(Ax,Ay)$

$\|\boldsymbol{A}\|$ はベクトルAの大きさを指す。

これを踏まえて、
単位ベクトルを利用した移動ベクトル計算処理を作成するとこうなる。

```c++
// 移動量を計算する
float x = (targetPosition.x - currentPosition.x);
float y = (targetPosition.y - currentPosition.y);
float length = Mathf.Sqrt((x * x) + (y * y));

// 移動ベクトルの単位ベクトルを計算する
vector = vector / length;

// 移動ベクトルの移動量を 0.1 にする
vector_ = vector * scalar_;
```
