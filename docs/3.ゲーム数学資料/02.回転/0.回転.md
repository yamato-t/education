# **回転**

---

ベクトル各軸の値を更新する事で移動出来る事を理解して頂いたと思う。  
次に回転に焦点を当てる。

ゲームにおいて物が回転するというのは、とても基本的な挙動にあたる。  
ところがこの回転というのはかなり奥が深い。  
回転を使いこなす為には、様々な事前知識が必要になる。

---
## **平行移動との計算方法の違い**

円運動はまっすぐ動くわけではない。  
平行移動とは違うアプローチで計算する。

この動きを実装する為に下記を押さえておく必要がある。  

- **極座標系**
- **三角関数**
- **ラジアン**

### **極座標系**

極座標系とは**半径 r と 角度 θ で位置を表す座標系**を指す。  
直交座標系と同じ見た目ではあるが XY といった軸がない。

![rotation1.png](https://bitbucket.org/repo/477kKqM/images/4263673140-rotation1.png)

円運動は、まず円の半径と角度から極座標系における位置を計算し、  
その結果を直交座標系に変換する事で、x や y の軸で表現できる座標を得る。

### **三角関数**

極座標系における位置を、直交座標系の各軸の座標に変換する。  
ここで必要になるのが <strong><span style ="color: #ff0000;"> 三角関数 </span></strong>。  
これらは<strong><span style ="color: #ff0000;"> 直角三角形の辺同士の比率 </span></strong>を指している。

| 名前              | 意味                          | 説明                        |
|------------------|-------------------------------|-------------------------------|
| sinθ          | 縦/斜辺  | 斜辺を基準として、縦がどれくらいの割合か               |
| cosθ          | 横/斜辺  | 斜辺を基準として、横がどれくらいの割合か               |
| tanθ          | 縦/横 | 横を基準として、縦がどれくらいの割合か               |

この関係性から  
<strong><span style ="color: #ff0000;"> 斜辺(直角三角形で一番長い辺) </span></strong> と  <strong><span style ="color: #ff0000;"> 各割合 </span></strong> を利用する事で、  
**各辺の長さ**が計算できるようになる。

つまり、

- **半径に cosθ を 乗算すると、向きに合わせた横の長さ（X）が分かる**
- **半径に sinθ を 乗算すると、向きに合わせた縦の長さ（Y）が分かる**

という事になる。

![rotation2.png](https://bitbucket.org/repo/477kKqM/images/1755474991-rotation2.png)

これは言い換えると  
<strong><span style ="color: #ff0000;"> 回転している半径を軸毎に分解して、  
直交座標系の座標に変換しているのと同じ </span></strong>になる。

sin, cos には周期があり、0 ~ 360 の変化をループする。  
この特性を利用する事で、回転し続ける移動を作成する事ができる。  
Y の値だけ変更するなどして、楕円も作る事が出来る。

### **ラジアン**

角度を指定して三角関数を計算する関数を利用するが、  
角度の単位には注意する事がある。

普段 1 周 360 度で認識しているが、これは <strong><span style ="color: #ff0000;"> 度数法 </span></strong>における角度の単位になる。  
対してプログラムでは <strong><span style ="color: #ff0000;"> 弧度法（ラジアン） </span></strong> と呼ばれる角度の単位が利用される。  

その為、度数法の値をそのまま指定する事は出来ず、  
弧度法の値に変換して指定しなければならない。  
変換には円周率 π(3.14) が必要。

- 度数法から弧度法に変換  
$rad=(\pi\div 180)\times angle$
- 弧度法から度数法に変換  
$angle=(180\div\pi)\times rad$

仮に 60度 を弧度法で表すとすると  
$(3.14\div 180)\times 60=1.04$   
程度となる

## **ベクトルの回転**

<strong><span style ="color: #ff0000;"> すでに何らかの方向を向いているベクトルをさらに回転させる </span></strong> という事を考えてみる。  

以前紹介した  
「飛んでいく位置を先に決めてから二つの位置の差から移動ベクトルを計算する方法」  
でも可能。  

だが、仮に作成するゲームでエイム操作があり、その仕様が   
<strong><span style ="color: #ff0000;"> 発射角度を変更してその方向に発射する </span></strong>  
だった場合、位置関係で計算する方法だと不十分となる。

<strong><span style ="color: #ff0000;"> すでにどこかの方向を向いている移動ベクトルを回転する計算 </span></strong>が必要になる。  

円運動では半径を回転させ座標に変換していた。  
その回転、変換計算を移動ベクトルでも流用する。

## **角度の情報がない場合の回転**

場合によっては「今向いている角度」の情報がない状況で、  
ベクトルを回転させる必要も出てくる。

そのような場合は「直交座標系でのベクトルの回転」を考える。  
極座標において、長さ r 角度 θ で (x, y) を表すとこの様になっていた。

$$\begin{pmatrix}x\cr y\end{pmatrix}=\begin{pmatrix}r\cos\theta\cr r\sin\theta\end{pmatrix}$$

ここから角度 A の回転を加えた (x', y') は下記の式で計算出来る。

$$\begin{pmatrix}x'\cr y'\end{pmatrix}=\begin{pmatrix}r\cos(\theta+A)\cr r\sin(\theta+A)\end{pmatrix}$$

sin, cos の内側に加算式が入っている事に注目。

### **加法定理**

三角関数での加算には<strong><span style ="color: #ff0000;"> 加法定理 </span></strong> と呼ばれる式が利用される。


$$\begin{aligned}
\cos(\theta\pm A)&=\cos\theta\cos A\mp\sin\theta\sin A \\
\sin(\theta\pm A)&=\sin\theta\cos A\pm\cos\theta\sin A
\end{aligned}$$


式をあてはめると、

$$\begin{pmatrix}x'\cr y'\end{pmatrix}=\begin{pmatrix}r\cos(\theta+A)\cr r\sin(\theta+A)\end{pmatrix}=\begin{pmatrix}r\cos\theta\cos A-r\sin\theta\sin A\cr r\sin\theta\cos A+r\cos\theta\sin A\end{pmatrix}$$

これで漸く移動ベクトルの回転が計算できる。

加法定理は「ベクトルが向いている今の角度」を必要としない計算方法だが、  
「ベクトルが向いている今の角度」自体を計算する方法もある。

### **逆三角関数**

「ベクトルが向いている今の角度」がわかれば、加法定理による計算は必要なく、  
単に「今の角度」から回転角度を加算して三角関数でベクトルを更新すればいい。  

ベクトルから角度を計算する為には、まず**逆三角関数**を知る必要がある。  
逆三角関数は、三角関数値から角度を得るための物。


| 名前             | 説明                        |
|------------------|-------------------------------|
| asin(x) (arcsin)          | sin 値 x から角度を得る              |
| acos(x) (arccos)          | cos 値 x から角度を得る               |
| atan(x) (arctan)          | tan 値 x から角度を得る               |
| atan2(y, x)          | y と x から角度を得る               |

プログラムにおいて、逆三角関数で得られる値はラジアンになる。  
ベクトル(x,y) から角度を得る為には、最後の atan2 を利用する。

Unity では `Mathf.Atan2` という関数があるのでこれを利用すればいい。
