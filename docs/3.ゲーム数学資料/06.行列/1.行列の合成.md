
# **行列の合成**

---

## **回転行列の合成**

軸別に回転行列があり「X 軸 30 度回転、Y 軸 40 度回転、z 軸 20 度回転させたい」場合、  
<span style ="color: #ff0000;"> **回転行列の合成** </span> で実現できる。  
要するに<span style ="color: #ff0000;">**行列同士を乗算**</span>してやればいい。
$$
Rxyz = RxRyRz
$$

ただしここで注意するべきなのが<span style ="color: #ff0000;">**掛け算する順番によって結果が変わる**</span>という事。

たとえ同じベクトルであっても $Rxyz$ と $Rzxy$ といった様な、  
計算順番の違う行列で回転させると結果が変わる。

ベクトルをどの様に回転させたいかによって、順番を考える必要がある。

## **移動と回転と拡縮を一括反映**

オブジェクトの位置や向きは

- 平行移動
- 回転移動（原点を中心）  

とそれぞれ個別に計算してきたが、予め各要素の行列を計算し、  
<span style ="color: red;">**全ての情報を反映した一つの行列を作成しておくことも可能**</span>。

平行移動行列、回転行列、拡縮行列を合成した新しい同次座標行列を作成し、  
今の位置の同次座標とかけ合わせる事で、  
平行移動、回転、拡縮を反映した位置に移動させる事が出来る。

行列を合成する事で<strong><span style ="color: #ff0000;">移動、回転、拡大縮小を一括で計算出来る</span></strong>と覚えておくのがいい。

### **Unity における乗算順**

前述でも挙げているが合成の際は<span style ="color: red;">**掛け算する順番**</span>に注意する。  
Unity では行列を右から左の順に適用するため、  
移動行列 $T$ 、回転行列 $R$ 、拡縮行列 $S$ は下記の順で合成する。

$$
M = TRS
$$

右端の $S$ から順に適用されるので、  
<span style ="color: red;">**拡縮 → 回転 → 移動**</span>の順で反映される。

```c#
// 移動・回転・拡縮の各行列を作成する
Matrix4x4 t = Matrix4x4.Translate(new Vector3(1.0f, 0.0f, 0.0f));
Matrix4x4 r = Matrix4x4.Rotate(Quaternion.Euler(0.0f, 90.0f, 0.0f));
Matrix4x4 s = Matrix4x4.Scale(new Vector3(2.0f, 2.0f, 2.0f));

// T x R x S の順で合成する（拡縮 → 回転 → 移動 の順に反映）
Matrix4x4 m = t * r * s;

// Unity には同じ結果を一括で作る専用関数も用意されている
Matrix4x4 trs = Matrix4x4.TRS(
    new Vector3(1.0f, 0.0f, 0.0f),       // 移動量
    Quaternion.Euler(0.0f, 90.0f, 0.0f), // 回転量
    new Vector3(2.0f, 2.0f, 2.0f));      // 拡縮率
```

順番を変えると下記の様に結果が変わってしまう。

- $M = RTS$（移動してから回転）  
  移動後の座標ごと原点中心に回転するため、  
  その場で向きを変えるのではなく、  
  原点の周りを公転する様な位置に飛んでいく
- $M = STR$（拡縮を最後に適用）  
  移動量まで拡縮されるため、  
  拡縮率 2 で移動量 1 のつもりが 2 移動してしまう

<span style ="color: red;">**$TRS$ の順で合成する**</span>と覚えておく。

## **物同士の連結**

行列を使う事で<strong><span style ="color: #ff0000;">ある物を別の物に追従させる事</span></strong>が簡単に出来る。 

例えば人の体をイメージする。  
人の体は関節で繋がっており、別の関節の動作に影響を受けている。  
指の位置は肘の位置の影響を受け、肘の位置は肩の位置の影響を受ける。  
行列はこのような親子関係を表現する事が可能。

「肩から肘」「肘から指」を表す独立した行列があり、  
それら行列が親子関係で繋がる事で末端の指の場所を決める事が出来る。  
3D ゲームにおいて、キャラクターモデルがアニメーション出来るのはこのおかげ。

![親子関係.png](https://bitbucket.org/repo/jKRX45X/images/426277732-%E8%A6%AA%E5%AD%90%E9%96%A2%E4%BF%82.png)

こちらも注意すべきなのは<span style ="color: red;">**行列の掛け算順**</span>。  
<span style ="color: red;">**最終行列 = 親行列 x 子行列**</span>になる。

例を当てはめると  
- 「指の場所」の行列 =「肩から肘」の行列 x 「肘から指」の行列  
- 「まず子（肘から指）のローカル変換が適用され、その結果が親（肩から肘）の変換で運ばれる」  
となる。

**〇の位置や向きの影響を受けて、△の位置や向きを変化させる**といった際に行列は役に立つ。

## **DirectX では掛け算順が逆になる**

ここまでの説明は、ベクトルを縦に並べて行列の右側に置く  
<span style ="color: red;">**列ベクトル形式**</span>（$v' = Mv$）を前提としてきた。  
Unity の `Matrix4x4` もこの形式を採用している。

一方 DirectX（DirectXMath や HLSL の `mul()` の慣例）では、  
ベクトルを横に並べて行列の左側に置く  
<span style ="color: red;">**行ベクトル形式**</span>（$v' = vM$）が使われる。

$$
\begin{pmatrix}
x' & y' & z' & 1
\end{pmatrix}=
\begin{pmatrix}
x & y & z & 1
\end{pmatrix}
M
$$

行ベクトル形式では行列が<span style ="color: red;">**左から右の順に適用される**</span>ため、  
合成の掛け算順がこれまでの説明と逆になる。

- 移動・回転・拡縮の合成  
  $M = TRS$ ではなく $M = SRT$ と書く  
  （適用順は同じく 拡縮 → 回転 → 移動）
- 物同士の連結  
  最終行列 = 親行列 x 子行列 ではなく  
  <span style ="color: red;">**最終行列 = 子行列 x 親行列**</span> と書く

どちらの形式でも<span style ="color: red;">**変換が適用される順序自体は変わらない**</span>。  
「ローカルな変換から先に適用する」という考え方は共通で、  
式の上での書き順だけが変化する。

利用するライブラリがどちらの形式なのかを確認してから、  
掛け算順を組み立てる事が重要となる。
