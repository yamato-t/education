# **落下運動**

---

以前<strong><span style ="color: #ff0000;"> 等加速運動 </span></strong>に関して話し、  
式をコーディングで利用する例を挙げた。

今回は前回の内容を元に、物理の入門としてよく取り扱われている  
重力を考慮した挙動のプログラムを作成する。

---

## **自由落下**

<strong><span style ="color: #ff0000;"> 自由落下 </span></strong>とは物体が空気などの影響を受けずに、  
純粋に重力の働きだけで落下する現象を指す。

自由落下における加速度は、  
<strong><span style ="color: #ff0000;"> 重力加速度 </span></strong>とよばれ値も決まっている。  

- 重力加速度  
$9.8\hspace{2pt}\mathrm{m/s^2}$

### **滞空時間の予測**

前回同様、移動にかかる予測時間を計算する。

前回は壁まで 30m と決まっていたが、今回は高さを自由に決められる。  
position の y の値がゼロになっている位置を地面として計算する。  

自由落下において地面までの滞空（落下）時間 t を求める計算式は下記の通り。

- 落下距離  
$h (\mathrm{m})$  
- h の距離の落下に掛かる時間 t を求める式  
$t = \sqrt{\frac{2h}{g}}$  

---

## **地面から真上に飛んで自由落下**

次は「地面から真上に物を飛ばして自由落下で地面に戻る挙動」  
垂直ジャンプを作成する。

単純にその場から落とすのではない為、  
「その場から落下する」と比べて下記の点が変化する。

- 「真上」に飛ばすという事は、最初のベクトル向きが上向きになる事  
- 真上に「飛ばす」という点から、初速が必要な事  
- 「真上に飛んだものが落下する」ので、ベクトル向きが途中で反転する事  

### **滞空時間の予測**

初速がある場合、滞空時間の計算式も変わるので注意。

- 初速  
$v0 (\mathrm{m/f})$  
- 滞空時間 t を求める式  
$t = \frac{2v_{0}}{g}$

## **放物運動**

「その場で真上にジャンプできる」だけで成り立つゲームはあまりない。  
「ジャンプ中も移動できる」のが一般的。  

つまり、ゲームでは放物運動を作る必要がある。  
ただ、この放物運動は「垂直ジャンプ」さえ出来ればすぐに作れる。  
垂直ジャンプ のプログラムに「XZ 平面上の移動量」を追加するだけ。  

放物運動は

- <span style ="color: #219ddd;"> 水平の等速運動 </span>  
- <span style ="color: #219ddd;"> 垂直の自由落下（重力による等加速運動） </span>  

の <strong><span style ="color: #ff0000;"> 二つが独立して作用する </span></strong>事で起こっている。

![放物運動.png](https://bitbucket.org/repo/nK7ojxo/images/3257129301-%E6%94%BE%E7%89%A9%E9%81%8B%E5%8B%95.png)

向きと速さから初速を表す速度ベクトルを定義し、  
そこから水平（XZ）と垂直（Y）の速度を分離し、  
水平の移動はそのまま、垂直の移動は重力加速度を加え続ける。

### **滞空時間と着地位置の予測**

放物運動の滞空時間の計算方法は「垂直ジャンプ時と同じ」になる。  
「空中にいるかどうか」は垂直の移動だけに注目すればよく、水平の移動を考慮する必要はない。

また、この例の場合、滞空時間（フレーム）が分かれば着地位置も簡単に割り出せる。  
水平移動は等速運動の為、単に「滞空フレーム分 XZ ベクトルで進んだ位置」が着地点になる。  

但し、上記は地面が水平な前提の話。  
凹凸がある場合は、単純な計算では出せない。

## **最高度を固定した放物運動**

例えばアクションゲーム等を作っていると、  
ジャンプの最高到達高度を調整できる様にしたい場合があったりする。  

例えば「絶対に 3m に届かない様にしたい」等の仕様では、  
初速を調整するよりも、到達高度を調整できる様にする方が好ましい。
