    移動

    座標系

    ゲーム画面で物の位置を動かす際、 座標 という概念が必要になってくる。
    大抵の場合 x, y, z といった各軸の値の組み合わせで座標が表される。
    このような軸毎の値を組み合わせて位置を決める座標系の事を 直交座標系 と呼ぶ。

    まずは下記二つの座標系をイメージ出来るようになってほしい。

    • 2D 座標
    • 3D 座標

    2D 座標

    二つの軸 x, y で表される座標で二次元の位置を表す。
    PCのモニタも「スクリーン座標」という2D座標系で表す事が出来る。
    スクリーン座標の場合 y は下に向かうにつれて大きい値になる。

    3D 座標

    二つの軸 x, y, z で表される座標で三次元の位置を表す。
    3D ゲームを作る際は画面の中に、横と縦と奥行の三次元の仮想空間を作る必要があり、
    その仮想空間内では、当然三次元を扱える座標系を利用する。

    ベクトルの利用

    ゲームにおいて座標の計算にはベクトルが用いられる。
    ベクトルとは向きと大きさを持つ量を表す。

    現在の座標にベクトルを加算、あるいは減算する事で、
    座標を変化させていくのが最も基本的な座標更新処理になる。

    ゲームプログラムにおいて、
    座標とベクトルは大概の場合両方とも (x, y, z) の形で扱われる事が多い。

    座標も 原点からのベクトル（位置ベクトル） という意味で扱う事ができる。

    ベクトルの表し方はこの様に書かれることが多い。

    \(A=(x,y)\)

    高校では \(\vec{A}=(x,y)\) の様に矢印を入れている場合が多いと思う。

    座標を移動するプログラム

    座標を踏まえた上で移動を考える。

    ベクトルは各要素別に値が独立している為、
    例えば横の移動だけしたい場合は x を
    縦の移動だけしたい場合は y を変化させれば良い。

    ベクトルの加減算

    ベクトル A(Ax, Ay) とベクトル B(Bx, By) があった場合、
    その加減算の計算式は下記の通り

    加算

    \(A + B = (Ax + Bx, Ay + By)\)

    減算

    \(A - B = (Ax - Bx, Ay - By)\)
    \(A - B = (Ax + -Bx, Ay + -By)\)

    これをコーディングしている。

    移動ベクトルの算出

    ある位置 p から別の位置 o に一定時間で移動させる事を考える。
    「 p から o までのベクトル」は前述の減算で計算できる。

    ところが減算で算出したベクトルは、文字通り「 p から o までのベクトル」である為、
    そのベクトルで加算すると、1 フレームで o にワープするような移動になってしまう。
    そこで、ゆっくり動かす為に移動ベクトルを 30 で割って 30 回加算を繰り返す。

    つまり 30 フレームで o に到着するように計算している。

    移動ベクトルの移動量を変化させる計算をしている。
    1/30 しているが、これはベクトルの大きさに倍率を乗算している。
    ベクトル A (Ax, Ay) を n 倍する場合の計算式は下記の通り

    \(nA=n(Ax,Ay)=(n\times Ax,n\times Ay)\)

    この計算の場合、必ず 30 回加算を繰り返す事でクリックした座標に到着してしまう。
    つまり移動速度が一定ではない。

    指定の移動量を持つ移動ベクトルの算出

    ある位置 p から別の位置 o に一定速度で移動させる事を考える。

    前述の方法では、1/30 していた為、
    どんなに距離があっても到着するまでの時間は同じ 30 フレームだった。

    今回は時間ではなく移動量を固定する。
    移動量を固定すると聞くと、
    「移動ベクトルの xy それぞれに指定値を代入すればいい」
    と思えるかもしれないがそれは誤り。

    例えば「移動量が 2 の移動ベクトルを作る」となった時、
    x,y それぞれに 2 を代入するとベクトルの移動量は 2 ではなくなる。

    計算アプローチは逆で、p から o に向かう移動ベクトルを算出した上で
    移動量が「2」となる様に x と y を逆算しなければならない。

    その為には - 二点間の距離を計算する - 距離を使って移動ベクトルを単位ベクトルにする - 単位ベクトルに指定した大きさを乗算する

    という手順が必要。

    二点間の距離の計算

    三平方の定理（ピタゴラスの定理）が使える。
    直角三角形の場合、斜辺を C、その他二辺を A、 B とすると
    $\(C^{2}=A^{2}+B^{2}\)$

    が成り立つという法則。

    これを踏まえて、改めて二点の位置関係を考えると、
    二点とその移動ベクトルは、必ず直角を含める三角形になる事が分かる。
    p から o に向かう移動ベクトルの x、y 要素の差を自乗して加算した結果の平方根が二点間の距離となる。

    float x = (o.x - p.x);
float y = (o.y - p.y);
float length = Mathf.Sqrt((x * x) + (y * y));

    単位ベクトル

    任意の向きで大きさが「1」のベクトルを単位ベクトルと呼ぶ。
    あるベクトルの単位ベクトルを計算する事を正規化と呼ぶ。

    ベクトルを指定した大きさにしたい場合、
    1. ベクトルを単位ベクトルにする 2. 単位ベクトルに「指定の大きさ」を乗算する

    という方法が一般的。

    ベクトル A(Ax, Ay) の単位ベクトル \(\hat{A}\) の計算方法は以下の通り。
    $\(\hat{A}=\frac{1}{\|\boldsymbol{A}\|}(Ax,Ay)\)$

    \(\|\boldsymbol{A}\|\) はベクトル A の大きさを指す。
    ここでいう「大きさ」は前述の二点間の距離と同義。

    これを踏まえて、 単位ベクトルを利用した移動ベクトル計算処理を作成するとこうなる。

    // 移動量を計算する
float x = (targetPosition.x - currentPosition.x);
float y = (targetPosition.y - currentPosition.y);
float length = Mathf.Sqrt((x * x) + (y * y));

// 移動ベクトルの単位ベクトルを計算する
vector = vector / length;

// 移動ベクトルの移動量を　scalar_ の値にする
vector_ = vector * scalar_;

    あとはこの vector_ を毎フレーム座標に加算し続けれれば、
    一定速度で移動する処理が完成する。
