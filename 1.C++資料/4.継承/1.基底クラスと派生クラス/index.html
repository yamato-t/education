    基底クラスと派生クラス

    クラスの継承方法

    まずはクラスを継承する最小のコーディングを紹介する。
    下記は自動車クラスを継承したタンクローリークラスの定義となる。

    // 自動車クラス
class Car
{
public: 
    Car() {}
    ~Car() {}
};

// タンクローリークラス
// 車クラスを継承する
class Tank : public Car
{
public:
    Tank() {}
    ~Tank() {}
};

    継承において重要な点は下記の部分になる。 

    class Tank : public Car

    クラス名を定義するまでは同じで、その後ろに : public Car が増えている。
    こうする事で Car クラスの継承を行う事が出来る。

    この時、
    Car　クラスは基底クラス（親クラス、スーパークラス）
    Tank クラスは Car クラスの派生クラス（子クラス、サブクラス）
    と呼ばれる。

    なお複数クラスを継承することも可能で、
    その場合は , で繋げていく。 

    class Tank  : public Car , public xxx

    基底クラスの内容を受け継ぐ派生クラス

    派生クラスは基底クラスのメンバ関数やメンバ変数を利用する事が出来る。

    #include <iostream>

// 自動車クラス
class Car
{
public:
    Car() {}
    ~Car() {}

public:
    // エンジンを始動する
    void startEngine()
    {
        std::cout << "エンジンを始動する" << std::endl;
        running_ = true;
    }

    // エンジンを停止する
    void stopEngine()
    {
        std::cout << "エンジンを停止する" << std::endl;
        running_ = false;
    }


public:
    bool running_ = false; // エンジンが動いている
};

// タンクローリークラス
class Tank : public Car
{
public:
    Tank() {}
    ~Tank() {}

public:
    // 燃料を送出する
    void pumpOut()
    {
        // running_ を宣言しているのは Car クラスだが、
        // Tank クラスが Car クラスを継承している為利用できる
        if (running_) {
            std::cout << "エンジンが動いているので燃料の送出は危険" << std::endl;
        }
        else
        {
            std::cout << "燃料を送出する" << std::endl;
        }
    }

};

// エントリー関数
int main()
{
    Car car;
    // 車のエンジンを始動する
    car.startEngine();

    Tank tank;
    // startEngine() と stopEngine() を定義しているのは Car クラスだが、
    // Tank クラスが Car クラスを継承している為利用できる
    tank.startEngine();
    tank.stopEngine();

    // 当然 Tank クラスで定義した関数は呼び出せる
    tank.pumpOut();
}

    running_ 変数と startEngine(), stopEngine() 関数に注目してほしい。
    それぞれ Carクラスで宣言、定義しているが、
    派生クラスであるTankクラスや、そのインスタンスからも利用できている。

    基底クラスで宣言、定義したものは派生クラスで再利用できる。
    ただし基底クラス側のアクセス指定子によっては利用できない場合もある。

    名称 説明
    public 完全に公開され、何処でも利用できる
    private 非公開とし、自身のクラス内でのみ利用できる
    protected 非公開とし、自身と、自身を継承したクラス内でのみ利用できる

    下記は基底クラスのアクセス指定子を幾つか変更した結果になる。

{

    }

    メンバ関数の再定義

    派生クラスは基底クラスのメンバ関数を再定義する事が出来る。 

    // 自動車クラス
class Car
{
・・・　省略

    // （何かを）運ぶ
    void carry()
    {
        std::cout << "何かを運ぶ" << std::endl;
    }

・・・　省略
};

// タンクローリークラス
class Tank : public Car
{
・・・　省略

    // 基底クラスと全く同じ構成の関数を再定義する
    void carry()
    {
        // 中身はタンクローリー用
        std::cout << "燃料を運ぶ" << std::endl;
    }

・・・　省略
};

// エントリー関数
int main()
{
    Car car;
    car.carry();

    Tank tank;
    tank.carry();
}

    結果は下記の通り 

    何かを運ぶ
燃料を運ぶ

    両方に引数も戻り値も名前も同じ carry() 関数があるが、
    実行すると tank 側は再定義した内容で処理されている事がわかる。

    派生クラスのコンストラクタとデストラクタ

    クラスがインスタンス化される際にコンストラクタが、
    インスタンスが破棄される際にデストラクタが処理される事は、
    すでに把握されていると思う。

    派生クラスであっても勿論処理されるが、
    その場合基底クラスのコンストラクタとデストラクタも処理される。

    #include <iostream>

// 自動車クラス
class Car
{
public: 
    Car() {
        std::cout << "Car のコンストラクタが処理されました" << std::endl;
    }

    ~Car() {
        std::cout << "Car のデストラクタが処理されました" << std::endl;
    }
};

// タンクローリークラス
class Tank : public Car
{
public:
    Tank() {
        std::cout << "Tank のコンストラクタが処理されました" << std::endl;
    }

    ~Tank() {
        std::cout << "Tank のデストラクタが処理されました" << std::endl;
    }
};

// エントリー関数
int main()
{
    {
        // タンクローリークラスをインスタンス化する
        Tank* tank = new Tank;
        // 破棄する
        delete tank;
    }
}
    上記コードを実行すると、下記の様に出力される 
    Car のコンストラクタが処理されました
Tank のコンストラクタが処理されました
Tank のデストラクタが処理されました
Car のデストラクタが処理されました

    インスタンス化の際は下記の順で処理される。
    1. 基底クラスのコンストラクタ
    2. 派生クラスのコンストラクタ

    逆に破棄の際は下記の順で処理される。
    1. 派生クラスのデストラクタ
    2. 基底クラスのデストラクタ

    派生クラスインスタンスのアドレスを基底クラスポインタ型で利用できる

    継承における非常に重要な点。 

    // エントリー関数
int main()
{
    Tank* tank = new Tank;
    // Tank のアドレスを Car ポインタで受け取る事が出来る
    Car* car = tank;

    // Car の carry() が呼び出される
    car->carry();
    // Car に 宣言されていないのでコンパイルエラー
    car->pumpOut();

・・・省略
}
    Tank インスタンスのアドレスを持つポインタを Car ポインタに代入し、利用する事が出来る。
    ただし car-> で利用できるのはCar クラスで宣言されているメンバ変数とメンバ関数のみになる。
    Tank(派生)クラスで再定義した carry() も処理されない。

    継承関係にあるクラスポインタ型の変換

    お互いが継承関係にあるクラスである場合、
    インスタンスのアドレスが入ったポインタを、
    相互のポインタ型に変換（キャスト）する事ができる。

    上記の項目で挙げた様に、派生クラスのポインタを、
    基底クラスのポインタに変換する事をアップキャストと呼ぶ。

    逆に、基底クラスのポインタを、
    派生クラスのポインタに変換する事はダウンキャストと呼ぶ。 

    // エントリー関数
int main()
{
    Tank* tank = new Tank;

    // 派生クラスを基底クラスに変換 →　アップキャスト
    Car* car = tank;

    // 基底クラスを派生クラスに変換 →　ダウンキャスト(※ただしこの書き方は非推奨)
    tank = (Tank*)car; 
}

    注意点として、上記の様なダウンキャストの記述は非推奨になる。
    というよりしない方がいい。
    ダウンキャストは、メモリ上で確保されていない領域を参照してしまう可能性がある。
    C++では少しでもダウンキャストを安全に行う為の機能（dynamic_cast）も提供されており、
    それを利用する事をお勧めする。

    もう一点、アップキャストして受け取ったポインタを delete した際、
    派生クラスのデストラクタが呼ばれなくなる。

    // エントリー関数
int main()
{
    // 派生クラスインスタンスのアドレスをアップキャストで受け取る
    Car* tank = new Tank;
    // 破棄する
    delete tank;
}
    結果は下記の通り、Tank のデストラクタが処理されなくなる。 
    Car のコンストラクタが処理されました
Tank のコンストラクタが処理されました
Car のデストラクタが処理されました
