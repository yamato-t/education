    代入演算子オーバーロード

    代入演算子オーバーロード

    演算子の中でも「複製」に利用される代入演算子のオーバーロードについて説明する。

    代入演算子(=)にはコピー代入演算子やムーブ代入演算子があり、
    これらもオーバーロードは可能になっている。

    下記で「同型のconst参照を受け取って、自分自身の参照を返す」という処理を定義しているが、
    これはコピー代入演算子オーバーロードを定義している。

    // 動物クラス
class Animal final
{
public:
    // コピー代入演算子オーバーロード
    Animal& operator =(const Animal& animal);

public:
    std::string     name_       = {};
    int             footNum_    = 0;
};

// コピー代入演算子オーバーロード
Animal& Animal::operator =(const Animal& src)
{
    // 引数で受け取ったインスタンス（参照）の内容で上書きする
    name_ = src.name_;
    footNum_ = src.footNum_;

    // 自分自身の参照を返す
    return *this;
}

    コピー代入演算子オーバーロードの利用方法

    下記の様に = を使うだけでコピー代入演算子オーバーロードが処理される。

    // 各インスタンスを作成する
Animal animal1("動物１", 2);
Animal animal2;
Animal animal3;

// 内容を複製する
// コピー代入演算子オーバーロードを宣言、定義した事によって
// 独自に定義した内容の関数が呼び出される
animal3 = animal2 = animal1;

    コピーコンストラクタとコピー代入演算子

    コピーコンストラクタとコピー代入演算子の呼び別けは、
    間違って覚えてしまう人も多いので、正しく把握しておいてほしい。

    Animal animal1("動物１", 2); // コンストラクタが処理される

Animal animal2 = animal1;    // コピーコンストラクタが処理される

Animal animal3;
animal3 = animal1;           // コピー代入演算子オーバーロードが処理される

    コピー代入演算子のオーバーロードをわざわざ定義する必要性も、
    コピーコンストラクタと似ている    。

    演算子オーバーロードの省略

    大半の演算子のオーバーロードは宣言、定義を省略して利用すると当然コンパイルエラーになるが、
    代入演算子は省略してもコンパイルエラーにはならない。
    省略した場合、メンバ変数の内容を全て複製する処理が自動で定義される。
    が、複製されるメンバ変数が何なのかを良く理解しておく必要がある。

    特定のクラスの複製を禁止する為に、
    代入演算子オーバーロードが処理されるようなコードを書くと、
    コンパイルエラーになる様に実装する事も珍しくない。
