    可変長引数

    関数の引数の数を増減させる

    様々の処理の関数を作成していると、
    同じ関数でも引数の数を任意に増減させたいといった状況がしばしば発生する。

    基本的に関数の宣言時に、引数の数は決まってしまうのだが、
    引数の数を任意に変更する事は C 言語の時代から可能になっている。

    例えば printf 関数を思い出してほしい。

    // 引数の数は 4 つ
printf("%d, %f, %c\n", 10, 2.4f, 'a');

// 引数の数は 3 つ    
printf("%f, %s\n", 1.3f, "aaaaa");

    同じ関数を呼び出しながらも、引数で渡している数が違う。
    こういった引数を可変長引数と呼ぶ。

    可変長引数に対応する関数は自作可能。

    C 言語における可変長引数関数の実装方法

    #include <iostream>

// 可変長引数を扱う為に必要になる
#include <stdarg.h>

// 渡された引数をすべて合計する関数
// count    計算に利用する引数の数
// ...      計算に利用する可変長の引数
int sum(int count, ...) {

    // 可変長の引数のリストを保持する変数を宣言する
    va_list args;

    // 可変長の情報を利用した処理を開始する
    // 第二引数には可変部分が始まる直前の引数を指定する
    va_start(args, count);

    int answer{};

    // count の数だけループして数値を加算していく
    for (auto i = 0; i < count; ++i) {
        // 可変長の引数が順番に取得できる
        answer += va_arg(args, int);
    }

    // 可変長の情報を利用した処理を終了する
    va_end(args);

    return answer;
}

// エントリー関数
int main() {

    auto a1 = sum(5, 1, 2, 3, 4, 5);
    auto a2 = sum(7, 1, 2, 3, 4, 5, 6, 7);

    std::cout << a1 << std::endl;
    std::cout << a2 << std::endl;


    return 0;
}

    sum 関数の引数で、第二引数が ... となっている。
    これに可変長の引数である事を表す指定になっている。

    可変長引数の情報は引数を利用するのではなく、
    関数内で可変長部分を特殊な方法で取得して利用する。

    この実装方法の場合は可変長引数だけではなく、
    必ず通常の引数も必要になる。

    C ++における可変長引数関数の実装方法

    C言語の場合、可変長引数のみで関数を作成することが出来ない。
    ただ C++ においてはテンプレートを利用する事で、
    より直感的に使いやすい可変長引数関数を作成できる。
    これを可変引数テンプレートと呼ぶ。

    #include <iostream>

// 渡された引数をすべて合計する関数
template <class... T>
int sum(T... args) {

    // 可変長の引数の内容を配列にすべて詰め込む
    int list[] = {args...};

    int answer{};

    // 配列を範囲 for 文 でループさせて合計する
    for (auto i : list) {
        answer += i;
    }

    return answer;
}

// エントリー関数
int main() {
    auto a1 = sum(1, 2, 3, 4, 5);
    auto a2 = sum(1, 2, 3, 4, 5, 6, 7);

    std::cout << a1 << std::endl;
    std::cout << a2 << std::endl;

    return 0;
}

    sum 関数のテンプレートの仮引数部分と引数部分に注目する。
    テンプレートの仮引数部分で <class ...T> というように、
    仮引数の型名の前に ... が追加されている。

    さらに引数部分でも T ...args というように、
    引数名の前に ... が追加されている。

    この ... に続く T や args は パラメータパックと呼ばれる。

    前者の T はテンプレートで渡される引数の型がまとめられたパラメータパックを指し、
    後者の args は関数に渡された引数がまとめられたパラメータパックを指している。
    全ての引数がパラメータパックとしてまとめられて、関数の引数に渡されてくる。

    関数内でパラメータパック化した引数を利用する場合、
    パックされている内容を展開（個別に分解）する必要がある。
    それがこの一行になる。

    // 可変長の引数の内容を配列にすべて詰め込む
int list[] = {args...};

    分解する場合は args の後ろに ... を指定する。
    args...でパックを分解し、分解後の内容で配列を初期化している事になる。
    つまりこのコードは 

    // 可変長の引数の内容を配列にすべて詰め込む
int list[] = {1, 2, 3, 4, 5};　// 渡された順番と内容でそのまま配列に入れる
    という記述と同じ事になる。

    C++ 可変長引数の様々な用途

    パラメータパックは、まとめられた全ての引数が同じ型になっている必要はない。
    任意の複数の型と引数をパラメータパックにまとめる事ができる。

    // パラメータパック T にまとめられている型は順に int, float, char となる
template <class... T>
void func(T... args) {
    // args を展開すると 1, 2.1f, 'a' が順に入っている
}

// 引数の型が全て違う
func(1, 2.1f, 'a');

    この仕組みを利用して、下記のようなコーディングも可能

    #include <iostream>

// プレイヤーの列挙
enum class Player {
    Hero,
};

// プレイヤーの列挙
enum class Enemy {
    Boss,
};

// プレイヤークラス
class PlayerChara {
public:
    PlayerChara() { std::cout << "プレイヤーを生成します" << std::endl; }
};
// 敵クラス
class EnemyChara {
public:
    EnemyChara() { std::cout << "敵を生成します" << std::endl; }
};
// NPC クラス
class NPCChara {
public:
    NPCChara() { std::cout << "NPCを生成します" << std::endl; }
};

// プレイヤーを生成する関数
std::unique_ptr<PlayerChara> createChara(Player player, int attack, float speed, char weaponId) {
    return std::make_unique<PlayerChara>();
}

// 敵を生成する関数
std::unique_ptr<EnemyChara> createChara(Enemy enemy, int attack, float speed) {
    return std::make_unique<EnemyChara>();
}

// NPCを生成する関数
std::unique_ptr<NPCChara> createChara() {
    return std::make_unique<NPCChara>();
}

// キャラクター生成のテンプレート関数
// 戻り値の型を後置宣言する方法を利用して、関数の結果から戻り値型を推論する
template <class... T>
auto create(T&&... args) {
    // パラメータパックを展開して完全転送で対応する関数に渡す
    return createChara(std::forward<T>(args)...);
}

// エントリー関数
int main() {
    // プレイヤーを生成する関数が呼び出される
    auto p = create(Player::Hero, 10, 1.0f, 3);

    // 敵を生成する関数が呼び出される
    auto e = create(Enemy::Boss, 10, 1.0f);

    // NPCを生成する関数が呼び出される
    auto n = create();

    return 0;
}

    少々回りくどい実装になっているが、
    create 関数を呼び出す際の引数を変えると、違うクラスのインスタンスが取得出来る様になっている。

    create 関数は引数の型や数に依存せず、
    どの様な呼び出し方でも受け付けるようになっており、
    パラメータパックを展開して createChara を呼び出す事で、
    展開されたパラメータパックに合致する createChara が呼び出され、
    それぞれ違う戻り値を返している。

    展開されたパラメータパックに合致する createChara がない場合はコンパイルエラーになる。
