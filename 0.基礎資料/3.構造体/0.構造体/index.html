    構造体

    構造体を話す前に

    本題に入る前に、まずどういった時に有用なのかを話しておく。

    実際にゲームを作る際、プレイヤーや敵、弾など様々な物をプログラムで作る事になる。
    当然それらが持つ情報もプログラムで用意する必要がある。

    例えば、プレイヤーが持つパラメータであれば、

    • HP
    • MP
    • 攻撃力
    • 防御力
    • 魔力
    • 素早さ

    等々が挙げられる。
    これらはすべて変数で保持する必要がある。

    パラメータを保持する変数の宣言

    試しにプレイヤーのパラメータを表示するプログラムを作る。

    表示される各パラメータはそれぞれ変数に代入し、
    printf でその変数を指定して表示する。

    • HP：100
    • MP：50
    • 攻撃力：11
    • 防御力：15
    • 魔力：20.5
    • 素早さ：A
    #include <stdio.h>
// ---------------------------------------------------------
/**
 * @brief   main 関数
 */
int main() 
{
    int hp = 100;
    int mp = 50;
    int attack = 11;
    int defense = 15;
    float magic = 20.5;
    char speed = 'A';

    printf("HP:%d", hp);
    printf("MP:%d", mp);
    printf("攻撃力:%d", attack);
    printf("防御力:%d", defense);
    printf("魔力:%2.1f", magic);
    printf("素早さ:%c", speed);

    return 0;
}

    ここからさらに、敵のパラメータも用意する。
    敵のパラメータは下記の通り。

    • HP：110
    • MP：20
    • 攻撃力：30
    • 防御力：1
    • 魔力：0
    • 素早さ：B

    こちらも敵用の変数をそれぞれ用意し、
    printf で変数を指定して表示する。

    printf の部分は関数する。
    各パラメータを引数で渡し、関数内でそれぞれ printf を行う。
    先程用意したプレイヤーのパラメータもその関数を利用して表示するように変更する。

    // ---------------------------------------------------------
/**
 * @brief   各パラメータを表示する関数
 * @param   hp      HP
 * @param   mp      MP
 * @param   speed   素早さ
 * @param   attack  攻撃力
 * @param   defense 防御力
 * @param   magic   魔力
 */
void DispParameter(int hp, int mp, char speed, int attack, int defense, float magic)
{
    printf("HP:%d\n", hp);
    printf("MP:%d\n", mp);
    printf("素早さ:%c\n", speed);
    printf("攻撃力:%d\n", attack);
    printf("防御力:%d\n", defense);
    printf("魔力:%2.1f\n", magic);
}

// ---------------------------------------------------------
/**
 * @brief   main 関数
 */
int main() 
{
    int player_hp = 100;
    int player_mp = 50;
    char player_speed = 'A';
    int player_attack = 11;
    int player_defense = 15;
    float player_magic = 20.5;

    // プレイヤーのパラメータ表示
    DispParameter(player_hp, player_mp, player_speed, player_attack, player_defense, player_magic);

    int enemy_hp = 110;
    int enemy_mp = 20;
    char enemy_speed = 'B';
    int enemy_attack = 30;
    int enemy_defense = 1;
    float enemy_magic = 0;

    // 敵のパラメータ表示
    DispParameter(enemy_hp, enemy_mp, enemy_speed, enemy_attack, enemy_defense, enemy_magic);

    return 0;
}

    これでやりたい事は実現できている。

    間違いの起きやすいプログラム

    ただこのコードは間違いが起こりやすいといえる。

    仮に NPC など別のキャラクタのパラメータを新規で用意する事になった時、
    同じような変数がさらに追加され、
    各パラメータの変数を宣言する場所によっては容易に間違ってしまう。

    とにかく宣言された変数が多く、纏まっておらず、
    どの変数が誰の何なのかが一見して分かりにくくなる。
    自分が作ったものを第三者に見せたとき、
    「結局誰がどんなパラメータを持ってて、値はどうなってる？」といった事を言われてしまう。

    もう一点着目してもらいたい点として、
    プレイヤーも敵も保持される値は違えど持っているパラメータの種類が同じ事が挙げられる。

    両方とも

    • HP
    • MP
    • 攻撃力
    • 防御力
    • 魔力
    • 素早さ

    のパラメータで成り立つ。

    つまりこのパラメータの集まりを一つの物として扱い、
    それをプレイヤー用、敵用として分けて用意できると、
    シンプルな形にする事が出来る。

    そんな時に 構造体 という仕組みが役に立つ。

    構造体とは

    構造体 とは、 複数の変数を一つに纏めて繋げた物 の事を指す。
    複数を一つに纏めるというのは、配列と似たような説明だが全く違う物。

    一つの変数は、その変数型の値を一つしか保持できない。
    配列は「同じ型の変数」を複数繋げた物であり、
    別の型の変数を配列内に混ぜる事は出来ない。

    「要素数 4 の int 型の配列の 2 番目だけ float にする」といった事は出来ない。

    構造体を利用すると、型の違う複数の変数を一つにまとめて、
    それを 自分がオリジナルで作成した新しい型 として使用できる様になる。

    構造体の定義

    構造体を作成してみる。

    // 構造体「パラメータ型」を定義する
struct Parameter
{
    int hp;         // HP
    int mp;         // MP
    char speed;     // 素早さ
    int attack;     // 攻撃力
    int defense;    // 防御力
    float magic;    // 魔法
};

    structというキーワードが付いているが、
    これが構造体を用意するときに使用するキーワード。

    次のParameterが、構造体の型名を表す。

    間違えてはいけないのは、Parameterは変数ではなく型になる。 つまり int や floatと同じ。

    今までコンピュータが用意している型しか利用してこなかったが、
    構造体にする事で自分の好きな型を作って利用できるという事になります。

    そしてParameter 型のなかに、それぞれのパラメータを保持する変数を宣言する。
    これらの変数は メンバー、またはフィールド と呼ばれる。

    この様に構造体型を用意する事を構造体を定義する という。

    構造体の利用

    先程のプレイヤーと敵のパラメータを構造体を利用した形に変更する。
    まずは先ほど用意したParameter 型の変数を宣言する。

    型の中で宣言したメンバーを利用する為には、
    ドット演算子 を利用する。

    // プレイヤー用のパラメータ型変数を宣言
Parameter player_parameter;
// プレイヤーパラメータのメンバー変数に値を代入する
player_parameter.hp = 100;
player_parameter.mp = 50;
player_parameter.speed = 'A';
player_parameter.attack = 11;
player_parameter.defense = 15;
player_parameter.magic = 20.5;
// プレイヤーパラメータを表示
DispParameter(player_parameter.hp, player_parameter.mp, player_parameter.speed,player_parameter.attack, player_parameter.defense, player_parameter.magic);

// 敵用のパラメータ型変数を宣言
Parameter enemy_parameter;
// 敵パラメータのメンバー変数に値を入れる
enemy_parameter.hp = 110;
enemy_parameter.mp = 20;
enemy_parameter.speed = 'B';
enemy_parameter.attack = 30;
enemy_parameter.defense = 1;
enemy_parameter.magic = 0;
// 敵パラメータを表示
DispParameter(enemy_parameter.hp, enemy_parameter.mp, enemy_parameter.speed, enemy_parameterattack, enemy_parameter.defense, enemy_parameter.magic);

    先程の例では至る所で変数宣言していたが、
    今回宣言している変数は、プレイヤー、敵ともに、
    自作した構造体である Parameter 型の変数一つだけとなる。

    Parameter 型が hp や mp といったメンバーを持っている為、
    一つ宣言するだけで、必要な変数をすべて利用できるようになる。

    ここにNPCのパラメータを追加したいとなったら、
    Parameter 型の変数を、NPC用にもう一つ宣言すれば良いだけ。

    複数の変数を用途に合わせて一つに纏められる というのが構造体の利点。

    構造体の複製

    構造体型の変数は、当たり前ですが変数になる。
    つまり代入が可能。

    Parameter temp_parameter = player_parameter;

    この場合 player_parameter の中身が全て temp_parameter に複製される。
    つまりこの二つはメンバー含めて全く同じ内容となる。

    勿論、特定のメンバーのみ代入する事も可能。

    temp_parameter.hp = player_parameter.hp;

    この場合はHPのみが複製される。

    これは複製なので同じ内容を持つ別物が作られている。
    その為 temp_parameter の内容を書き換えても player_parameter に変化はない。

    構造体型の引数

    ここまで進めると次に気になるのがDispParameter関数になるかと思う。
    とにかく引数の数が多い。

    ただ現状、種類ごとに必要なパラメータは構造体として一つの型に纏まっている。
    という事は、関数の引数を構造体型にすれば、渡す引数が一つで済むことになる。

    // ---------------------------------------------------------
/**
 * @brief   各パラメータを表示する関数
 * @param   parameter 各パラメータ
 */
void DispParameter(Parameter parameter)
{
    printf("HP:%d\n", parameter.hp);
    printf("MP:%d\n", parameter.mp);
    printf("素早さ:%c\n", parameter.speed);
    printf("攻撃力:%d\n", parameter.attack);
    printf("防御力:%d\n", parameter.defense);
    printf("魔力:%2.1f\n", parameter.magic);
}

    // プレイヤーパラメータを表示
DispParameter(player_parameter);
// 敵パラメータを表示
DispParameter(enemy_parameter);

    関数の引数を、自作したParameter 型の変数一つだけに変える。

    これによってParameter 型の内容を丸々引数として受け取るようになり、
    Parameter 型に纏められているメンバも関数内に渡す事が出来るようになる。

    構造体の初期化

    構造体を宣言した後、各パラメータを指すメンバーに値を代入している。
    配列と同様に、構造体も宣言と同時に代入、つまり初期化が可能。

    書き方も配列と似ている。

    // プレイヤー用のパラメータ型変数を宣言と同時に初期化
Parameter player_parameter = {
    100,
    50,
    'A'
    11,
    15,
    20.5f,
};

    ただし数値を記述する順番には注意が必要。

    配列と同じく、カンマ区切りで初期値を代入する事は一緒だが、
    構造体の場合 その初期値がどのメンバーに入るのか を意識しなければならない。

    初期値の指定順は構造体を定義した時のメンバーの順番に依存する。
